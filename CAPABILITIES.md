## Engine вАФ source pipeline

### lumen-dom (`crates/engine/dom`)
- вЬЕ Arena DOM: `Vec<Node>` + `NodeId(u32)`, no `Rc/RefCell`, no cycles (deny-linted).
- вЬЕ Node model: Document / Doctype / Element / Text / Comment / ShadowRoot; `QualName`, 6 namespaces, attributes.
- вЬЕ Core API: create/append/detach/get, `base_href()`, `find_first_element(predicate)`, tree-print.
- вЬЕ `InputType` (22 HTML5 input types), `DocumentMode` (NoQuirks/Quirks/LimitedQuirks, set by parser).
- вЬЕ Shadow DOM: `attach_shadow`, `FlatTree` + `build_flat_tree` with `<slot name>` assignment.
- вЬЕ Hibernation snapshot: `Document::to_bytes()/from_bytes()` (bincode); JS-wrapper refcounting for GC (`acquire/release_js_ref`, `dead_node_ids()`).
- вЬЕ Drag-and-drop draggability; contenteditable editing layer (`Range`, `Selection`, `CommandHistory` undo/redo, paste/drag transfer).
- вђЬ Arena compaction / free-list; auto-set `:target` from URL fragment is shell-side.
- ~232 tests.

### lumen-html-parser (`crates/engine/html-parser`)
- вЬЕ Iterator FSM tokenizer (RAWTEXT/RCDATA/DOCTYPE/comments); all 23 HTML5 insertion modes.
- вЬЕ ~250 named entities + numeric refs; DOCTYPE public/system id; quirks-mode detection (detection only).
- вЬЕ `srcset` + `sizes` (media conditions incl. `prefers-color-scheme`), `<picture>`/`<source>` selection.
- вЬЕ Preload scanner (`scan_preload_hints`); push/incremental parsing (`PushTokenizer`, `IncrementalTreeBuilder` with partial-UTF-8 buffering, byte-equal to pull parse).
- вЬЕ Declarative Shadow DOM (`<template shadowrootmode>`).
- вђЬ CDATA, legacy entities without `;`, `<plaintext>`/`<noembed>`, `calc()` in sizes, `loading="lazy"`.
- ~394 tests.

### lumen-css-parser (`crates/engine/css-parser`)
- Parses selectors + **untyped string declarations**; typed values + cascade live in `lumen-layout/style.rs` (~139 properties wired end-to-end вАФ see CSS-SPECS.md).
- вЬЕ Selectors L3 full set + L4: attribute operators (`= ~= |= ^= $= *=`, case flag), structural pseudo, form/UI-state pseudo (DOM-attr-based), `:nth-*(of вА¶)`, `:not/:is/:where`, `:has` (in layout).
- вЬЕ `:lang/:dir/:link/:visited(always false)/:scope/:target/:state()` (custom-state, matches `data-lumen-state-<name>` sentinel attr reflected by `ElementInternals.states`); interactive pseudo (`:hover/:focus`) parsed as always-false (runtime state applied in layout).
- вЬЕ `!important` extraction; at-rules parsed+stored: `@media` (cascade-integrated), `@font-face`, `@import`, `@property`, `@layer`, `@supports` (typed `evaluate()` вАФ incl. `selector()`, `font-tech()`/`font-format()` matched against lumen-font capabilities), `@keyframes`, `@scope`, `@container`, `@color-profile` (`ColorProfileRule`), `@function` (`FunctionRule`, call-site evaluation wired in layout вАФ see CSS-SPECS.md).
- вЬЕ `@import` **file loading** (shell `inline_css_imports`, 2026-07-18): parsed `@import` URLs are fetched (local file / http via the prefetch cache) and their rules inlined **before** the importing sheet's own rules (Cascade L4 ¬І6.5); media-query gate skips non-matching imports, cycle + depth (16) guards, nested imports resolve against the importing sheet's own URL.
- вЬЕ Cascade wiring for `@layer` (layer priority in `layer_pri`)/`@scope` (`node_in_scope`: root + donut `to (<limit>)` resolved in one ancestor walk, nearest boundary wins)/`@container` (`apply_container_rules`) вАФ all applied in `compute_style`/`box_tree.rs`. Nested `@media`/`@supports`/`@layer`/`@container`/`@scope` inside another conditional-group at-rule body now parse too (bubble to stylesheet-level, flat model, done 2026-07-15).
- вђЬ Namespace prefixes.
- ~304 tests.

### lumen-encoding (`crates/engine/encoding`)
- вЬЕ Decoders: Windows-1251, KOI8-R, CP866, UTF-16 LE/BE (surrogates), UTF-8; BOM strip; `from_label` (WHATWG aliases).
- вЬЕ `detect()` chain: BOM вЖТ `<meta charset>` вЖТ HTTP hint вЖТ valid-UTF-8 вЖТ Russian-frequency heuristic.
- вЬЕ ICU4x 2.2 unicode provider (line-break UAX#14, grapheme/word UAX#29, bidi UAX#9); Knuth-Liang hyphenation (11 locales, used when `hyphens: auto`).
- вђЬ ISO-8859-5, MacCyrillic, full HTML5 prescan, UTF-32.
- ~90 tests.

## Engine вАФ layout & rendering

### lumen-layout (`crates/engine/layout`)
- вЬЕ Block + inline flow (line wrap, margin collapsing, `margin: 0 auto`, `line-height-step` vertical rhythm).
- вЬЕ `white-space` (normal/nowrap/pre/pre-wrap/pre-line/break-spaces) + CSS Text L4 longhands `white-space-collapse` (collapse/preserve/preserve-breaks/preserve-spaces/break-spaces) and `text-wrap-mode` вАФ the legacy value is the L4 shorthand, layout reads an effective `WhiteSpace` recomputed via `WhiteSpace::combine`; `pre-line`/`preserve-breaks` keep source `\n` as forced breaks while collapsing surrounding spaces (`preserve-spaces` вЙИ `preserve`, Phase 0).
- вЬЕ Flexbox (full: direction, grow/shrink/basis, justify/align, gap, wrap). вђЬ column-direction wrapping.
- вЬЕ CSS Grid (px/fr/auto/repeat/minmax, explicit+auto placement, dense, subgrid, `order`). вђЬ grid-template-areas, named lines. `grid-template-*: masonry` / `display: masonry` fall back to a regular grid / multicol (matches Edge, which ships no CSS masonry).
- вЬЕ Multi-column (`column-count`/`column-width`/`column-gap`/`column-rule`/`column-span`, `column-fill: balance|auto` вАФ balanced atomic-box distribution via binary-searched column height).
- вЬЕ Table layout (colspan/rowspan, column widths) вАФ live path `box_tree.rs` (note: `table.rs` is dead code).
- вЬЕ `border-radius` elliptical corners (`10px / 20px` syntax, CSS Backgrounds L3 ¬І5.5): separate x/y radius fields per corner (`ComputedStyle::border_top_left_radius_y` etc.), threaded through `CornerRadii::from_style_and_box` into both the femtovg and CPU raster paths.
- вЬЕ Positioned: relative, absolute/fixed (out-of-flow + containing-block threading); `position: sticky` partial (offsets computed, scroll wiring shell-side).
- вЬЕ SVG layout pass (viewBox, rect/circle/ellipse/line/path, `<use>` with cycle detection); `<text>` with `text-anchor`/`dominant-baseline` (presentation attribute **and** CSS property вАФ CSS overrides the attribute and inherits from `<g>`); vertical writing modes (`vertical-rl/lr`).
- вЬЕ Replaced: `<img>` (picture/srcset picker), `<iframe>` placeholder.
- вЬЕ Cascade: specificity + `!important`, RTL selector matching, all CSS3 structural + L4 form/UI pseudo, `:has()`, `:state()` (custom-state, matches `data-lumen-state-<name>` sentinel attr), `::before/::after` (string content), `::first-line/::first-letter` (drop-cap float), `initial-letter` (size/sink drop cap, Phase 0).
- вЬЕ Values: `calc/min/max/clamp` + math fns, `var()`, `@property` registration, viewport units, intrinsic sizing (`min/max/fit-content`); `@function` custom-function calls (`--name(<args>)`, positional args + defaults, local decls, `result:` вАФ direct calls and calls reached through a custom-property chain both resolve; `returns` typing deferred).
- вЬЕ Forced Colors Mode (CSS Color Adjust L1 ¬І3): a11y-panel preference forces the system palette element-aware (LinkText/ButtonText/GrayText/Field), clears shadows, drops non-`url()` background images, honours `forced-color-adjust: auto|none|preserve-parent-color`; drives `@media (forced-colors: active)`. JS `matchMedia` shim not yet wired.
- вЬЕ Animations/transitions scheduling (`@keyframes` interpolation, timing functions, transform/gradient/filter interpolation; `background:<color>` shorthand in keyframes); `content-visibility: auto` skip; Shadow DOM flat-tree integration.
- вЬЕ Scroll-Driven Animations L1: `animation-timeline: scroll()|view()|<named>` drives animation progress from scroll/viewport position (not the clock) вАФ opacity/transform/animated `background-color`/`color` all composite in the live window (BUG-231, 2026-06-23). Remaining deviation vs Edge is font-parity/sub-pixel only (BUG-127, KNOWN_DEBTOR).
- рЯЯ° `font-palette` + `@font-palette-values` (CSS Fonts L4 ¬І11.3/¬І13, 2026-07-04): `normal|light|dark|<dashed-ident>` parsed (inherited); custom idents resolved against `@font-palette-values` (name + family match, `base-palette`, `override-colors`) at the end of `compute_style`; carried as `DrawText.font_palette`. Renderer ignores the field вАФ lumen-font has no COLR/CPAL rasterization (color glyphs deferred), so palettes have no visual effect yet.
- рЯЯ° `ruby-position` / `ruby-align` / `ruby-merge` (CSS Ruby L1 ¬І4, 2026-07-04): parsed as inherited `ComputedStyle` fields (`over|under`, `start|center|space-between|space-around`, `separate|merge|auto`); `RubyBox::from_style` drives `lay_out_ruby` (align distribution + separate/merge per-base pairing). No visual effect yet вАФ `<ruby>` elements are not built into the box tree (module has no pipeline callers; P1 inline-flow integration deferred).
- рЯЯ° `math-style` / `math-depth` (MathML Core ¬І2.1, 2026-07-04): parsed as inherited `ComputedStyle` fields (`normal|compact`; `auto-add | add(<integer>) | <integer>` resolved to a computed integer against the inherited depth); `collect_mathml_structure` carries `math-style` into `MathmlBox`, `lay_out_mathml` scales compact mfrac children and derives script scale from `math-depth` deltas (`MATH_SCRIPT_SCALE^ќФ`, 0.71/level). No visual effect yet вАФ `<math>` elements are not built into the box tree (module has no pipeline callers; P1 integration deferred); `font-size: math` not implemented.
- вЬЕ `backface-visibility` (CSS Transforms L2 ¬І5.1, 2026-07-04 parse + 2026-07-05 culling): `visible|hidden` parsed into a non-inherited `ComputedStyle::backface_visibility` field; `is_backface_hidden()` (paint/display_list.rs) culls the box (and subtree) once its own 3D transform's `forward_box_transform().0[10]` goes negative (face normal `(0,0,1)` rotated past 90¬∞ from the viewer) вАФ same matrix already used by depth-sort, no GPU dependency; `perspective`/`perspective-origin` projection wiring still вђЬ (P4).
- вЬЕ CSS Anchor Positioning L1 (`anchor-name`/`position-anchor`/`inset-area`/`anchor()`/`anchor-size()`): post-layout pass (`apply_anchor_positions`) repositions anchored elements; `anchor()` in `top`/`right`/`bottom`/`left` and `anchor-size()` in `width`/`height` resolved through the same two-phase local+global registry (`resolve_inset`/`resolve_inset_scoped`, `resolve_anchor_size`, 2026-07-15). CSS Grid subgrid (`SubgridContext`): child track sizes/gaps/offsets inherited from the parent grid via thread-local propagation in `box_tree.rs`. Both fully wired, not stubs.
- вЬЕ `%` lengths in block flow (RP-1): `width`/`margin-*`/`padding-*` resolve against the containing block's **width** (vertical margin/padding included, per CSS 2.1 ¬І8); percentage `height` resolves against the CB **height** when definite, else computes to `auto`.
- вЬЕ `float: left/right` + `clear` (RP-4, CSS 2.1 ¬І9.5): placement, shrink-to-fit auto width, wrap to next line (rule 8), float context establishes a BFC, container height encloses floats, inline text wraps around floats, `clear` clearance (margin-absorbing), `shape-outside` (circle/ellipse/inset/polygon/path). A non-BFC block beside a float keeps its full containing-block width вАФ only its (and its descendants') line boxes are shortened, propagated into nested in-flow blocks; BFC blocks shift past floats; `clear` and nested floats work at depth. Remaining edge: per-line widening of an inline run *below* a shorter float (single-width approximation).
- вЬЕ `ch`/`ex` units (CSS Values L4 ¬І5.1.1): resolve against the box's real font metrics вАФ `ch` = advance of the "0" glyph, `ex` = x-height at the used font-size вАФ published per box by `lay_out_inner` via the `FONT_CH_EX` thread-local; the spec `0.5em` assumption is the fallback when no measurer is available. `cap`/`lh` remain em-approximated.
- вђЬ real `direction: rtl` reordering, CSS4 color spaces (lab/lch/oklab/oklch), `attr()`/`counter()` content. Many L3/L4 properties are **parse+store only** (text-emphasis, container queries, touch-action, appearance, resize).

### lumen-paint (`crates/engine/paint`)
- вЬЕ **Live default render path is `WgpuBackend`** (probe-selected: Vulkan вЖТ GL вЖТ DX12 on Windows, ADR-017); femtovg is the init-failure fallback and explicit `LUMEN_BACKEND=femtovg` override. `LUMEN_BACKEND` env var overrides the probe. **Paint bugs from graphic_tests on the wgpu path are fixed in `renderer.rs`; femtovg-specific issues in `femtovg_backend.rs`.**
- вЬЕ **Wide-gamut output surface (wgpu backend only)** вАФ wgpu renderer selects a wide-gamut swap-chain format (`Rgba16Float` for DisplayP3/Rec2020, non-sRGB fallback for sRGB) based on the display profile queried at startup (`Lumen::target_color_space()`); canvas background is colour-managed from sRGB into the destination encoding before the clear op commits the frame. Femtoovg remains 8-bit sRGB (platform API limitation).
- вЬЕ **Target-aware image decode pipeline (ph3 Step 6)** вАФ `lumen-image::decode_to(bytes, target)` colour-converts decoded RGB images to the display target (DisplayP3/Rec2020/sRGB) at decode time, replacing the previous sRGB-only path. Shell propagates `self.target_color_space()` through all image decoders (`decode_image`, `fetch_and_decode_images`, `fetch_and_decode_background_images`, `render_bytes`, `parse_and_layout`). 8-bit precision retained; per-pixel converters for P3вЖФRec2020 added alongside the existing ICC matrix-shaper path.
- вЬЕ DisplayCommand primitives (all in enum + handled by femtovg): FillRect, FillRoundedRect (SDF), DrawBorder (solid/dashed/dotted/double), DrawText, DrawOutline, DrawImage (object-fit/position), Linear/Radial/Conic gradients (incl. repeating; radial `circle`/`ellipse` ending shape + size keywords via `radial_gradient_radii`, CSS Images L3 ¬І3.5 вАФ BUG-239; repeating/radial rendered per-pixel to bypass the 256-texel LUT вАФ BUG-085; premultiplied stop interpolation for fades to `transparent`, CSS Images L4 ¬І3.1 вАФ BUG-190), SvgPath, clip, opacity, blend modes, transforms, filters, backdrop-filter, scroll layers, masks, layer snapshots, page breaks.
- вЬЕ Stacking contexts + paint order (CSS 2.1 Appendix E), stacking-aware hit testing (transform inversion).
- вЬЕ `isolation: isolate` (CSS Compositing & Blending L1 ¬І2.1) вАФ forms an isolated blend group: `box_layer_ops` emits a full-alpha offscreen group (`PushOpacity{1.0}`) so a descendant `mix-blend-mode` composites against the group's transparent backdrop, not the page behind it (only when `isolate` is the sole group trigger вАФ opacity/filter/backdrop-filter/mix-blend-mode already isolate). Correct on CPU/femtovg (pixel-identical to Edge); the wgpu window inherits the mix-blend divergence BUG-277.
- вЬЕ CSS Motion Path L1 (`offset-path: path()`/`ray()`, `offset-distance`, `offset-rotate` auto/reverse/fixed, `offset-anchor`) вАФ box's anchor placed on the path point, rotated around it (`forward_box_transform` + property-tree `walk`). TEST-76 boxes pixel-identical to Edge.
- вЬЕ box-shadow (outset+inset), text-shadow, text-decoration (underline/overline/line-through, wavy/dotted/dashed/double, thickness), border-radius SDF.
- вЬЕ CSS filters (GPU color-matrix + Gaussian blur), backdrop-filter (LRU cache), clip-path (bbox approximation).
- вЬЕ 3D transforms (perspective, preserve-3d depth sort) in wgpu renderer; multi-size + variation-aware glyph atlas, per-char codepoint fallback cascade.
- вЬЕ **Synthetic bold/italic fallback (RP-6, 2026-07-02)** вАФ when no true bold/italic face is available, `font-weight >= 600` is emulated by a double glyph blit with a small horizontal offset and `font-style: italic/oblique` by a ~12¬∞ shear, in **both** text engines: CPU/headless (`cpu_raster::rasterize_text` вАФ outline shear + double coverage blit; the bundled Inter Regular finally renders headings bold) and the live femtovg window (`draw_text_styled` вАФ second `fill_text` + `Transform2D` shear about the baseline; skipped when `FontProvider` resolves a real bold/italic face). Advance metrics are intentionally unchanged вАФ layout is unaffected.
- вЬЕ **`writing-mode` vertical text glyph rotation (P3-vertical, —Б—А–µ–Ј—Л 1вАУ4)** вАФ CPU rasterizer (`cpu_raster.rs`) and wgpu renderer (`renderer.rs`, live default) both honor `text-orientation`: `sideways` rotates the whole run 90¬∞ CW; `mixed` (initial) does a per-glyph split (`display_list::split_mixed_runs` + `lumen_layout::vertical::is_cjk`) вАФ CJK ideographs paint upright, non-CJK runs rotate, shaped as one block each to keep Latin kerning/ligatures; `upright` stays unrotated. femtovg (fallback backend) still ignores `text_orientation` вАФ out of scope (`docs/tasks/ph3-writing-mode-vertical.md`).
- вЬЕ Compositor scaffolding (two-buffer commit, threaded compositor, 60fps vsync); print (`render_print_pages` вЖТ images); CPU rasterizer (`cpu_raster.rs`, feature `cpu-render`, cross-OS bit-identical, snapshot gate; **femtovg-parity for `<img>` decode+`object-fit`+area-averaged downscale, circular **and elliptical** radial gradients (`circle`/`ellipse` shape+size вАФ BUG-239), clamped `border-radius` fills вАФ BUG-221; `background-image` url()/`image-set()` tiling (`background-size`/`position`/`repeat` via shared `bg_tile_geometry`) and `cross-fade()` blending**); software WebGL 1.0 (GLSL ES vertex+fragment interpreter executes вАФ `glsl.rs`; framebuffer readable via `readPixels`, not yet presented to the page `<canvas>`).
- рЯЯ° femtovg `mask-image` gradient masks are **true per-pixel alpha masks** (offscreen FBO + `DestinationIn`, linear/radial/conic; BUG-183). `mask-mode: luminance` вЬЕ wired (BUG-218): `emit_push_mask` bakes `luminance(rgb)¬Јalpha` into each gradient stop's alpha, so both femtovg and CPU paths honour it. `mask-origin` вЬЕ wired (2026-06-22): the mask positioning rect comes from `background_origin_rect` (border/padding/content box, initial `border-box`); `mask-position` threaded into `PushMaskImage` (initial `center`). `url()` image masks still scissor bbox (no decoded source), so `mask-position`/`mask-repeat` tiling have no visible effect yet. `mask-clip` вЬЕ wired: full `<coord-box> | no-clip` grammar (2026-07-18): `padding-box`/`content-box`/`fill-box` wrap the mask group in a `PushClipRect`/`PopClip` pair (`mask_clip_paint_rect`), reusing the scissor path (`fill-box` = content box for CSS boxes, CSS Box 4 ¬І1); `border-box`/`stroke-box`/`view-box`/`no-clip` stay no-ops (border box or unclipped). `mask-composite` (multi-layer) not wired.
- вђЬ GPU shadow pipeline, Groove/Ridge/Inset/Outset borders, exact polygon clip-path, Vello backend (no-op stub).

### lumen-font (`crates/engine/font`)
- вЬЕ Table parsers (head/maxp/cmap fmt4+12 incl. SMP/emoji/hhea/hmtx/loca/glyf/name/OS2/post); rasterizer (simple + composite glyphs, 4√Ч4 supersampling).
- рЯЯ° Variable fonts runtime (fvar/avar/HVAR/VVAR/MVAR/gvar, IUP + deltas) вАФ applied on the **CPU rasterizer / wgpu paths** (`--screenshot`, snapshot gate) via `apply_variations_to_simple_outline`. **Not** wired into the live femtovg window (femtovg renders the default instance via its own `fill_text`), same fork as GSUB/GPOS shaping below. font matching/fallback (`SystemFontIndex` scans OS fonts, weight/style matcher, per-char cascade); WOFF2 (Brotli) + WOFF1 (zlib) decode.
- вЬЕ **`font-display: swap` (PH3-19)**: `@font-face url()` sources fetched asynchronously off the critical paint path (FOUT). First paint uses Inter fallback; background thread fetch+decode вЖТ `FontLoaded` event вЖТ relayout with `MultiFontMeasurer` to swap in the web font. `local()` sources still loaded synchronously (no network round-trip needed).
- рЯЯ° **Shaping (GSUB/GPOS) вАФ U-2 stage 1**: `Shaper::shape()` applies GSUB ligatures (Type 1 single + Type 4 ligature, incl. Type 7 extension) and GPOS kerning (Type 1 single + Type 2 pair, formats 1/2, incl. Type 9 extension) for Latin/Cyrillic; default features `liga`/`clig`/`calt`/`rlig`/`ccmp` (GSUB) + `kern` (GPOS). Wired into the **CPU rasterizer** (`render_to_image_cpu` вЖТ `--screenshot`, snapshot gate). **Not** wired into the live femtovg window (femtovg shapes via its own `fill_text`) nor the per-char layout measurement. Out of scope: contextual lookups (GSUB 5/6, GPOS 7/8), mark positioning (GPOS 3вАУ6), complex scripts (Arabic/Indic), LookupFlag mark filtering. **CSS `font-feature-settings` honoured** (2026-07-04): `Shaper::with_features()` / `Gsub::parse_with_features()` / `Gpos::parse_with_features()` apply CSS overrides on top of the default feature set (`(tag, 0)` disables вАФ e.g. `"calt" 0`, `"kern" 0`; `(tag, >=1)` enables вАФ e.g. `"dlig" 1`); carried by `DrawText.font_features`, applied on the CPU rasterizer and the femtovg varied-text (variable-font outline) path; native femtovg text ignores it (same fork as above).
- вЬЕ **CFF outlines (`.otf` PostScript) вАФ U-2 stage 2**: `lumen-font::cff` parses the `CFF ` table (INDEX/DICT, Private DICT, global+local subrs) and interprets Type 2 charstrings (all path/hint operators, the four flex ops, subr bias, `seac` composites). Cubics are flattened to on-curve segments so the existing rasterizer is reused. CID-keyed fonts (`ROS`/`FDArray`/`FDSelect` fmt 0+3) supported. Routed transparently through `Font::glyph_resolved`, so CPU raster, the wgpu renderer, and Canvas 2D all draw `.otf` text. Deferred: CFF2 (variable PostScript), charstring arithmetic ops.
- вђЬ No hinting, no color glyphs (COLR/CPAL/sbix), no bitmap strikes. Fallback covers only already-loaded faces.

### lumen-image (`crates/engine/image`)
- вЬЕ PNG, JPEG (baseline + progressive), WebP (VP8 + VP8L), **GIF** (static + animated), **AVIF** (behind `avif` feature).
- вЬЕ **External SVG images (RP-5, 2026-07-02)** вАФ `<img src=*.svg>` and `background-image: url(*.svg)` render. `lumen_image::is_svg` sniffs SVG markup (BOM/whitespace skip; `<svg`, `<!DOCTYPE svg`, or `<?xml` prolog + `<svg` in the first 4 KB, case-insensitive; `image/svg+xml` added to `supported_mime_types`); the shell (`svg_image.rs`) wraps the markup in a minimal HTML document and renders it through the normal deterministic layout/paint pipeline (`parse_and_layout` вЖТ `paint_ordered` вЖТ `render_to_image_cpu`) at the intrinsic size (`width`/`height` attrs вЖТ `viewBox` вЖТ CSS default 300√Ч150, clamped 1..=4096). Deferred: re-raster at CSS box size (icons upscale from intrinsic), `currentColor` from the host element.

(4) `Target::Point` (BiDi `input.performActions` pointer coordinates, MCP `click{point}`) was passed straight through unconverted вАФ off by exactly `TAB_BAR_HEIGHT` (36px) since pointer coordinates are in *content-viewport* space (the same space `captureScreenshot` renders вАФ no scroll subtraction needed, unlike `NodeId`/`Selector`) вАФ now gets the tab-bar/panel offset added. Verified end-to-end via both protocols: navigate вЖТ click a real `<a href>` (by selector, and separately by raw pointer coordinates read straight off a screenshot) вЖТ real cross-page navigation, confirmed by screenshot/`a11y_tree`/`query()`.

### lumen-driver (`crates/driver`) вАФ headless engine interface
- вЬЕ `BrowserSession` trait: 6 resources (screenshot/a11y_tree/layout/computed_style/network_log/console_log) + 6 tools (navigate/click/type/scroll/wait/eval/query); `InProcessSession` full headless pipeline; simple selector engine (tag/#id/.class); deterministic CPU snapshot (`screenshot_cpu_rgba/png`, cross-OS-identical, 57-page gate).
- вЬЕ `WinitSession` (SDC-1a, 8A.7 –§4 driver-side): `click`/`type_text` implement headless DOM-level semantics (click follows `<a href>` / toggles checkbox-radio `checked`; type_text writes `value`); `eval(js)` runs a one-shot QuickJS runtime bound to a snapshot of the current DOM under `--features quickjs` (returns a JSON-string result); `AutomationCommand`/`AutomationReply` published from `lumen-driver`, consumed by the live shell window (SDC-1b, done вАФ see Automation surfaces above).
- вЬЕ **DEVX-5: `InProcessSession` click/type_text/scroll/eval/screenshot** (DEVX-5 complete; cargo features `v8`/`cpu-render`, both on by default in `lumen-driver`/`lumen-shell`): `wait` real (poll loop); `scroll` wired to the existing off-main-thread compositor scroll (`scroll_page_by`); `screenshot` uses the CPU rasterizer (tiny-skia, `cpu-render`), no GPU adapter needed; `click`/`type_text` вАФ same headless DOM-level semantics as `WinitSession` (link nav, checkbox/radio toggle, `value` overwrite) but dispatched as real events (`click` synthesizes `mousedown`вЖТ`mouseup`вЖТ`click` via `_lumen_dispatch_mouse_event`, with `lumen_paint::hit_test` for `Target::Point`; `type_text` fires per-char `keydown`/`input`/`keyup` plus native `value` mutation); `eval(js)` runs against a **persistent** V8 runtime installed on the document at `navigate()` time (`Arc<Mutex<Document>>` shared with `click`/`type_text`, mirrors the shell's `V8JsRuntime::new()`+`install_dom()`) вАФ unlike `WinitSession::eval`'s one-shot runtime, JS-side DOM mutations persist across `eval()` calls within the same navigation, returning the JSON-serialized result. None of this triggers a relayout вАФ `layout_snapshot`/`screenshot` still reflect the DOM as of `navigate()`.
- вђЬ `WinitSession.eval()` without `--features quickjs` still errors; auto-wait combinators/pseudo-selectors deferred; `WinitSession` itself has no native OS-level input dispatch (that lives only in the shell window's own code path, not this headless session); DEVX-5 mutations don't trigger relayout (no interactive-JS-driven layout in headless mode).

### lumen-mcp (`crates/mcp`) вАФ Model Context Protocol server
