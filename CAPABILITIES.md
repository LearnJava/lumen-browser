## Engine вҖ” source pipeline

### lumen-dom (`crates/engine/dom`)
- вң… Arena DOM: `Vec<Node>` + `NodeId(u32)`, no `Rc/RefCell`, no cycles (deny-linted).
- вң… Node model: Document / Doctype / Element / Text / Comment / ShadowRoot; `QualName`, 6 namespaces, attributes.
- вң… Core API: create/append/detach/get, `base_href()`, `find_first_element(predicate)`, tree-print.
- вң… `InputType` (22 HTML5 input types), `DocumentMode` (NoQuirks/Quirks/LimitedQuirks, set by parser).

- вң… Hibernation snapshot: `Document::to_bytes()/from_bytes()` (bincode); JS-wrapper refcounting for GC (`acquire/release_js_ref`, `dead_node_ids()`).
- вң… Drag-and-drop draggability; contenteditable editing layer (`Range`, `Selection`, `CommandHistory` undo/redo, paste/drag transfer).
- в¬ң Arena compaction / free-list; auto-set `:target` from URL fragment is shell-side.
- ~232 tests.

### lumen-html-parser (`crates/engine/html-parser`)
- вң… Iterator FSM tokenizer (RAWTEXT/RCDATA/DOCTYPE/comments); all 23 HTML5 insertion modes.
- вң… ~250 named entities + numeric refs; DOCTYPE public/system id; quirks-mode detection (detection only).
- вң… `srcset` + `sizes` (media conditions incl. `prefers-color-scheme`), `<picture>`/`<source>` selection.
- вң… Preload scanner (`scan_preload_hints`); push/incremental parsing (`PushTokenizer`, `IncrementalTreeBuilder` with partial-UTF-8 buffering, byte-equal to pull parse).

### lumen-css-parser (`crates/engine/css-parser`)

- ~292 tests.

## Engine вҖ” layout & rendering

- вң… Multi-column (`column-count`/`column-width`/`column-gap`/`column-rule`/`column-span`, `column-fill: balance|auto` вҖ” balanced atomic-box distribution via binary-searched column height).

- вң… Positioned: relative, absolute/fixed (out-of-flow + containing-block threading); `position: sticky` partial (offsets computed, scroll wiring shell-side).
- вң… SVG layout pass (viewBox, rect/circle/ellipse/line/path, `<use>` with cycle detection); `<text>` with `text-anchor`/`dominant-baseline` (presentation attribute **and** CSS property вҖ” CSS overrides the attribute and inherits from `<g>`); vertical writing modes (`vertical-rl/lr`).
- вң… Replaced: `<img>` (picture/srcset picker), `<iframe>` placeholder.
- вң… Cascade: specificity + `!important`, RTL selector matching, all CSS3 structural + L4 form/UI pseudo, `:has()`, `::before/::after` (string content), `::first-line/::first-letter` (drop-cap float), `initial-letter` (size/sink drop cap, Phase 0).
- вң… Values: `calc/min/max/clamp` + math fns, `var()`, `@property` registration, viewport units, intrinsic sizing (`min/max/fit-content`).
- вң… Forced Colors Mode (CSS Color Adjust L1 В§3): a11y-panel preference forces the system palette element-aware (LinkText/ButtonText/GrayText/Field), clears shadows, drops non-`url()` background images, honours `forced-color-adjust: auto|none|preserve-parent-color`; drives `@media (forced-colors: active)`. JS `matchMedia` shim not yet wired.
- вң… Animations/transitions scheduling (`@keyframes` interpolation, timing functions, transform/gradient/filter interpolation; `background:<color>` shorthand in keyframes); `content-visibility: auto` skip; Shadow DOM flat-tree integration.
- рҹҹЎ Scroll-Driven Animations L1: `animation-timeline: scroll()|view()|<named>` drives animation progress from scroll/viewport position (not the clock) вҖ” opacity/transform render in the live window; animated `background-color` not yet composited (BUG-231).
- рҹҹЎ `font-palette` + `@font-palette-values` (CSS Fonts L4 В§11.3/В§13, 2026-07-04): `normal|light|dark|<dashed-ident>` parsed (inherited); custom idents resolved against `@font-palette-values` (name + family match, `base-palette`, `override-colors`) at the end of `compute_style`; carried as `DrawText.font_palette`. Renderer ignores the field вҖ” lumen-font has no COLR/CPAL rasterization (color glyphs deferred), so palettes have no visual effect yet.
- рҹҹЎ `ruby-position` / `ruby-align` / `ruby-merge` (CSS Ruby L1 В§4, 2026-07-04): parsed as inherited `ComputedStyle` fields (`over|under`, `start|center|space-between|space-around`, `separate|merge|auto`); `RubyBox::from_style` drives `lay_out_ruby` (align distribution + separate/merge per-base pairing). No visual effect yet вҖ” `<ruby>` elements are not built into the box tree (module has no pipeline callers; P1 inline-flow integration deferred).
- рҹҹЎ `math-style` / `math-depth` (MathML Core В§2.1, 2026-07-04): parsed as inherited `ComputedStyle` fields (`normal|compact`; `auto-add | add(<integer>) | <integer>` resolved to a computed integer against the inherited depth); `collect_mathml_structure` carries `math-style` into `MathmlBox`, `lay_out_mathml` scales compact mfrac children and derives script scale from `math-depth` deltas (`MATH_SCRIPT_SCALE^О”`, 0.71/level). No visual effect yet вҖ” `<math>` elements are not built into the box tree (module has no pipeline callers; P1 integration deferred); `font-size: math` not implemented.
- вң… `backface-visibility` (CSS Transforms L2 В§5.1, 2026-07-04 parse + 2026-07-05 culling): `visible|hidden` parsed into a non-inherited `ComputedStyle::backface_visibility` field; `is_backface_hidden()` (paint/display_list.rs) culls the box (and subtree) once its own 3D transform's `forward_box_transform().0[10]` goes negative (face normal `(0,0,1)` rotated past 90В° from the viewer) вҖ” same matrix already used by depth-sort, no GPU dependency; `perspective`/`perspective-origin` projection wiring still в¬ң (P4).
- вң… Algorithm stubs awaiting P4 CSS wiring: anchor positioning, subgrid context.
- вң… `%` lengths in block flow (RP-1): `width`/`margin-*`/`padding-*` resolve against the containing block's **width** (vertical margin/padding included, per CSS 2.1 В§8); percentage `height` resolves against the CB **height** when definite, else computes to `auto`.
- вң… `float: left/right` + `clear` (RP-4, CSS 2.1 В§9.5): placement, shrink-to-fit auto width, wrap to next line (rule 8), float context establishes a BFC, container height encloses floats, inline text wraps around floats, `clear` clearance (margin-absorbing), `shape-outside` (circle/ellipse/inset/polygon/path). A non-BFC block beside a float keeps its full containing-block width вҖ” only its (and its descendants') line boxes are shortened, propagated into nested in-flow blocks; BFC blocks shift past floats; `clear` and nested floats work at depth. Remaining edge: per-line widening of an inline run *below* a shorter float (single-width approximation).
- в¬ң `ch`/`ex` units, real `direction: rtl` reordering, CSS4 color spaces (lab/lch/oklab/oklch), `attr()`/`counter()` content. Many L3/L4 properties are **parse+store only** (text-emphasis, container queries, touch-action, appearance, resize).

### lumen-paint (`crates/engine/paint`)
- вң… **Live default render path is `FemtoovgBackend`** (OpenGL ES via glutin), with wgpu auto-fallback; `LUMEN_BACKEND` overrides. **Paint bugs from graphic_tests are fixed in `femtovg_backend.rs`, not `renderer.rs`.**
- вң… **Wide-gamut output surface (wgpu backend only)** вҖ” wgpu renderer selects a wide-gamut swap-chain format (`Rgba16Float` for DisplayP3/Rec2020, non-sRGB fallback for sRGB) based on the display profile queried at startup (`Lumen::target_color_space()`); canvas background is colour-managed from sRGB into the destination encoding before the clear op commits the frame. Femtoovg remains 8-bit sRGB (platform API limitation).
- вң… **Target-aware image decode pipeline (ph3 Step 6)** вҖ” `lumen-image::decode_to(bytes, target)` colour-converts decoded RGB images to the display target (DisplayP3/Rec2020/sRGB) at decode time, replacing the previous sRGB-only path. Shell propagates `self.target_color_space()` through all image decoders (`decode_image`, `fetch_and_decode_images`, `fetch_and_decode_background_images`, `render_bytes`, `parse_and_layout`). 8-bit precision retained; per-pixel converters for P3вҶ”Rec2020 added alongside the existing ICC matrix-shaper path.
- вң… DisplayCommand primitives (all in enum + handled by femtovg): FillRect, FillRoundedRect (SDF), DrawBorder (solid/dashed/dotted/double), DrawText, DrawOutline, DrawImage (object-fit/position), Linear/Radial/Conic gradients (incl. repeating; radial `circle`/`ellipse` ending shape + size keywords via `radial_gradient_radii`, CSS Images L3 В§3.5 вҖ” BUG-239; repeating/radial rendered per-pixel to bypass the 256-texel LUT вҖ” BUG-085; premultiplied stop interpolation for fades to `transparent`, CSS Images L4 В§3.1 вҖ” BUG-190), SvgPath, clip, opacity, blend modes, transforms, filters, backdrop-filter, scroll layers, masks, layer snapshots, page breaks.
- вң… Stacking contexts + paint order (CSS 2.1 Appendix E), stacking-aware hit testing (transform inversion).
- вң… CSS Motion Path L1 (`offset-path: path()`/`ray()`, `offset-distance`, `offset-rotate` auto/reverse/fixed, `offset-anchor`) вҖ” box's anchor placed on the path point, rotated around it (`forward_box_transform` + property-tree `walk`). TEST-76 boxes pixel-identical to Edge.
- вң… box-shadow (outset+inset), text-shadow, text-decoration (underline/overline/line-through, wavy/dotted/dashed/double, thickness), border-radius SDF.
- вң… CSS filters (GPU color-matrix + Gaussian blur), backdrop-filter (LRU cache), clip-path (bbox approximation).
- вң… 3D transforms (perspective, preserve-3d depth sort) in wgpu renderer; multi-size + variation-aware glyph atlas, per-char codepoint fallback cascade.
- вң… **Synthetic bold/italic fallback (RP-6, 2026-07-02)** вҖ” when no true bold/italic face is available, `font-weight >= 600` is emulated by a double glyph blit with a small horizontal offset and `font-style: italic/oblique` by a ~12В° shear, in **both** text engines: CPU/headless (`cpu_raster::rasterize_text` вҖ” outline shear + double coverage blit; the bundled Inter Regular finally renders headings bold) and the live femtovg window (`draw_text_styled` вҖ” second `fill_text` + `Transform2D` shear about the baseline; skipped when `FontProvider` resolves a real bold/italic face). Advance metrics are intentionally unchanged вҖ” layout is unaffected.
- вң… Compositor scaffolding (two-buffer commit, threaded compositor, 60fps vsync); print (`render_print_pages` вҶ’ images); CPU rasterizer (`cpu_raster.rs`, feature `cpu-render`, cross-OS bit-identical, snapshot gate; **femtovg-parity for `<img>` decode+`object-fit`+area-averaged downscale, circular **and elliptical** radial gradients (`circle`/`ellipse` shape+size вҖ” BUG-239), clamped `border-radius` fills вҖ” BUG-221; `background-image` url()/`image-set()` tiling (`background-size`/`position`/`repeat` via shared `bg_tile_geometry`) and `cross-fade()` blending**); software WebGL 1.0 (GLSL ES vertex+fragment interpreter executes вҖ” `glsl.rs`; framebuffer readable via `readPixels`, not yet presented to the page `<canvas>`).
- рҹҹЎ femtovg `mask-image` gradient masks are **true per-pixel alpha masks** (offscreen FBO + `DestinationIn`, linear/radial/conic; BUG-183). `mask-mode: luminance` вң… wired (BUG-218): `emit_push_mask` bakes `luminance(rgb)В·alpha` into each gradient stop's alpha, so both femtovg and CPU paths honour it. `mask-origin` вң… wired (2026-06-22): the mask positioning rect comes from `background_origin_rect` (border/padding/content box, initial `border-box`); `mask-position` threaded into `PushMaskImage` (initial `center`). `url()` image masks still scissor bbox (no decoded source), so `mask-position`/`mask-repeat` tiling have no visible effect yet. `mask-clip` (painting-area clip) and `mask-composite` (multi-layer) not wired.
- в¬ң GPU shadow pipeline, Groove/Ridge/Inset/Outset borders, exact polygon clip-path, elliptical border-radius (rxвү ry), Vello backend (no-op stub).

### lumen-font (`crates/engine/font`)
- вң… Table parsers (head/maxp/cmap fmt4+12 incl. SMP/emoji/hhea/hmtx/loca/glyf/name/OS2/post); rasterizer (simple + composite glyphs, 4Г—4 supersampling).
- рҹҹЎ Variable fonts runtime (fvar/avar/HVAR/VVAR/MVAR/gvar, IUP + deltas) вҖ” applied on the **CPU rasterizer / wgpu paths** (`--screenshot`, snapshot gate) via `apply_variations_to_simple_outline`. **Not** wired into the live femtovg window (femtovg renders the default instance via its own `fill_text`), same fork as GSUB/GPOS shaping below. font matching/fallback (`SystemFontIndex` scans OS fonts, weight/style matcher, per-char cascade); WOFF2 (Brotli) + WOFF1 (zlib) decode.
- вң… **`font-display: swap` (PH3-19)**: `@font-face url()` sources fetched asynchronously off the critical paint path (FOUT). First paint uses Inter fallback; background thread fetch+decode вҶ’ `FontLoaded` event вҶ’ relayout with `MultiFontMeasurer` to swap in the web font. `local()` sources still loaded synchronously (no network round-trip needed).
- рҹҹЎ **Shaping (GSUB/GPOS) вҖ” U-2 stage 1**: `Shaper::shape()` applies GSUB ligatures (Type 1 single + Type 4 ligature, incl. Type 7 extension) and GPOS kerning (Type 1 single + Type 2 pair, formats 1/2, incl. Type 9 extension) for Latin/Cyrillic; default features `liga`/`clig`/`calt`/`rlig`/`ccmp` (GSUB) + `kern` (GPOS). Wired into the **CPU rasterizer** (`render_to_image_cpu` вҶ’ `--screenshot`, snapshot gate). **Not** wired into the live femtovg window (femtovg shapes via its own `fill_text`) nor the per-char layout measurement. Out of scope: contextual lookups (GSUB 5/6, GPOS 7/8), mark positioning (GPOS 3вҖ“6), complex scripts (Arabic/Indic), LookupFlag mark filtering. **CSS `font-feature-settings` honoured** (2026-07-04): `Shaper::with_features()` / `Gsub::parse_with_features()` / `Gpos::parse_with_features()` apply CSS overrides on top of the default feature set (`(tag, 0)` disables вҖ” e.g. `"calt" 0`, `"kern" 0`; `(tag, >=1)` enables вҖ” e.g. `"dlig" 1`); carried by `DrawText.font_features`, applied on the CPU rasterizer and the femtovg varied-text (variable-font outline) path; native femtovg text ignores it (same fork as above).
- вң… **CFF outlines (`.otf` PostScript) вҖ” U-2 stage 2**: `lumen-font::cff` parses the `CFF ` table (INDEX/DICT, Private DICT, global+local subrs) and interprets Type 2 charstrings (all path/hint operators, the four flex ops, subr bias, `seac` composites). Cubics are flattened to on-curve segments so the existing rasterizer is reused. CID-keyed fonts (`ROS`/`FDArray`/`FDSelect` fmt 0+3) supported. Routed transparently through `Font::glyph_resolved`, so CPU raster, the wgpu renderer, and Canvas 2D all draw `.otf` text. Deferred: CFF2 (variable PostScript), charstring arithmetic ops.
- в¬ң No hinting, no color glyphs (COLR/CPAL/sbix), no bitmap strikes. Fallback covers only already-loaded faces.

### lumen-driver (`crates/driver`) вҖ” headless engine interface
- вң… `BrowserSession` trait: 6 resources (screenshot/a11y_tree/layout/computed_style/network_log/console_log) + 6 tools (navigate/click/type/scroll/wait/eval/query); `InProcessSession` full headless pipeline; simple selector engine (tag/#id/.class); deterministic CPU snapshot (`screenshot_cpu_rgba/png`, cross-OS-identical, 57-page gate).
- вң… `WinitSession` (SDC-1a, 8A.7 РӨ4 driver-side): `click`/`type_text` implement headless DOM-level semantics (click follows `<a href>` / toggles checkbox-radio `checked`; type_text writes `value`); `eval(js)` runs a one-shot QuickJS runtime bound to a snapshot of the current DOM under `--features quickjs` (returns a JSON-string result); `AutomationCommand`/`AutomationReply` published from `lumen-driver`, consumed by the live shell window (SDC-1b, done вҖ” see Automation surfaces above).
- в¬ң `InProcessSession`'s own click/type/scroll/wait/eval remain headless-pipeline stubs (no persistent JS runtime there); `WinitSession.eval()` without `--features quickjs` still errors; auto-wait combinators/pseudo-selectors deferred; `WinitSession` itself has no native OS-level input dispatch (that lives only in the shell window's own code path, not this headless session).

### lumen-devtools (`crates/devtools`) вҖ” CDP-over-WebSocket (Phase 0 minimal)
- вң… RFC 6455 WebSocket (handshake, frames, close/ping/pong, 1 MB guard); CDP `Browser.getVersion` (real), `DOM.getDocument` (stub), `*.enable` ACKs.
- в¬ң Real DOM tree, computed styles, Network events, Debugger domain, WSS.

### lumen-a11y (`crates/engine/a11y`)
- вң… `build_ax_tree` over Shadow-DOM composed tree (`aria-hidden` pruned); 67 ARIA roles + implicit mapping; accessible name/description (WAI-ARIA В§4); full state set + relationships; shell pushes tree after load/restore + focus-change on click.
- в¬ң Platform bridges (Windows UIA / macOS NSAccessibility / Linux AT-SPI) are **in-memory stubs** (no real OS bindings yet); live-region timing deferred.

### lumen-bidi-server (`crates/bidi-server`) вҖ” standalone WebDriver BiDi
- вң… Rich protocol state machine: session.*, browsingContext.* (create/close/navigate/getTree/captureScreenshot/setViewport/handleUserPrompt), script.* (evaluate/callFunction/preloadScript/getRealms), network.* (intercept/continue/fail/getResponseBody), input.performActions, storage.*, browser.*, emulation.setUserAgentOverride; event emission for context/storage/network.
- вң… **SDC-2 live wiring**: when `--bidi-port` is combined with an open window, each connection's `BidiState` holds a `LiveWindowSession` (`BidiState::with_live_session`) вҖ” `browsingContext.navigate`, `script.evaluate` (primitives get a real BiDi `RemoteValue`; objects/arrays fall back to their JSON text), `browsingContext.captureScreenshot` (base64 PNG), and the pointer-click/key-input subset of `input.performActions` execute for real against the live shell window.
- в¬ң Without a live window (`--bidi-port` alone, e.g. with `--dump`/`--mcp`), or for commands outside the SDC-2 MVP set (network interception, cookie events, full input-action-chain fidelity, `domContentLoaded`), state stays **in-memory `BidiState` only** вҖ” full wiring is an 8H.3 handoff.

### lumen-mcp (`crates/mcp`) вҖ” Model Context Protocol server
- вң… Wraps `BrowserSession`; `resources/list+read` (screenshot/a11y_tree/layout/console/network), `tools/list+call` (navigate/click/type/scroll/wait/eval/query).
- вң… **SDC-2 live wiring**: `lumen --mcp-live-port <N> <path-or-url>` opens a real window and serves MCP over TCP against `LiveWindowSession` вҖ” `screenshot`/`eval`/`query`/`a11y_tree`/navigate/click/type/scroll/wait return real results instead of `Err`.
- в¬ң Headless `--mcp`/`--mcp-port` (`InProcessSession`) unaffected by SDC-2: `screenshot`/`eval` still `Err` there (no persistent JS runtime, no GPU-backed rasterizer in that path); `LiveWindowSession`'s `layout_snapshot`/`computed_style`/`network_log`/`console_log`/fingerprint-isolation methods are local stub defaults вҖ” not yet threaded through the automation channel.
