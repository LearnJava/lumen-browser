**Правило:** STATUS-PN.md — это **навигационный индекс** для разработчика, не полная документация.

**Структура ссылок в STATUS-PN.md:**

Каждая задача в STATUS-PN.md должна содержать **точные ссылки на строки** в документах, где найти полную информацию:

```markdown
In progress: task-name  branch: p<N>-task-name
Next step: Реализовать X  file.rs:123 lumen-plan.md:451
```

Формат: `file:line` или `document.md:line` — разработчик знает, куда смотреть, не читает весь файл.

**Why:** 
- Разработчик может быстро найти нужную информацию (grep по номеру строки — 3 секунды)
- Не нужно пролистывать весь план/документ
- Разработчик видит только релевантную часть

**Critical rule:** Когда изменяешь lumen-plan.md, CLAUDE.md, DECISIONS.md или другой документ:
1. Обновить саму документацию
2. **НАЙТИ все ссылки на этот документ в STATUS-PN.md**
3. **ПЕРЕПРАВИТЬ номера строк, если строки сместились**

Это обслуживание документации, такое же важное как само изменение.

**Пример нарушения** (текущая ситуация):
- STATUS-P1.md:42 указывает `extras-p2: ... → lumen-plan.md:195`
- На line 195 в lumen-plan.md находится совсем другая задача (font-loading API)
- Разработчик идёт по ссылке и видит неправильную информацию

**Как исправить:**
1. Найти правильное расположение extras-p2 в lumen-plan.md (например, line 231)
2. Обновить STATUS-P1.md:42 → `lumen-plan.md:231`
3. Добавить в commit-message: "docs: update STATUS-P1.md line references after lumen-plan.md restructure"

**Инструмент:** Grep по строкам быстрее, чем поиск по тексту
```bash
# Перед правкой: найти все ссылки на документ
grep "lumen-plan.md:" STATUS-P*.md

# После правки: проверить что ссылки ещё работают
sed -n '195p' lumen-plan.md  # убедиться что это нужное содержимое
```
