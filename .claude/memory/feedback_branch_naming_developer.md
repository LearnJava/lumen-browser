Ветки для параллельных сессий должны включать номер разработчика в названии: `p1-<task>`, `p2-<task>`, `p3-<task>`, `p4-<task>`.

**Why:** Если сессия упадёт, нужно легко определить, какой разработчик (P1–P4) работал над веткой, чтобы продолжить работу в правильном контексте.

**How to apply:** Всегда при создании ветки в сессии P1/P2/P3/P4 добавлять префикс: `git checkout -b p2-font-metrics` вместо `git checkout -b font-metrics`. Это дополняет (не заменяет) существующие соглашения об именовании веток.
