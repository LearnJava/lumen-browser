# Старт задачи — протокол координации Lumen

$ARGUMENTS — имя задачи (kebab-case, например `font-fallback`).
Если аргумент не передан — спроси у пользователя.

## Шаг 1 — Проверь, что уже занято

```bash
git branch
```

!`git branch`

Прочитай блок «🔄 В работе сейчас» в `lumen-plan.md`:

!`sed -n '/^## 🔄 В работе сейчас/,/^## /p' lumen-plan.md | head -20`

Если задача с таким именем уже есть в ветках или в блоке «В работе» —
сообщи пользователю и остановись. Не дублируй чужую работу.

## Шаг 2 — Определи программиста и доменную зону

Из CLAUDE.md (таблица «Распределение задач»):
- P1 — `lumen-html-parser`, `lumen-css-parser`, `lumen-layout`
- P2 — `lumen-font`, `lumen-paint`, `lumen-image`
- P3 — `lumen-network`, `lumen-storage`, `lumen-core`
- P4 — `lumen-shell`, JS, AI, UI-фичи

Если пользователь указал «ты программист N» — проверь, что задача помечена
`[PN]` в roadmap CLAUDE.md. Если нет — уточни.

## Шаг 3 — Создай worktree и ветку

Worktree **обязательно** внутри папки браузера (не снаружи):

```bash
git worktree add .claude/worktrees/$ARGUMENTS -b $ARGUMENTS
```

Если имя ветки занято — добавь суффикс `-v2` или уточни имя у пользователя.

После создания worktree — работа ведётся **только** в нём:
```
.claude/worktrees/$ARGUMENTS/
```

## Шаг 4 — Зарезерви задачу (первый коммит)

Перейди в новый worktree и добавь строку резервации в `lumen-plan.md`:

Найди блок `## 🔄 В работе сейчас` и замени строку `_(никто ничего не зарезервировал)_`
(или добавь перед закрывающим `##`) на:

```
- 🔄 <описание задачи> [PN] — <имя-ветки> — <YYYY-MM-DD>
```

Дата — сегодня. `[PN]` — номер программиста (P1..P4).

Затем сделай первый коммит:
```bash
git -C .claude/worktrees/$ARGUMENTS add lumen-plan.md
git -C .claude/worktrees/$ARGUMENTS commit -m "Зарезервировать задачу $ARGUMENTS

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>"
```

## Шаг 5 — Сообщи пользователю

Выведи итог:
- Имя ветки и путь worktree
- Строка резервации, добавленная в план
- Следующий шаг: какой файл открывать первым для реализации

Теперь можно начинать работу. Все дальнейшие коммиты — в этом worktree.
