# lumen-network ✅ (HTTP/1.1 + HTTPS)

- **Done:** `HttpClient` implements `NetworkTransport` from `lumen-core::ext`. HTTP and HTTPS support (rustls + webpki-roots, exception #3). Redirect following up to 5 hops (absolute + relative `Location`). `chunked` Transfer-Encoding decoder. URL parsing (scheme/host/port/path), case-insensitive headers. Box wrapper around TLS stream (clippy large-enum-variant). **IDN domains** converted to Punycode at `parse_url` via `lumen_core::idn::domain_to_ascii` — DNS lookup, TLS SNI (`ServerName::try_from`) and `Host:` header (RFC 7230 §5.4) always receive ASCII form.
- **Done (EventSink — principle #4):** `HttpClient::with_sink(Arc<dyn EventSink>)` + `with_tab(TabId)` — fluent builder. `fetch_with_redirect` emits `Event::RequestStarted { tab_id, url }` **after** successful `parse_url` and before TCP socket; `Event::RequestCompleted { tab_id, url, status }` — after reading status line, **before** response code analysis (2xx / 3xx / 4xx — all emitted as completed). Each redirect hop generates its own pair (Started/Completed). Invariant: "Started without Completed = network failure". `sink: Option<Arc<dyn EventSink>>` — defaults to None.
- **Done (RequestFilter):** `HttpClient::with_filter(Arc<dyn RequestFilter>)` — fluent builder, defaults to `Option::None`. When filter is attached, `fetch_with_redirect` calls `f.should_block(&url)` **after** `parse_url` and **before** `RequestStarted`/TCP: blocked request does NOT emit Started/Completed, emits `Event::RequestBlocked { tab_id, url, reason }` (if sink attached) and returns `Error::Network("blocked: {reason}")`. Each redirect hop checked independently. Bad scheme — filter not called.
- **Done (HTTP/1.1 keep-alive + ConnectionPool):** Persistent `Connection` stores `BufReader<RawStream>` (not recreated between requests — otherwise input buffer with remainder from previous response would be dropped) plus `closed` flag; `read_response(&mut Connection) -> Response` does not consume the connection. Request sends `Connection: keep-alive` explicitly. `read_chunked` **reads trailer section** to final `CRLF` — without this second request on pooled socket would read previous response tail as new status line. `ConnectionPool` — `Mutex<HashMap<(host, port, is_tls), Vec<Entry { conn, returned_at }>>>`, LIFO, **`IDLE_TIMEOUT = 30s`**, **`MAX_IDLE_PER_HOST = 6`**; stale on acquire — drop without use. `Connection: close` from server / EOF / read error — `closed = true`, not returned to pool. `HttpClient::with_pool(Arc<ConnectionPool>)` for shared pool. **Retry-on-stale:** `is_stale_error` recognizes `BrokenPipe` / `ConnectionReset` / `ConnectionAborted` / `UnexpectedEof` in `Debug` format of `io::Error` plus our "EOF before status line" / "EOF in headers".
- **Done (DnsResolver hook):** `HttpClient::with_dns_resolver(Arc<dyn DnsResolver>)` replaces default `SystemDnsResolver` (over `(host, port).to_socket_addrs()` from std). `connect(host, port, is_tls, resolver)` is two-stage: (1) `resolver.resolve` gives `Vec<SocketAddr>`; (2) try `TcpStream::connect(&addr)` for each until first success — correct for DNS round-robin and IPv4+IPv6 dual-stack. Resolver called per redirect hop, symmetric with sink/filter.
- **Done (DohResolver — RFC 8484):** `lumen_network::DohResolver` implements `DnsResolver` over arbitrary `NetworkTransport` (typically own `HttpClient`). For each `resolve(host, port)` sends two GET requests to DoH endpoint — AAAA (QTYPE=28) and A (QTYPE=1) — and merges results (IPv6 first, then IPv4 — RFC 6724 §6 dual-stack preference). Wire format — own encoder/decoder for DNS message (RFC 1035 §4). CNAME chains skipped; final A/AAAA RDATA converted to `Ipv4Addr` / `Ipv6Addr`. RCODE≠0 and TC=1 → Err. **DoH wrapper per RFC 8484 §4.1:** GET with `?dns=<base64url(query)>` — own base64url implementation without padding (RFC 4648 §5, alphabet `+→-`, `/→_`). **Bootstrap chain** documented in doc-comment: inner HttpClient in DohResolver must resolve endpoint host via non-DohResolver, otherwise infinite recursion. IPv4 literals (`8.8.8.8`), IPv6 (`::1`), and bracketed IPv6 (`[::1]`) handled locally without DoH lookup. TTL from DNS response ignored — cache above (`CachedDnsResolver`) sets its own TTL.
- **Done (DotResolver — RFC 7858):** `lumen_network::DotResolver` implements `DnsResolver` over own TCP+TLS socket (rustls, exception #3), without HTTP wrapper. Constructed as `DotResolver::new(server_name, server_addr)` — `server_name` for TLS SNI and cert validation, `server_addr` — pre-resolved IP+port of DoT server; factories `cloudflare()` / `google()` / `quad9()` hardcode IP literals `1.1.1.1:853` / `8.8.8.8:853` / `9.9.9.9:853` for bootstrap without system DNS. Default port 853 (RFC 7858 §3.1) exported as `DOT_DEFAULT_PORT`. **Bootstrap separation** server_name/server_addr solves circular problem. Wire format reuses `doh::encode_query` / `doh::decode_answer_ips`; TCP framing per RFC 1035 §4.2.2: `[u16 BE length][message]`. `query_over_stream<S: Read+Write>` — generic over stream for testing via `Cursor<Vec<u8>>` mock without TLS. `rustls::ClientConfig` with webpki-roots cached in `OnceLock`. IPv4 / IPv6 / bracketed `[::1]` literals handled locally. RCODE≠0 / TC=1 → Err. TTL ignored (cache above).
- **Done (HSTS enforcement — RFC 6797 end-to-end):** `HttpClient::with_hsts(Arc<dyn HstsEnforcement>)` attaches HSTS store. Trait bound `lumen-core::ext::HstsEnforcement` keeps lumen-network independent from lumen-storage. Module `lumen-network::hsts` contains pure functions `maybe_upgrade_url_to_https` and `process_sts_response`. **Pre-request:** if `url.scheme == "http"` and `store.is_https_only(host_ascii, now)` — rewrite to `https://`. RFC 6797 §8.3: explicit `:80` removed, custom port (`:8080`) preserved. **Post-response:** parse `Strict-Transport-Security` from response headers; RFC 6797 §8.1 — STS on HTTP response ignored. Done on EACH hop.
- **Done (HTTP Range requests — RFC 7233 single + multi + suffix + If-Range):** `HttpClient::fetch_range(url, RangeSpec, Option<RangeValidator>) -> Result<RangeResponse>` for single range + `HttpClient::fetch_multi_range(url, &[RangeSpec], Option<RangeValidator>) -> Result<MultiRangeResponse>` for multiple. `RangeSpec` enum with variants `Closed { start, end }` / `OpenEnded { start }` / `Suffix { length }` (RFC §2.1); constructors `closed(s, e)`, `from(s)`, `suffix(n)`. `RangeValidator` enum `ETag(String) | LastModified(String)` (RFC §3.2). `ContentRange { start, end, total: Option<u64> }` (inclusive end). `parse_boundary_from_content_type` and `parse_multipart_byteranges` (RFC 7233 §A). 206 → content_range parsed; 200 OK → full body; 416 / 4xx/5xx → Err. Range + If-Range forwarded through redirects. +26 unit tests + +7 e2e tests via mock-TcpListener.
- **Done (HTTP auth — RFC 7235 + 7617 Basic + 7616 Digest):** `HttpClient::with_credentials(Arc<dyn HttpCredentialProvider>)`. Trait point `lumen-core::ext::HttpCredentialProvider`. `StaticCredentialProvider` (in-memory table with fallback chain) for tests. Module `crates/network/src/auth.rs` contains `WWW-Authenticate` parser, `select_best_challenge` (Digest+SHA-256 > Digest+MD5 > Basic), builders `build_basic_authorization` and `build_digest_authorization` (RFC 7616 §3.4: HA1/HA2/response by MD5/MD5-sess/SHA-256/SHA-256-sess; per-request `nc` via `AtomicU32` + pseudo-random `cnonce`). Own **MD5** (RFC 1321) locally + **SHA-256** via `lumen_core::hash::sha256_hex`. **401 retry loop in `fetch_with_redirect`**: first request without `Authorization`; on 401 + provider + `WWW-Authenticate` → parse, select strongest, ask provider, build header → `continue`. One retry per hop: repeated 401 = `Err`. Authorization intentionally NOT forwarded on 3xx redirect — new request starts with `None` and asks provider for new origin (RFC 7235 §3.1).
- **Done (Content-Encoding pipeline + Brotli):** `HttpClient::with_content_decoder(Arc<dyn ContentDecoder>)` registers decoder from `lumen-core::ext::ContentDecoder` for one encoding name. `accept_encoding_header()` builds `Accept-Encoding` from registered decoders. `apply_content_encoding(body, headers, decoders)` parses `Content-Encoding` header (case-insensitive, comma-separated), applies decoders in **reverse** order (RFC 7231 §3.1.2.2). Unknown encoding without registered decoder → `Err`. **`BrotliContentDecoder`** — implementation of `ContentDecoder` (`encoding = "br"`) over provisional `brotli-decompressor = "5"` (RFC 7932). Without brotli, 60–70% of the real web sends `Content-Encoding: br`. Phase 0 limitation: full-body decode (no streaming).
- **Done (Security base — Origin / Mixed-Content / Sandbox classifiers, [P3] 2A partial):** Three pure classifiers. `Origin` (module `origin.rs`) — HTML LS §7.5 tuple origin `(scheme, host, port)` + `Origin::from_url`; `is_potentially_trustworthy` (W3C Secure Contexts §3.1: https/wss + localhost/`.localhost`/`127.0.0.0/8`/`::1`). **`MixedContent`** (module `mixed_content.rs`) — W3C Mixed Content `classify_subresource_request(top_origin, sub_url, RequestDestination) -> MixedContentLevel { NotMixed | OptionallyBlockable | Blockable }`. `RequestDestination` — 10 values per Fetch §3.2.7. **`SandboxFlags`** (module `sandbox.rs`) — HTML LS §7.6.5 "sandboxing flag set" as `u32` bitset on 18 bits. Inverse semantics: `empty()` = "no restrictions". `parse_sandbox_value(Option<&str>)` implements §"parse a sandboxing directive". 22 sandbox + 18 mixed-content + 29 origin = 69 classifier tests.
- **Done (Mixed-content enforcement in HttpClient, [P3] 2A step b):** `MixedContentPolicy { top_level: Origin, mode: MixedContentMode }` + builder `HttpClient::with_mixed_content_policy(top_level, mode)` + public method `fetch_subresource(url, destination) -> Result<Vec<u8>>`. `MixedContentMode` — `Disabled` (default), `SpecDefault`, `Strict`. Enforce block in `fetch_with_redirect` AFTER HSTS upgrade and BEFORE RequestFilter/RequestStarted. 16 new tests.
- **Done (CORS preflight classifier + cache, [P3] 2A continuation):** Pure CORS preflight logic in module `crates/network/src/cors.rs`. Predicates: `is_cors_safelisted_method`, `is_forbidden_request_header`, `is_cors_safelisted_request_header` (RFC Fetch §4.4). `CorsRequest { origin, target, method, headers, credentials_mode }`. `needs_preflight(req)` true if method not safelisted OR any header not safelisted. `build_preflight_headers` (Fetch §4.8 step 2-7). `PreflightResult { allowed_methods, allowed_headers, allow_credentials, max_age_seconds }`. `evaluate_preflight_response` (Fetch §4.8 steps 9-15 + §4.10). `check_cors_response_headers` for actual response. `PreflightCache` — thread-safe (`Mutex<HashMap<CacheKey, CacheEntry>>`), key `(requestor_origin, target_origin, credentials_mode)`. 56 unit tests.
- **Done (CORS preflight enforcement in HttpClient, [P3] 2A continuation finish):** `HttpClient::with_cors_cache(Arc<PreflightCache>)` + `HttpClient::fetch_cors(CorsRequest, Option<RequestDestination>) -> Result<Vec<u8>>`. Per-hop flow: classify cross-origin; cache shortcut; `needs_preflight` true → send OPTIONS preflight via same `fetch_single`; `evaluate_preflight_response` → success → `cache.insert`; fail → `Event::RequestBlocked`. Actual-response validation: cross-origin → `check_cors_response_headers` before status-branching. Each 3xx-hop reclassified under `Origin::from_url(next)`. +8 integration tests via mock-TcpListener.
- **Done (ALPN h2 negotiation, [P3] 5A.1):** `default_tls_config()` — global `OnceLock<Arc<ClientConfig>>`. Раньше config + webpki-roots пересоздавались на каждый connect; теперь parse — один раз. `alpn_protocols = [b"h2", b"http/1.1"]` (h2 предпочтительнее). В `connect()` после `ClientConnection::new` — `conn.complete_io(&mut tcp)`, чтобы handshake завершился до выбора версии запроса и можно было прочитать `conn.alpn_protocol()`. `check_negotiated_alpn`: `None | Some(b"http/1.1")` → Ok; `Some(b"h2")` → Err placeholder "HTTP/2 negotiated via ALPN, but client not implemented yet" (клиентский H2-стек — 5A.2–5A.6); любой другой ALPN → Err "unexpected ALPN" (rustls должен такое сам отвергать, defensive). +5 unit-тестов.
