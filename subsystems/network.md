# lumen-network ‚úÖ (HTTP/1.1 + HTTPS)

- **Done:** `HttpClient` implements `NetworkTransport` from `lumen-core::ext`. HTTP and HTTPS support (rustls + webpki-roots, exception #3). Redirect following up to 5 hops (absolute + relative `Location`). `chunked` Transfer-Encoding decoder. URL parsing (scheme/host/port/path), case-insensitive headers. Box wrapper around TLS stream (clippy large-enum-variant). **IDN domains** converted to Punycode at `parse_url` via `lumen_core::idn::domain_to_ascii` ‚Äî DNS lookup, TLS SNI (`ServerName::try_from`) and `Host:` header (RFC 7230 ¬ß5.4) always receive ASCII form.
- **Done (EventSink ‚Äî principle #4):** `HttpClient::with_sink(Arc<dyn EventSink>)` + `with_tab(TabId)` ‚Äî fluent builder. `fetch_with_redirect` emits `Event::RequestStarted { tab_id, url }` **after** successful `parse_url` and before TCP socket; `Event::RequestCompleted { tab_id, url, status }` ‚Äî after reading status line, **before** response code analysis (2xx / 3xx / 4xx ‚Äî all emitted as completed). Each redirect hop generates its own pair (Started/Completed). Invariant: "Started without Completed = network failure". `sink: Option<Arc<dyn EventSink>>` ‚Äî defaults to None.
- **Done (RequestFilter):** `HttpClient::with_filter(Arc<dyn RequestFilter>)` ‚Äî fluent builder, defaults to `Option::None`. When filter is attached, `fetch_with_redirect` calls `f.should_block(&url)` **after** `parse_url` and **before** `RequestStarted`/TCP: blocked request does NOT emit Started/Completed, emits `Event::RequestBlocked { tab_id, url, reason }` (if sink attached) and returns `Error::Network("blocked: {reason}")`. Each redirect hop checked independently. Bad scheme ‚Äî filter not called.
- **Done (HTTP/1.1 keep-alive + ConnectionPool):** Persistent `Connection` stores `BufReader<RawStream>` (not recreated between requests ‚Äî otherwise input buffer with remainder from previous response would be dropped) plus `closed` flag; `read_response(&mut Connection) -> Response` does not consume the connection. Request sends `Connection: keep-alive` explicitly. `read_chunked` **reads trailer section** to final `CRLF` ‚Äî without this second request on pooled socket would read previous response tail as new status line. `ConnectionPool` ‚Äî `Mutex<HashMap<(host, port, is_tls), Vec<Entry { conn, returned_at }>>>`, LIFO, **`IDLE_TIMEOUT = 30s`**, **`MAX_IDLE_PER_HOST = 6`**; stale on acquire ‚Äî drop without use. `Connection: close` from server / EOF / read error ‚Äî `closed = true`, not returned to pool. `HttpClient::with_pool(Arc<ConnectionPool>)` for shared pool. **Retry-on-stale:** `is_stale_error` recognizes `BrokenPipe` / `ConnectionReset` / `ConnectionAborted` / `UnexpectedEof` in `Debug` format of `io::Error` plus our "EOF before status line" / "EOF in headers".
- **Done (DnsResolver hook):** `HttpClient::with_dns_resolver(Arc<dyn DnsResolver>)` replaces default `SystemDnsResolver` (over `(host, port).to_socket_addrs()` from std). `connect(host, port, is_tls, resolver)` is two-stage: (1) `resolver.resolve` gives `Vec<SocketAddr>`; (2) try `TcpStream::connect(&addr)` for each until first success ‚Äî correct for DNS round-robin and IPv4+IPv6 dual-stack. Resolver called per redirect hop, symmetric with sink/filter.
- **Done (DohResolver ‚Äî RFC 8484):** `lumen_network::DohResolver` implements `DnsResolver` over arbitrary `NetworkTransport` (typically own `HttpClient`). For each `resolve(host, port)` sends two GET requests to DoH endpoint ‚Äî AAAA (QTYPE=28) and A (QTYPE=1) ‚Äî and merges results (IPv6 first, then IPv4 ‚Äî RFC 6724 ¬ß6 dual-stack preference). Wire format ‚Äî own encoder/decoder for DNS message (RFC 1035 ¬ß4). CNAME chains skipped; final A/AAAA RDATA converted to `Ipv4Addr` / `Ipv6Addr`. RCODE‚âÝ0 and TC=1 ‚Üí Err. **DoH wrapper per RFC 8484 ¬ß4.1:** GET with `?dns=<base64url(query)>` ‚Äî own base64url implementation without padding (RFC 4648 ¬ß5, alphabet `+‚Üí-`, `/‚Üí_`). **Bootstrap chain** documented in doc-comment: inner HttpClient in DohResolver must resolve endpoint host via non-DohResolver, otherwise infinite recursion. IPv4 literals (`8.8.8.8`), IPv6 (`::1`), and bracketed IPv6 (`[::1]`) handled locally without DoH lookup. TTL from DNS response ignored ‚Äî cache above (`CachedDnsResolver`) sets its own TTL.
- **Done (DotResolver ‚Äî RFC 7858):** `lumen_network::DotResolver` implements `DnsResolver` over own TCP+TLS socket (rustls, exception #3), without HTTP wrapper. Constructed as `DotResolver::new(server_name, server_addr)` ‚Äî `server_name` for TLS SNI and cert validation, `server_addr` ‚Äî pre-resolved IP+port of DoT server; factories `cloudflare()` / `google()` / `quad9()` hardcode IP literals `1.1.1.1:853` / `8.8.8.8:853` / `9.9.9.9:853` for bootstrap without system DNS. Default port 853 (RFC 7858 ¬ß3.1) exported as `DOT_DEFAULT_PORT`. **Bootstrap separation** server_name/server_addr solves circular problem. Wire format reuses `doh::encode_query` / `doh::decode_answer_ips`; TCP framing per RFC 1035 ¬ß4.2.2: `[u16 BE length][message]`. `query_over_stream<S: Read+Write>` ‚Äî generic over stream for testing via `Cursor<Vec<u8>>` mock without TLS. `rustls::ClientConfig` with webpki-roots cached in `OnceLock`. IPv4 / IPv6 / bracketed `[::1]` literals handled locally. RCODE‚âÝ0 / TC=1 ‚Üí Err. TTL ignored (cache above).
- **Done (HSTS enforcement ‚Äî RFC 6797 end-to-end):** `HttpClient::with_hsts(Arc<dyn HstsEnforcement>)` attaches HSTS store. Trait bound `lumen-core::ext::HstsEnforcement` keeps lumen-network independent from lumen-storage. Module `lumen-network::hsts` contains pure functions `maybe_upgrade_url_to_https` and `process_sts_response`. **Pre-request:** if `url.scheme == "http"` and `store.is_https_only(host_ascii, now)` ‚Äî rewrite to `https://`. RFC 6797 ¬ß8.3: explicit `:80` removed, custom port (`:8080`) preserved. **Post-response:** parse `Strict-Transport-Security` from response headers; RFC 6797 ¬ß8.1 ‚Äî STS on HTTP response ignored. Done on EACH hop.
- **Done (HTTP Range requests ‚Äî RFC 7233 single + multi + suffix + If-Range):** `HttpClient::fetch_range(url, RangeSpec, Option<RangeValidator>) -> Result<RangeResponse>` for single range + `HttpClient::fetch_multi_range(url, &[RangeSpec], Option<RangeValidator>) -> Result<MultiRangeResponse>` for multiple. `RangeSpec` enum with variants `Closed { start, end }` / `OpenEnded { start }` / `Suffix { length }` (RFC ¬ß2.1); constructors `closed(s, e)`, `from(s)`, `suffix(n)`. `RangeValidator` enum `ETag(String) | LastModified(String)` (RFC ¬ß3.2). `ContentRange { start, end, total: Option<u64> }` (inclusive end). `parse_boundary_from_content_type` and `parse_multipart_byteranges` (RFC 7233 ¬ßA). 206 ‚Üí content_range parsed; 200 OK ‚Üí full body; 416 / 4xx/5xx ‚Üí Err. Range + If-Range forwarded through redirects. +26 unit tests + +7 e2e tests via mock-TcpListener.
- **Done (HTTP auth ‚Äî RFC 7235 + 7617 Basic + 7616 Digest):** `HttpClient::with_credentials(Arc<dyn HttpCredentialProvider>)`. Trait point `lumen-core::ext::HttpCredentialProvider`. `StaticCredentialProvider` (in-memory table with fallback chain) for tests. Module `crates/network/src/auth.rs` contains `WWW-Authenticate` parser, `select_best_challenge` (Digest+SHA-256 > Digest+MD5 > Basic), builders `build_basic_authorization` and `build_digest_authorization` (RFC 7616 ¬ß3.4: HA1/HA2/response by MD5/MD5-sess/SHA-256/SHA-256-sess; per-request `nc` via `AtomicU32` + pseudo-random `cnonce`). Own **MD5** (RFC 1321) locally + **SHA-256** via `lumen_core::hash::sha256_hex`. **401 retry loop in `fetch_with_redirect`**: first request without `Authorization`; on 401 + provider + `WWW-Authenticate` ‚Üí parse, select strongest, ask provider, build header ‚Üí `continue`. One retry per hop: repeated 401 = `Err`. Authorization intentionally NOT forwarded on 3xx redirect ‚Äî new request starts with `None` and asks provider for new origin (RFC 7235 ¬ß3.1).
- **Done (Content-Encoding pipeline + Brotli):** `HttpClient::with_content_decoder(Arc<dyn ContentDecoder>)` registers decoder from `lumen-core::ext::ContentDecoder` for one encoding name. `accept_encoding_header()` builds `Accept-Encoding` from registered decoders. `apply_content_encoding(body, headers, decoders)` parses `Content-Encoding` header (case-insensitive, comma-separated), applies decoders in **reverse** order (RFC 7231 ¬ß3.1.2.2). Unknown encoding without registered decoder ‚Üí `Err`. **`BrotliContentDecoder`** ‚Äî implementation of `ContentDecoder` (`encoding = "br"`) over provisional `brotli-decompressor = "5"` (RFC 7932). Without brotli, 60‚Äì70% of the real web sends `Content-Encoding: br`. Phase 0 limitation: full-body decode (no streaming).
