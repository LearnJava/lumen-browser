# lumen-layout 🟡 (block + inline-flow + replaced + word-wrap + cascade)

- **Done:** `LayoutBox` tree, **specificity-based style cascade with !important**: for each rule the maximum specificity among its complex selectors is taken, all matched declarations sorted by `(important, specificity, rule_order, decl_index)` and applied in ascending order. `important` is first in key — `true > false` places !important-declarations at end, and they win over normal even with lower specificity (CSS Cascade L4 §8.1). On tie — later declared wins. **Matching complex selector — right-to-left with back-tracking** (`matches_chain`): for descendant and later-sibling combinators ALL candidates (ancestors / earlier-siblings) are iterated and the suffix is recursively verified — this correctly solves pathological cases like `.x + a ~ span` with multiple `a`-siblings without class `.x`. For child / next-sibling — single candidate, no back-tracking. Combinators: descendant / child / next-sibling / later-sibling. **Pseudo-classes:** all CSS3 structural and functional. **CSS Selectors L4 `:placeholder-shown`** (§15.1) — matcher `matches_placeholder_shown(doc, node)`. **CSS Selectors L4 form-state pseudo (§14.2 / §15.4 / §15.5):** `:required` / `:optional`, `:read-only` / `:read-write`, `:disabled` / `:enabled` — all pure attribute-based. **CSS Selectors L4 UI-state pseudo (§10.1 / §10.2 / §10.4):** `:checked`, `:indeterminate`, `:default` — pure DOM-based. **`:lang(...)` matcher** — RFC 4647 §3.3.1 basic filtering. **`:link` / `:visited` / `:any-link` matchers** — pure DOM-based, `:visited` always `false` (privacy-safe). **`:target` matcher** (`matches_target`) — case-sensitive id-attribute match against `Document::target()`. **`:target-within` matcher** (`matches_target_within`) — separate matcher via `any_descendant`. **`:fullscreen` / `:modal` / `:popover-open` matchers** — always `false` in Phase 0. **`:current` / `:past` / `:future` matchers** — always `false` in Phase 0. **`:dir(...)` matcher** (`matches_dir`) — walking up parents via `element_directionality`. **CSS4 `:has(rs-list)`** via `matches_relative`. Attribute selectors: all operators, with ASCII case-insensitive flag support. Inherited properties: `color`, `font-size`, `line-height`, `font-style`, `font-weight`, `font-family`, `text-transform`, `text-align`, `text-decoration-line`. **`FontStyle`** via `font-style` property + UA stylesheet for semantic tags. **`FontWeight`** with keyword and numeric scale support. **`FontStretch`** (CSS Fonts L4 §2.5) — stored in tenths of percent, 9 keywords, inherited. `text_rendering_eq` compares font_style, font_weight, font_variant and font_stretch. **`font-family`** — `Vec<String>` prioritized list. **`TextTransform`** — applied to `InlineSegment.text` at assembly. **`text-indent`** — first-line indent, inherited. **`white-space`** (Normal / Nowrap). **`opacity`** — CSS Color L3 §3.2, not inherited. **`outline`** — CSS Basic UI L4 §5. **`visibility`** — CSS Display L3 §4, **inherited**. **`overflow` / `overflow-x` / `overflow-y`** — CSS Overflow L3. **`text-overflow`** — CSS UI L4 §10.1. **`cursor`** — CSS UI L4 §8.1, inherited, full set of 36 standard keywords. **`box-shadow`** — `Vec<BoxShadow>`, parsing with balanced `()`. **`text-shadow`** — `Vec<TextShadow>`, **inherited**. **`border-radius`** — 4 fields, shorthand with 1-4 tokens. **`letter-spacing`** — added to word-width and word-boundary; `text_rendering_eq` accounts for letter_spacing. **`word-spacing`** — added only between words; word width unchanged. Properties with parsing: `display` (block/inline/none), `color` (full CSS3 named colors — 147 colors + `rebeccapurple` from CSS4 §6.1 + `transparent`; hex `#RGB`/`#RRGGBB`/`#RGBA`/`#RRGGBBAA` + `rgb()` / `rgba()` / `hsl()` / `hsla()` with commas or whitespace, slash-alpha modern syntax, percentages for channels, hue in `deg`/`turn`/`rad`/`grad` (CSS Color L4 §9)), `background-color`, `font-size`, `line-height`, `margin` (+ 4 sides), `padding` (+ 4 sides), `text-align` (left/center/right), **`text-decoration` / `text-decoration-line`**, **`width` / `height` (px/em/rem; `auto` = not set)**. Whitespace-only text nodes and comments are skipped. **Line wrapping:** `TextMeasurer` trait + `layout_measured()` split text by words onto lines with real font metrics. **Inline-flow:** `BoxKind::InlineRun { segments, lines }` — text nodes and inline elements grouped into one flow; each segment holds its own style; words with same rendering-style merged into one `InlineFrag` per line. `align_lines()` shifts `frag.x` after wrap for center/right alignment.

## Gotchas

- **`<picture>` / `<img srcset>` integrated in layout, but DPR=1.0 — Phase 0 stub.** `lumen-layout::box_tree::resolve_image_source` calls `pick_picture_source` / `pick_img_source` with `PictureParams { dpr: 1.0, supported_types: None }`. Density candidates (`srcset="… 2x"`) are **never selected** — picker always picks `1x`. Real DPR lives in `Renderer.scale_factor` and is not yet passed to layout. When relayout-on-resize lands, extend `resolve_image_source` with `dpr: f32` and pass it into `PictureParams` — picker API already supports it. Intrinsic dimensions from `<source>` are applied as presentational hints after CSS cascade (only into empty `style.width`/`style.height`). **`<source>` and `<track>` are `display: none`** in the UA stylesheet — previously they generated empty `BoxKind::Block` nodes.
