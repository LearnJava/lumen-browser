## Gotchas

- **`<picture>` / `<img srcset>` integrated in layout, but DPR=1.0 вЂ” Phase 0 stub.** `lumen-layout::box_tree::resolve_image_source` calls `pick_picture_source` / `pick_img_source` with `PictureParams { dpr: 1.0, supported_types: None }`. Density candidates (`srcset="вЂ¦ 2x"`) are **never selected** вЂ” picker always picks `1x`. Real DPR lives in `Renderer.scale_factor` and is not yet passed to layout. When relayout-on-resize lands, extend `resolve_image_source` with `dpr: f32` and pass it into `PictureParams` вЂ” picker API already supports it. Intrinsic dimensions from `<source>` are applied as presentational hints after CSS cascade (only into empty `style.width`/`style.height`). **`<source>` and `<track>` are `display: none`** in the UA stylesheet вЂ” previously they generated empty `BoxKind::Block` nodes.
