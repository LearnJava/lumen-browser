# lumen-layout 🟡 (block + inline-flow + replaced + word-wrap + cascade)

- **Done:** `LayoutBox` tree, **specificity-based style cascade with !important**: for each rule the maximum specificity among its complex selectors is taken, all matched declarations sorted by `(important, specificity, rule_order, decl_index)` and applied in ascending order. `important` is first in key — `true > false` places !important-declarations at end, and they win over normal even with lower specificity (CSS Cascade L4 §8.1). On tie — later declared wins. **Matching complex selector — right-to-left with back-tracking** (`matches_chain`): for descendant and later-sibling combinators ALL candidates (ancestors / earlier-siblings) are iterated and the suffix is recursively verified — this correctly solves pathological cases like `.x + a ~ span` with multiple `a`-siblings without class `.x`. For child / next-sibling — single candidate, no back-tracking. Combinators: descendant / child / next-sibling / later-sibling. **Pseudo-classes:** all CSS3 structural and functional. **CSS Selectors L4 `:placeholder-shown`** (§15.1) — matcher `matches_placeholder_shown(doc, node)`. **CSS Selectors L4 form-state pseudo (§14.2 / §15.4 / §15.5):** `:required` / `:optional`, `:read-only` / `:read-write`, `:disabled` / `:enabled` — all pure attribute-based. **CSS Selectors L4 UI-state pseudo (§10.1 / §10.2 / §10.4):** `:checked`, `:indeterminate`, `:default` — pure DOM-based. **`:lang(...)` matcher** — RFC 4647 §3.3.1 basic filtering. **`:link` / `:visited` / `:any-link` matchers** — pure DOM-based, `:visited` always `false` (privacy-safe). **`:target` matcher** (`matches_target`) — case-sensitive id-attribute match against `Document::target()`. **`:target-within` matcher** (`matches_target_within`) — separate matcher via `any_descendant`. **`:fullscreen` / `:modal` / `:popover-open` matchers** — always `false` in Phase 0. **`:current` / `:past` / `:future` matchers** — always `false` in Phase 0. **`:dir(...)` matcher** (`matches_dir`) — walking up parents via `element_directionality`. **CSS4 `:has(rs-list)`** via `matches_relative`. Attribute selectors: all operators, with ASCII case-insensitive flag support. Properties with parsing: `display` (block/inline/none), `color` (full CSS3 named colors — 147 colors + `rebeccapurple` from CSS4 §6.1 + `transparent`; hex `#RGB`/`#RRGGBB`/`#RGBA`/`#RRGGBBAA` + `rgb()` / `rgba()` / `hsl()` / `hsla()` with commas or whitespace, slash-alpha modern syntax, percentages for channels, hue in `deg`/`turn`/`rad`/`grad` (CSS Color L4 §9)), `background-color`, `font-size`, `line-height`, `margin` (+ 4 sides), `padding` (+ 4 sides), `text-align` (left/center/right), **`text-decoration` / `text-decoration-line`**, **`width` / `height` (px/em/rem; `auto` = not set)**.
