# lumen-storage ✅ (in-memory + SQLite + IndexedDB + Cookies + History + Bookmarks + HttpCache + Profiles + SearchProviders + Downloads + Permissions + Workspaces + TabSessions + Plugins + SearchHistory + SiteEngagement + DnsCache + ReferrerPolicies + Autofill + Notifications + SafeBrowsing + PslProvider)

- **Done (InMemoryStorage):** `HashMap<PartitionedKey, Vec<u8>>` with full origin-partitioning: each call accepts `origin: Option<&str>` and `top_level_site: Option<&str>`. `None` and `""` — one namespace (global profile). Implements `lumen_core::ext::StorageBackend` (get/put/delete/list_keys). Snapshot format `LUMEN_KV_V1` — text, hex-encoded composite key + hex-encoded value. `serialize()` / `deserialize()` for in-memory round-trip; `save(path)` / `load(path)` for disk.
- **Done (SqliteStorage):** persistent KV over **SQLite** (exception #5 in §5). Same `StorageBackend` signature, same origin-partitioning semantics. One table `kv (origin TEXT, top_level_site TEXT, key TEXT, value BLOB, PRIMARY KEY (origin, top_level_site, key)) WITHOUT ROWID`; `INSERT ... ON CONFLICT DO UPDATE` for put (upsert). WAL + `synchronous = NORMAL` by default. `Connection` wrapped in `Mutex`. `prepare_cached` for get/list_keys. Constructors: `open(path)`, `open_in_memory()`.
- **Done (IdbStore):** per-origin IndexedDB persistence over any `StorageBackend` (in-memory or SQLite). Implements `lumen_core::ext::IdbBackend` (`load`/`save`): the JS shim serializes the whole per-origin database set into one opaque JSON snapshot, stored under key `__indexeddb__` partitioned by origin. `new(Arc<Mutex<dyn StorageBackend>>, origin)`; several stores share one backend. `save` is best-effort (swallows errors so a failed write never aborts the JS transaction). Makes IndexedDB databases survive page reload. 7 unit tests (round-trip, overwrite, origin isolation, store-recreation across both in-memory and SQLite backends, Unicode).
- **Done (parse_set_cookie + parse_set_cookie_with_psl):** parser of HTTP `Set-Cookie` header into `Cookie` struct (RFC 6265 §5.2 + RFC 6265bis §5.5). PSL-check (RFC 6265bis §5.5 step 5): if Domain= matches known public suffix and is not equal to default_domain → cookie rejected. Conversion of RFC 1123 date to Unix timestamp via Howard Hinnant algorithm `civil_to_unix`. 19 unit tests (+8).
- **Done (CookieJar):** RFC 6265 + RFC 6265bis (SameSite) over SQLite. Structure `Cookie { domain, path, name, value, expires_at, secure, http_only, same_site: SameSite }`. **Total cookie protection** via `top_level_site` partitioning — third-party cookie for one source domain is invisible from different top-level context. Constructors `open(path)` / `open_in_memory()`. `Set-Cookie` HTTP parser and integration with `lumen-network` — separate tasks.
- **Done (Autofill):** saved form values per-(origin, field_name, value) with frequency-counter for autocomplete UI. `record(origin, field_name, value, now)` (empty value silently skipped); `suggestions(origin, field_name, limit)` ORDER BY frequency DESC + last_used DESC; `best_for(origin, field_name)` — top-1. 10 unit tests.
- **Done (Notifications):** Web Notifications API persistence. `Notification { id, origin, title, body, icon_url, tag, click_url, shown_at, dismissed_at, clicked_at }`. Tag-based replace for active notifications. 14 unit tests.
- **Done (ReferrerPolicies):** per-origin Referrer-Policy settings (W3C Referrer Policy spec). `ReferrerPolicy` enum — 8 values, `StrictOriginWhenCrossOrigin` default. `get_or_default(origin)` fallback to StrictOriginWhenCrossOrigin. 14 unit tests.
- **Done (SearchHistory):** recent omnibox queries for autocomplete. `record(query, now)` (upsert: trim, lowercase for dedup; on conflict — frequency++ and last_used MAX). `recent(limit)`, `popular(limit)`, `prefix_match(prefix, limit)`. 15 unit tests.
- **Done (SiteEngagement):** per-origin engagement metrics for omnibox ranking. `score(now, half_life_days) -> f64` — engagement score with exponential decay: `(visit_count + total_time_seconds/300) * 0.5^(age_days/half_life)`. `top_by_score(now, half_life, limit)`. 12 unit tests.
- **Done (DnsCache):** hostname → IP addresses with TTL. `DnsEntry { hostname, addresses: Vec<String>, cached_at, expires_at }`. `is_fresh(now)` — true if `now < expires_at`. Empty addresses (`&[]`) are stored — NXDOMAIN negative cache. Supports Punycode domains (`xn--…`). 11 unit tests.
- **Done (CachedDnsResolver):** implementation of `lumen_core::ext::DnsResolver` over `DnsCache`. Located in lumen-storage (not lumen-network) — because depends on SQLite cache; reverse path "lumen-network → lumen-storage" would pull SQLite into any binary with just networking. `Clock` trait (+ `SystemClock` default, `MockClock` in tests) substitutes time source for deterministic expiration testing. 11 unit tests.
- **Done (Plugins):** §11 "Modularity and plugins" — manifest registry of installed WASM plugins. `PluginManifest { id, name UNIQUE, version, source, capabilities_json, enabled, installed_at, last_used_at }`. Runtime capability check during plugin calls — task for `lumen-shell` + wasmtime sandbox (§11.2). 14 unit tests.
- **Done (Workspaces):** §8.3 "Workspaces — sets of tabs, switching with Ctrl+1..9. Each with its own cookie context (optional)". `Workspace { id, name, color, icon, cookie_partition, created_at, position }`. Cookie-isolation integration with CookieJar — separate task. 11 unit tests.
- **Done (TabSessions):** §12.7 "Tab session export / import" storage layer. Two tables: `session_snapshots` — session snapshots, `tab_sessions` — tabs in those snapshots. Parent-tab relationship via `parent_tab_id: Option<i64>` for tab tree (Ctrl+click children). Workspace assignment via `workspace_id: Option<i64>`. 12 unit tests.
- **Done (SessionExport):** §12.7 portable JSON session file format (2C). `SessionFile { version=1, name, created_at, tabs: Vec<ExportedTab> }`. `ExportedTab { url, title, scroll_x, scroll_y, is_active }`. `to_json()` / `from_json()` / `active_tab()`. Separate from TabSessions (SQLite) — portable, no DB dependency. Shell integration: `--import-session <file.lsession>` opens first active tab with saved scroll; auto-save `last_session.lsession` on window close. 12 unit tests.
- **Done (Downloads):** user download journal. `DownloadEntry { id, url, file_path, filename, mime_type, total_size, bytes_received, status: DownloadStatus, started_at, completed_at, error }`. `DownloadStatus { Pending, Done, Cancelled, Failed }`. `clear_completed()` removes done/cancelled/failed, leaves pending untouched. Storage-only — real queue, MIME-sniffing, resume — tasks for `lumen-network` + `lumen-shell`. 12 unit tests.
- **Done (Permissions):** per-origin permission grants for camera/microphone/geolocation/notifications/clipboard/midi/persistent-storage + `Other(String)` for forward-compat. `PermissionState { Prompt, Granted, Denied }`. `query(origin, kind, now_unix)` returns Prompt if `expires_at < now` — expired grants treated as missing. `clear_origin(origin)` for clear-site-data. 15 unit tests.
- **Done (SearchProviders):** registry of search engines for omnibox. Implements `lumen_core::ext::SearchProvider` trait — `query_url(query)` substitutes URL-encoded query in `{query}` placeholder in template. URL-encode per RFC 3986: leaves `unreserved = A-Z a-z 0-9 - _ . ~`, everything else — percent-encode (including UTF-8 multibyte sequences for Cyrillic). 14 unit tests.
- **Done (ProfileRegistry):** §9.3 profiles — metadata registry. `Profile { id, name, storage_path, created_at, settings_json, is_default }`. Singleton `active_profile` table for global "active profile". `delete(id)` (active automatically becomes None via FK SET NULL). Phase 0 stores settings as raw JSON-string.
- **Done (HttpCache):** persistent cache of HTTP responses (RFC 9111 basic part). `CacheControl::parse(header)` parses comma-separated directives. **Total cache protection** via `top_level_site` partitioning. Deferred: revalidation (If-None-Match / If-Modified-Since), Vary header, partial responses (Range), stale-while-revalidate, integration with HttpClient.
- **Done (Bookmarks):** bookmarks with folders and tags. Two tables: `bookmarks` + `bookmark_tags` with `ON DELETE CASCADE`. `add(url, title, folder, &[tags], note, created_at) -> id` — upsert (created_at preserved from first record; tags fully overwritten with deduplication). 12 unit tests.
- **Done (History):** visited pages history — foundation for §12.1 "Full-text search". `record_visit(url, title, visit_date)` — upsert: new row with visit_count=1 or update existing (visit_count++; visit_date := MAX(old, new); empty title doesn't overwrite old). `recent(limit)` — last N by visit_date DESC; `most_visited(limit)` — top-N by visit_count DESC for new-tab. FTS5 index over text — separate module (`lumen-knowledge`), here only basic schema and visit-tracking.
- **Done (SafeBrowsingList):** local analog of Google Safe Browsing v4 without cloud API — corresponds to principle #1 "privacy". `ThreatType` enum: Malware / SocialEngineering / UnwantedSoftware / PotentiallyHarmful / Other(String). **canonicalize_url** (simplified Safe Browsing v4 §4.4). **canonical_expression_variants / canonical_expression_variants_with_psl** generate up to 5 host-suffix × 4 path-trim = up to 20 variants per request. **SafeBrowsingFilter** — thin wrapper `Arc<SafeBrowsingList>` + optionally `Arc<dyn PublicSuffixList>`, implements `lumen_core::ext::RequestFilter`; fail-open on lookup errors. 34 tests.
- **Done (PslProvider):** implementation of `lumen_core::ext::PublicSuffixList` over provisional crate **`psl = "2"`** (compiled-in table, codegen from `public_suffix_list.dat` at build time — no runtime I/O). Methods: `public_suffix(domain)`, `registrable_domain(domain)`, `is_public_suffix(domain)`. API expects **ASCII domains** (Punycode `xn--…` for IDN). **Graduation criterion (§5):** realistically — never, PSL format is stable, updates come via crate version bump. 18 unit tests.
- **Done (SessionStore):** cross-restart "last session" store (§10I). Single table `session_tabs(ord, url, title, scroll_x, scroll_y, is_active, dom_blob)`; `save(&[PersistedTab])` overwrites the whole table in one transaction so a saved session is always internally consistent; `load()` returns tabs in saved left-to-right order. `dom_blob` is `Document::to_bytes()` so a background tab restores via `Document::from_bytes()` without a network round-trip. Distinct from `session_export` (portable SQLite-free JSON) and `tab_snapshot` (volatile in-session hibernation keyed by `tab_id`). Shell glue in `shell/src/session_persist.rs`. 8 unit tests.
- **Deferred:** namespace helpers (`profile::`), `clear_origin(origin)` for quick removal of all source data, encryption (`SQLCipher`-style — Phase 3 profiles), schema migrations (via `user_version` PRAGMA), FTS5 table over `history.title + extracted_text` for §12.1, CookieJar/History integration with shell and HttpClient, background Safe Browsing list updates (URL feed + dedup).
- 443 tests: 17 InMemoryStorage + 13 SqliteStorage + 20 CookieJar + 8 PSL-cookie + 16 History + 34 SafeBrowsing + 18 PSL + 8 SessionStore + remaining modules (Bookmarks, HttpCache, Profiles, SearchProviders, Downloads, Permissions, Workspaces, TabSessions, Plugins, SearchHistory, SiteEngagement, DnsCache, ReferrerPolicies, Autofill, Notifications, BroadcastChannels, CacheStorage, CSP, PermissionsPolicy, PushSubscriptions, ServiceWorkers, WebManifest).
