# lumen-encoding 🟡 (detector + decoders)

- **Done:** decoding tables for `Windows-1251`, `KOI8-R`, `CP866` (per WHATWG Encoding Standard) + **UTF-16 LE/BE decoder** with surrogate pair handling (U+10000+ supplementary plane), lone surrogate → U+FFFD, odd byte count → trailing U+FFFD. Decoder `decode(encoding, bytes) → String` for all six variants with lossy handling of illegal bytes and automatic stripping of UTF-8/UTF-16 BOM. Detector `detect(bytes, content_type_hint) → Encoding` with priorities: BOM (`EF BB BF` → UTF-8, `FF FE` → UTF-16 LE, `FE FF` → UTF-16 BE) → `<meta charset>`/`<meta http-equiv>` in first kilobyte → HTTP content-type hint → valid UTF-8 → frequency heuristics over Russian letters (weighted score from 32 frequencies). Implements `lumen_core::ext::EncodingDetector` via `HeuristicDetector`. `Encoding::from_label` parses WHATWG aliases (`cp1251`, `koi8r`, `ibm866`, `utf-16` → LE, `utf-16le`, `utf-16be`, `unicode` → LE, `ucs-2` → LE).
- **Deferred:** ISO-8859-5 and MacCyrillic (not encountered in practice), full HTML5 prescan algorithm §12.2.3.2 (our sniff is simpler than spec, but sufficient in practice), UTF-32 (vanishingly rare in the wild).
- 53 unit tests + 6 integration round-trip tests (including 18 UTF-16: ASCII/cyrillic LE and BE, BOM-stripping, supplementary 😀 via surrogate pair in both endians, lone high/low surrogate, odd byte count, empty input, labels).
