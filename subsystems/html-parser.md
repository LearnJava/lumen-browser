# lumen-html-parser ✅ (all 23 HTML5 insertion modes + Declarative Shadow DOM)

- **Done:** iterator-based FSM (Tokenizer); states Data, TagOpen, TagName, EndTag, BeforeAttributeName, AttributeName, AfterAttributeName, BeforeAttributeValue, AttributeValue (quoted/unquoted), SelfClosingStartTag, MarkupDeclarationOpen, Comment, CommentEnd, **RAWTEXT** (for `<script>` and `<style>`), **RCDATA** (for `<title>` and `<textarea>`), **DOCTYPE**. Character references: **extended set of ~250 named entities** via `entities` module (binary search over sorted table — Latin-1/diacritics, Greek alphabet, quotes and dashes `&mdash;`/`&ldquo;`/`&rdquo;`, symbols `&copy;`/`&reg;`/`&trade;`/`&euro;`, math `&sum;`/`&prod;`/`&ne;`/`&le;`/`&ge;`/`&asymp;`/`&infin;`, arrows `&larr;`/`&rarr;`/`&hArr;`, whitespace `&ensp;`/`&emsp;`/`&thinsp;`/`&zwnj;`/`&zwj;`/`&lrm;`/`&rlm;`); all names case-sensitive with mandatory trailing `;`; legacy without `;` (~100 references) not supported. Numeric `&#NNN;` / `&#xHHHH;` via `char::from_u32`. Lenient tree builder with void elements and self-closing.
- **RAWTEXT / RCDATA details:** combined field `text_only: Option<(String, bool)>` — `bool` = `decode_entities`. After `<script>`/`<style>` (non self-closing) sets `(name, false)` — RAWTEXT, character references are literal. After `<title>`/`<textarea>` — `(name, true)` — RCDATA, entities are decoded (needed for `<title>Foo &amp; Bar</title>` → `Foo & Bar`). In both modes `<` without `/tag` — text; termination only `</tag` + terminator (whitespace / `/` / `>` / EOF), case-insensitive. `</scripto>` does NOT match (`o` is not a terminator). Self-closing `<script/>` / `<title/>` do **not** activate the mode (symmetric with tree_builder which for self-closing does not push element to stack). `is_raw_text_element(name)` and `is_rcdata_element(name)` — two separate functions; extensible for future special content (`iframe`, `noembed`, `noframes`, `noscript`, `plaintext`).
- **DOCTYPE details:** `Token::Doctype { name, public_id: Option<String>, system_id: Option<String> }` (HTML5 §13.2.5.53–72). After `<!DOCTYPE` keyword (case-insensitive `doctype`/`DOCTYPE`) parses: name (lowercase, up to whitespace or `>`), optionally `PUBLIC "id" "id"` or `SYSTEM "id"` with support for single and double quotes. Lenient: `<!DOCTYPE>` without name gives empty name, incomplete DOCTYPEs don't crash. **`public_id`/`system_id` — `Option<String>`**: `None` if keyword is absent, `Some("")` if present with empty string. The distinction is important for quirks-detection (limited-quirks rules for HTML 4.01 Frameset/Transitional depend on the presence of system_id, not its content). Tree builder creates `NodeData::Doctype` node (with `unwrap_or_default()` for DOM — arena doesn't distinguish missing/empty). Other markup declarations like `<![CDATA[...]]>` or `<!ENTITY ...>` are still silently skipped up to `>`.
- **Quirks mode detection (module `quirks_mode`, HTML5 §13.2.5.1 "The initial insertion mode"):** function `detect_document_mode(name, public_id, system_id) -> DocumentMode`. Algorithm: (1) `name != "html"` → `Quirks`; (2) exact-match public_id in [`HTML`, `-/W3C/DTD HTML 4.0 Transitional/EN`, `-//W3O//DTD W3 HTML Strict 3.0//EN//`] → `Quirks`; (3) exact-match system_id `http://www.ibm.com/data/dtd/v11/ibmxhtml1-transitional.dtd` → `Quirks`; (4) public_id starts (case-insensitive) with one of ~55 prefixes of historical DTDs (IETF HTML 2.0/3.x, Netscape, Microsoft IE 2.0/3.0, Sun HotJava, HTML 4.0 non-4.01 Frameset/Transitional) → `Quirks`; (5) system_id absent AND public_id starts with `-//W3C//DTD HTML 4.01 Frameset//`/`Transitional//` → `Quirks`; (6) public_id starts with `-//W3C//DTD XHTML 1.0 Frameset//`/`Transitional//` → `LimitedQuirks`; (7) system_id present AND public_id with HTML 4.01 Frameset/Transitional → `LimitedQuirks`; otherwise → `NoQuirks`. All matches ASCII case-insensitive (spec). Tree builder calls detection at first DOCTYPE token (subsequent ones are ignored as syntax errors); when DOCTYPE is completely absent — sets `Quirks` (§13.2.6.4.1). **Detection-only** — layout/cascade do not yet read `Document.mode`; real CSS-behavior switching (table sizing, body-background propagation, font-size in `<table>`, etc.) — separate tasks, as needed.
- **srcset parser + picker (HTML5 §4.8.4.3.5 + §4.8.4.3.7):** module `srcset` exports `SrcsetCandidate { url, descriptor }`, `SrcsetDescriptor::{Density(f32), Width(u32)}` + `SrcsetDescriptor::DEFAULT` (= `Density(1.0)`), `parse_srcset(&str) -> Vec<SrcsetCandidate>` and `pick_best_for_density(&[..], dpr)`. Parser grammar — simplified lenient: candidate-list separated by commas (top-level), inside candidate — `url descriptor?`; descriptor — `Nx` (float density) or `Nw` (integer width), case-insensitive suffix. Trailing commas in URL (HTML5 §4.8.4.3.5 step "remove all trailing U+002C COMMA characters") treated as "descriptor absent". Invalid descriptor (not x/w, negative, zero) — candidate silently dropped. **Density-picker** implements §4.8.4.3.7 for density-only candidates: (1) smallest sufficient density ≥ dpr; (2) fallback to largest if all are smaller; (3) tie-break — first in source-order. Invalid dpr (≤0 / NaN) treated as 1.0.
- **sizes attribute + width-picker (HTML5 §4.8.4.4 + §4.8.4.3.7):** `SizeLength` (Px/Em/Rem/Vh/Vw/Vmin/Vmax/Percent) with `resolve(viewport)` to CSS pixels — `em` ≡ `rem` (preload scanner works before DOM is built, element-context font-size unavailable); `Percent` resolves from viewport width (typical browser practice). `SizesViewport { width_px, height_px, root_font_size_px, prefers_dark }` + `SizesViewport::DEFAULT` (1024×768, 16px, light). `Orientation::{Portrait, Landscape}`. `ColorScheme::{Light, Dark}`. `MediaClause` — single feature (min-width / max-width / min-height / max-height / orientation / **prefers-color-scheme**) **or `Nested(Box<MediaCondition>)`** for L4 forms `(not <cond>)` / `((<cond>))` inside clause brackets. `MediaCondition::{All { negated, clauses }, Unsupported}` — AND-list with optional leading `not`. `SourceSize { condition: Option<MediaCondition>, value: SizeLength }`. **`parse_sizes(input)`** splits by top-level commas, each element — either single length (default-fallback), or `<media-condition> <length>` (split by last whitespace). Invalid length / unknown feature — silent drop of element. **`evaluate_sizes(sizes, viewport)`** returns CSS pixels of first matching `SourceSize`; fallback to `viewport.width_px` (100vw) if none matched. **`pick_best_for_width(candidates, source_size_px, dpr)`** — effective density `Nw / source_size_px`, then same algorithm "smallest sufficient ≥ dpr → fallback to largest → tie-break first in list". Density candidates (`Nx`) in w-picker are ignored: mixing `Nw`+`Nx` violates spec, we're lenient. Invalid `source_size_px` (≤ 0 / NaN / ∞) → None. **`parse_media_condition`** exported publicly — picture-picker uses same AST for `<source media>` attribute; supports nested L4 forms `(not <media-in-parens>)` and `((<cond>))` via `parse_media_clause`. Top-level split by `" and "` is **paren-aware** (`split_top_level_and` counts paren balance), so `and` inside nested group doesn't split the clause list. `only` intentionally not supported in `<source media>` / `sizes` (L3 backcompat modifier).
- **`<picture>` / `<source>` selection (HTML5 §4.8.4.4):** module `picture`. `PickedSource { url: String, intrinsic_width: Option<u32>, intrinsic_height: Option<u32> }` — final URL of selected source plus author-declared intrinsic dimensions in CSS pixels (HTML5 §10 "Mapped attributes" — `<source width|height>` / `<img width|height>` integer attributes). Fields for CLS-protection (Cumulative Layout Shift) — layout/shell can reserve space for the box BEFORE the image loads. `parse_dim_attr` parses HTML5 "rules for parsing non-negative integers": trim, optional `+`, digit-prefix; negative/percentage/non-numeric → None. MIME type and real pixel dimensions (after decode) remain the image-pipeline's job. `PictureParams { viewport: SizesViewport, dpr: f32, supported_types: Option<&[&str]> }` — `supported_types=None` disables type-filter (preload scanner before codec-registry is ready; default is also None). **`pick_picture_source(doc, picture_node, &params)`** — walks child nodes in source-order. For each `<source>`: skip without srcset; type-filter (case-insensitive lookup in supported_types; empty `type=""` after trim treated as "absent" = match-everything); media-filter (via `parse_media_condition`); if both passed — pick via srcset+sizes (width-picker when any `Nw`-candidate present, otherwise density-picker), then `fill_intrinsic_dims` takes width/height from the same `<source>`. If no source gave URL — fallback to first encountered `<img>` child via `pick_img_source` (intrinsic — from `<img>`, not unselected `<source>`-s). Misuse (`pick_picture_source` on non-`<picture>`, `pick_img_source` on non-`<img>`) → None. Does not resolve relative URLs — that's shell work (`Url::resolve`).
- **Push-tokenizer + incremental tree builder (P1 item 3B):** `PushTokenizer::feed(&str)` / `feed_bytes(&[u8])` / `end() -> Vec<Token>` — wrapper over same pull-Tokenizer as iterator-API. Accumulates chunks in owned `String` buffer, on each `feed` finds "safe split point" (`find_safe_split`) and tokenizes up to it, leaving tail for next call. Safe-split accounts for: unclosed `<tag…`/`</tag…` (terminator `>`), unclosed `<!--…-->` (terminator `-->`, `>` inside doesn't count), unclosed `<!DOCTYPE…>`, incomplete `&entity;` in data state and in RCDATA (`<title>`/`<textarea>`); in RAWTEXT (`<script>`/`<style>`) entity not decoded and buffering not needed. `Tokenizer::with_state(input, text_only)` + `pos()` + `text_only_state()` — pair of public methods for resuming state between chunks. Pull-Tokenizer changed so that on EOF in text-only loop it restores `text_only` field (previously `.take()` lost state — push mode switched to data state mid-RAWTEXT). `IncrementalTreeBuilder { feed(&str), finish() -> Document }` — push variant of `parse()` function: holds `Document` + DOM stack + `seen_doctype` between `feed` calls, drives push tokens through common `apply_token` helper shared with `parse()`. **Invariant: with same input push and pull produce byte-for-byte equal `Document`** — ensured by **text-node coalescing**: if last child of parent is a text-node, new `Token::Text` is appended to it instead of creating second text-node (push may split continuous text into many tokens at chunk boundary; pull always delivers whole piece, for it coalescing is a no-op). **`feed_bytes(&[u8])` with partial UTF-8 buffering (WHATWG Encoding §4):** field `partial_utf8: Vec<u8>` (max 3 bytes) buffers an incomplete UTF-8 sequence across chunk boundaries; valid prefix is drained to `buf` each call; genuinely invalid bytes (0xFF etc.) are replaced with U+FFFD inline; incomplete sequence remaining at `end()` → U+FFFD. UTF-8: `feed(&str)` still accepts already-valid UTF-8 from caller; `feed_bytes` handles arbitrary raw bytes. Unblocks P3 item 4B (streaming pipeline shell-side).
- **Preload scanner (HTML LS §13.2.6.4.7 "Speculative HTML parsing"):** module `preload_scanner`. `scan_preload_hints(input) -> Vec<PreloadHint>` runs over regular Tokenizer without building DOM and emits spec-defined set of subresource-hints in source-order. `PreloadHint` enum: `Stylesheet { url }` for `<link rel=stylesheet>`, `Script { url }` for `<script src>`, `Image { url, srcset, sizes }` for `<img>`, `SourceSet { srcset, sizes }` for `<source srcset>` (inside picture/video/audio — `media` attribute NOT considered: preload is speculative, we fetch all candidate URLs pessimistically), `Preload { url, as_kind }` for `<link rel=preload>`, `Preconnect { url, dns_only }` for `<link rel=preconnect>`/`rel=dns-prefetch`. `rel` — multi-token (`rel="preload stylesheet"`), emit separate hint for each known token; unknown `rel`-keywords (icon / canonical / manifest) — silent skip. URL is trimmed (whitespace-only `href` → drop), not validated or resolved — that's caller's work (shell via `Url::resolve(base)`). Content of `<script>`/`<style>` (RAWTEXT) is not parsed as tags (`<img>` inside is literal — correct per spec). Scanner caller does not yet exist: shell-pipeline currently collects stylesheet-href via DOM-walk; integration into streaming pipeline — P4 task.
- **All 23 HTML5 insertion modes (§13.2.4.1) implemented:** Initial, BeforeHtml, BeforeHead, InHead, **InHeadNoscript**, AfterHead, InBody, Text, InTable, InTableText, InCaption, InColumnGroup, InTableBody, InRow, InCell, InSelect, **InSelectInTable** (full, was stub), InTemplate, **InFrameset**, AfterBody, **AfterFrameset**, AfterAfterBody, **AfterAfterFrameset**. `reset_insertion_mode` uses `InFrameset` (was `InBody`) for frameset context and `InSelectInTable` when `<select>` has a table ancestor. `mode_after_head` creates real `<frameset>` element and switches to `InFrameset` (was fallback to `<body>`). `scripting_enabled: bool` field (default `true`) controls `<noscript>` parsing — `true` → raw text mode (scripting on), `false` → InHeadNoscript markup mode.
- **Deferred:** CDATA, legacy entities without `;` (HTML4 compat, ~100 names), `<iframe>` / `<noembed>` / `<plaintext>` text modes, application of Quirks-mode switches in layout/cascade (currently detection only), `calc()`/`min()`/`max()` in length values of sizes, OR-conditions in media-condition (spec L4 §3.2 `<media-or>`), `loading="lazy"` via IntersectionObserver (preload + scheduling — shell + runtime task).
- **Declarative Shadow DOM (WHATWG HTML §14.5):** When `<template shadowrootmode="open|closed">` is parsed, instead of creating a `DocumentFragment`, a shadow root is attached to the parent (host) element with the specified mode, and template content is parsed directly into the shadow root. The `<template>` element itself is detached on `</template>` — it is transparent and invisible in the final DOM. Invalid `shadowrootmode` values fall back to regular template behaviour. `IncrementalTreeBuilder` tracks declarative templates via `declarative_shadow_templates: HashSet<NodeId>`. 7 unit tests.
