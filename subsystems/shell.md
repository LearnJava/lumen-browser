# lumen-shell 🟡 (window + render + network)

- **Done:** winit 0.30 with `ApplicationHandler` API. Three window modes: `lumen` (empty window 1024×720), `lumen <path.html>` (file → encoding → HTML → layout → paint), `lumen <http(s)://...>` (network via `HttpClient` → same stages). External CSS: `<link rel="stylesheet" href="...">` loaded from disk (relative to HTML file) or over network (relative to base URL). `ResourceBase` enum isolates relative URL resolution logic. Inter-Regular.ttf bundled via `include_bytes!`. Handlers for Resized + RedrawRequested.
- **Done (fetch+decode `<img src>` + srcset/picture):** between HTML parse and layout, shell resolves image URLs via `lumen_layout::collect_image_requests(doc, viewport)` — same picker (`resolve_image_source`) as `BoxKind::Image { src }`. Downloads bytes (file:// or http via `HttpClient`), decodes via `lumen_image::decode` (PNG/JPEG sniff-dispatch). `fetch_and_decode_images(doc, base, sink, viewport) -> Vec<(String, lumen_image::Image)>`. Key in result Vec is the picked URL — always matches `DisplayCommand::DrawImage.src` including `<picture>`/`srcset` candidates. **Intrinsic dimensions (HTML5 §10 "mapped attributes"):** if `<img>` has neither `width` nor `height` attribute, both are set from decoded image as presentational hints via `apply_intrinsic_size`; author CSS overrides. Errors silently skipped with `eprintln!` — broken image doesn't crash the whole page, paint draws grey placeholder. `LoadedPage`/`ParsedPage` now carry `images: Vec<(String, lumen_image::Image)>`.
- **Done (network log):** `StdoutEventSink` — simplest network event observer, prints to **stderr** (not stdout): `→ GET <url>`, `← <status> <url>`, `✗ <url> (<reason>)`. Attached to `HttpClient` in shell. Stderr chosen to not pollute dump-mode stdout.
- **Done (window title):** `extract_title(&Document)` finds first `<title>` in tree, joins text children, collapses whitespace via `split_whitespace().join(" ")`. `LoadedPage { display_list, title }`. `window_title(Option<&str>)` formats: with title — `"<title> — Lumen"`, without — fallback to `Lumen <version>`. `Lumen::reload` updates window title on reload.
- **Done (keybindings):** `PageSource` enum (`Empty` / `File(PathBuf)` / `Url(String)`) stored in `Lumen`. `KeyCommand` enum (`Reload` / `Exit`) and `keybinding_for(KeyCode, ModifiersState) -> Option<KeyCommand>` — isolated from winit mapper (F5 / Ctrl+R → Reload, Esc / Ctrl+W → Exit), unit-testable. `WindowEvent::ModifiersChanged` and `WindowEvent::KeyboardInput` handled. Force-reload (`Ctrl+Shift+R`) intentionally not bound — no cache to bypass yet; reserved for future.
- **Done (runtime — integration in winit loop):** `Lumen` stores `runtime: runtime::EventLoop` and `epoch: Instant` for DOMHighResTimeStamp. Three dispatch points: `about_to_wait` → `step()` in loop `while StepResult::Ran` until `Idle` (cap 256 tasks per tick — runaway protection), then `run_idle_callbacks(remaining_ms, now_ms)`; `WindowEvent::Resized` → `deliver_observer_records(ObserverKind::Resize)`; `WindowEvent::RedrawRequested` → `run_rendering_step(epoch.elapsed_ms())` before `Renderer::render()`. In Phase 0 nobody is registered (JS engine absent), but dispatch points are in place.
- **Done (runtime — HTML event loop framework):** Module `lumen-shell::runtime` (Phase 0 framework, without winit-loop integration). Implements HTML Living Standard §8.1.4 "Event loops" + §8.1.5.1 "Update the rendering" in minimal form. **`TaskQueue`** — **per-source** FIFO queues `Box<dyn FnOnce>` (array `[VecDeque<Task>; 7]`, index via `TaskSource::as_index`). `pop` iterates sources in `TaskSource::PRIORITY_ORDER` (`UserInteraction > DomManipulation > HistoryTraversal > Networking > Timer > Rendering > IdleTask`). **`MicrotaskQueue`** — drain-all semantics. **`EventLoop::step()`** — one task per step + microtask checkpoint after it. **rAF** per §8.1.5.1: snapshot via `mem::take` before iteration; cancelled handles checked before each invocation. **requestIdleCallback** W3C §3: `IdleDeadline { time_remaining(), did_timeout() }`; `timeout_ms` optional. **Observer registries** three types (`ObserverKind::Resize / Intersection / Mutation`). **Reentrancy:** state under `Rc<RefCell<…>>` + thin `EventLoopHandle::clone()`. **Threading:** single-threaded (`Rc`/`RefCell`), like HTML event loop itself. 19 unit tests.
- **Done (headless dump modes):** three CLI flags run pipeline without winit and wgpu, print phase result to stdout, diagnostics to stderr. `--dump-source <path-or-url>` → `lumen_encoding::decode` after `detect`. `--dump-layout <path-or-url>` → `lumen_layout::serialize_layout_tree`. `--dump-display-list <path-or-url>` → `lumen_paint::serialize_display_list`. `parse_cli(args) -> Result<CliMode, String>` — isolated CLI parser, unit-testable. `parse_and_layout(bytes, ct, base, sink) -> ParsedPage` extracted from `render_bytes` — shared entry point for both window and dump modes.
- **Done (find in page — Ctrl+F):** module `lumen-shell::find`. `FindState { open, query, active }` stored in `Lumen`. `find_matches(&DisplayList, &str, &dyn TextMeasurer) -> Vec<FindMatch>` walks `DisplayCommand::DrawText`, matches case-insensitive (ASCII via `to_ascii_lowercase`, Unicode via `char::to_lowercase().next()`), computes `prefix_w` / `match_w` via `TextMeasurer`, returns absolute `Rect` + `dl_index`. Non-overlapping matches (after match skip `i += query.len()`). `build_with_overlay(base, state, matches, BarOverlay { window_size })` builds final display list: highlights inserted before their `DrawText` (active match — orange `#ff9632`, inactive — yellow `#ffeb5a`), plus UI bar in top-right corner. Bindings (`handle_find_key`): Ctrl+F opens; while bar is open — Esc closes, Backspace erases, Enter/F3 go to next (Shift+Enter/Shift+F3 — previous), printable symbols from `KeyEvent.text` go to query. Phase 0 limitations: letter-spacing/word-spacing not accounted in match rect, find bar fixed 480×40 in top-right corner.
- **Done (scroll state):** `Lumen { scroll_y: f32, content_height: f32 }`. `Renderer::render(content, overlay, scroll_y)` — page portion of display list gets Y-translate `-scroll_y`, overlay portion rendered without offset (viewport-locked). `content_height_of(&dl)` — `max(rect.y + rect.height)` across all rect-bearing commands. **Input:** `WindowEvent::MouseWheel` via `MouseScrollDelta` — `LineDelta(_, lines)` → `lines × 40 CSS px`, `PixelDelta(p)` → `p.y / DPR`. Keyboard: `ArrowDown/Up` (40 px line-step), `PageDown/PageUp/Space/Shift+Space` (90% viewport), `Home/End` (to `0.0` / `f32::INFINITY` → clamp to max_scroll). `scroll_by(delta)` / `scroll_to(target)` via `clamp_scroll`. 9 unit tests.
- **Done (scroll-to-match for find):** pure function `find::scroll_to_match(match_rect: Rect, viewport_height: f32, current_scroll: f32) -> Option<f32>`. If match already fully visible — `None`. Otherwise `target = max(0, match.y − viewport_height × SCROLL_MARGIN_FRACTION)` where `SCROLL_MARGIN_FRACTION = 0.25`. Called from `handle_find_key` after Backspace, Enter/F3, text input. 8 unit tests.
- **Done (vertical scrollbar overlay):** module `lumen-shell::scrollbar`. `build_scrollbar_overlay(scroll_y, content_height, viewport_width, viewport_height) -> DisplayList` — pure function, returns 2 `FillRect` (track + thumb) or empty `Vec` if scrollbar not needed / args are NaN-Inf. `thumb_geometry(scroll_y, ch, vh) -> (top, height)`: `thumb_h = max(MIN_THUMB_HEIGHT=24, vh²/ch) clamp to vh`, `thumb_top = (vh − thumb_h) × ratio`. Scrollbar commands inserted into overlay buffer **before** find-bar commands (painter's order: scrollbar below, bar above). Phase 0 limitations: Y-axis only, no fade-in/out. 12 unit tests.
- **Done (scrollbar drag + track-click page-jump):** `scrollbar::classify_track_click(x, y, scroll_y, ch, vw, vh) -> TrackClick { None | Thumb | Above | Below }`. `Thumb` → start `ScrollDrag { start_scroll_y, start_mouse_y }`; `Above` → `scroll_by(-page_step(vh))`; `Below` → `+page_step(vh)`; `None` → no-op. `ScrollDrag::scroll_for(current_mouse_y, ch, vh) -> f32` returns unclamped `scroll_y`. `Lumen` extended with `cursor_position: Option<PhysicalPosition<f64>>` and `scroll_drag: Option<ScrollDrag>`. Single-click track = single page-jump. 17 unit tests. Phase 0 limitations: held-track-click continuous mode not implemented.
- **Done (cursor-icon feedback on hover scrollbar thumb):** `Lumen::update_cursor_icon` called from `WindowEvent::CursorMoved` and `MouseInput Released`. Pure function `cursor_icon_for_hover(scrollbar::TrackClick, drag_active: bool) -> winit::CursorIcon`: drag active → `Pointer` regardless of position; hover thumb → `Pointer`; hover track or None → `Default`. `Lumen.last_cursor_icon: Option<CursorIcon>` caches previous value — `set_cursor` called only on change. Reuses `scrollbar::classify_track_click` for hit-test. 5 unit tests.
- **Done (smooth scroll):** module `lumen-shell::scroll_anim`. `ScrollAnim { start_y, target_y, start_time_ms }` + `ease_out_cubic(t)` + `sample(now_ms) -> (scroll_y, done)`; `DURATION_MS = 200.0` ms. `Lumen.scroll_anim: Option<ScrollAnim>` started via `start_smooth_scroll(target)` and `scroll_by_smooth(delta)` (additive over active animation target — repeated wheel/key-repeat appends delta to destination). `RedrawRequested` ticks `advance_scroll_anim()` before building display list and requests another redraw while not done. Wheel, arrow-keys, PageUp/Down/Space, Home/End, track-click, find-scroll-to-match — all go through smooth variants; thumb drag remains instant. 12 unit tests.
- **Done (preload scan + dispatch + URL resolution + dedup + priority):** `lumen_html_parser::scan_preload_hints(&source)` called before `parse()` (4B.1). `dispatch_preload_hints(hints, base, sink)` (1) resolves each raw `href`/`src` via `ResourceBase::resolve_str`, (2) stable-sorts by `FetchPriority::for_kind` (High → Medium → Low), (3) deduplicates by resolved URL (first = highest priority wins), (4) emits `Event::SubresourceHintFound { url, kind: SubresourceKind, priority: FetchPriority }`. `FetchPriority` enum added to `lumen-core::event` (High=CSS/font/preconnect, Medium=script, Low=img). `StdoutEventSink` logs `⤷ preload <kind> [<priority>] <url>`. `srcset`-strings emitted as-is. 4 tests for resolve_str + 3 for dispatch (emits, dedup, priority-sort).
- **Done (streaming pipeline — task 4B):** `EventLoop::<LoadEvent>` replaces bare `EventLoop<()>`. `LoadEvent { HtmlChunk(String) | LoadDone(RawPage) | LoadError(String) }`. Window created immediately with empty display list; `resumed()` calls `start_streaming_load()` — background thread fetches all bytes, decodes UTF-8, splits into 8 KB chunks, sends via `EventLoopProxy::send_event`. Main thread `user_event` handler: each `HtmlChunk` → `IncrementalTreeBuilder::feed` + paint every ≥150 ms using UA-stylesheet-only layout (`Stylesheet::default()`); `LoadDone` → full `render_bytes` pipeline (CSS + images); `LoadError` → log. `IncrementalTreeBuilder::as_doc()` added to `lumen-html-parser` for partial-DOM snapshots. `apply_loaded_page` helper extracted from `reload` for reuse. `paint_partial_dom` clones partial Document, calls `layout_measured` + `build_display_list`, requests redraw.
- **Done (U-1 stage 1 — non-blocking navigation):** `reload()` no longer runs the whole fetch+parse+JS+layout pipeline synchronously on the UI thread. When a window/renderer exists (i.e. every navigation after the first frame — link click, address bar, back/forward, JS `location.href=`, palette reload), it resets the streaming state (`stream_builder`/`stream_sheet`/`stream_layout_seeded`/dedup sets) and delegates to `start_streaming_load()` — the same async path the initial load uses in `resumed()`. The HTML body streams off-thread, the window keeps painting progressive frames, and the heavy final pipeline (`render_bytes`) runs once on the UI thread at `LoadEvent::LoadDone` via `apply_loaded_page`. A synchronous fallback (the old GPU-session / `source.load` body) is kept for the window-less path (headless/tests). **Load-generation guard:** `Lumen.load_generation` is bumped per navigation and stamped onto every streaming `LoadEvent` (`EarlyPreloadHints`/`HtmlChunk`/`CssLoaded`/`LoadDone`/`LoadError`); `user_event` drops events whose generation is stale, so a slow superseded load can't corrupt the new DOM or paint over the newer page (a race now reachable because the UI no longer blocks during a load). **Scroll restoration:** back/forward (and bfcache restore) stash the target offset in `Lumen.pending_restore_scroll` before `reload()` and `apply_loaded_page` applies it after resetting scroll to the top — the old code set `scroll_x/y` right after the (then synchronous) `reload()` returned, which would now be clobbered when the async `LoadDone` arrives. Stage 2 (the final `render_bytes` — incl. QuickJS — off the UI thread, ADR-006) remains, blocked by QuickJS not being `Send`.
- **Done (Web Storage persistence):** `Lumen` stores `ls_storage: HashMap<String, Arc<Mutex<WebStorage>>>` keyed by SOP origin (`scheme://host:port`). `ls_store_for_base(&ResourceBase, &mut HashMap) -> Option<Arc<Mutex<WebStorage>>>` — for URL-based pages: constructs origin key, inserts fresh `WebStorage` on first visit, returns `Arc::clone`; for file/snapshot/empty — returns `None` (no persistent storage). `origin_str()` on `PageSource` extracts the key via `lumen_core::url::Url::parse`. `ls_store` threaded through `render_bytes → parse_and_layout → run_scripts_with_dom → install_dom`. `sessionStorage` stays fresh per page load (created inside `install_dom`, never stored in shell). Phase 0: in-memory only, no disk persistence.

After JS click dispatch, shell polls `take_navigate_request` to handle `location.href=` navigation triggered by handlers.
- **Done (link click navigation + fragment anchors):** module `lumen-shell::links`. `find_link_href(doc, node_id) -> Option<String>` walks up the DOM ancestor chain from the hit-tested node to find the nearest `<a>` element with a non-empty `href` attribute (HTML5 activation behavior). `is_navigable_href(href) -> bool` suppresses `javascript:`, `mailto:` and fragment-only hrefs (`#id`). **Fragment-only navigation:** `fragment_only(href) -> Option<&str>` detects `#`-prefixed hrefs; `find_element_by_id(doc, id_value) -> Option<NodeId>` walks DOM by `id` attribute (HTML LS §3.2.6 case-sensitive); `Lumen::navigate_fragment(fragment)` updates `doc.set_target`, calls `relayout()` so `:target` CSS takes effect, then scrolls to the element rect. **`:target` on initial load:** `parse_and_layout` extracts URL fragment via `Url::fragment()` and calls `doc.set_target(fragment)` before `layout_measured`, so `:target` pseudo-class matches on first paint. 12 unit tests.

## Gotchas

- **HiDPI / DPR partially supported.** Renderer stores `scale_factor` from winit. `ScaleFactorChanged` updates it on-the-fly. **Not yet supported:** layout viewport is hardcoded 1024×720 — `inner_size` not passed to layout, no relayout on `Resized`. Requires structural pipeline refactor (layout runs before window creation) — P1 concern.
- **Scroll state implemented** (Y-axis, smooth scroll out-cubic 200 ms, scrollbar drag + track-click, find-scroll-to-match, cursor feedback). Limitations: no X-axis, no momentum, no relayout-on-resize.
- **Streaming pipeline (task 4B) implemented.** Window opens immediately; HTML is parsed incrementally with `IncrementalTreeBuilder`. HTTP fetch is still blocking in background thread — TCP-level streaming is a separate task.
