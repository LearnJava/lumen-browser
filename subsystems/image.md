- **Done (RP-5 SVG sniffing, 2026-07-02):** `pub fn is_svg(bytes) -> bool` тАФ SVG has no magic bytes, so the sniffer skips an optional UTF-8 BOM + ASCII whitespace and matches `<svg` / `<!DOCTYPE svg` / `<?xml` prolog followed by `<svg` within the first 4 KB (all ASCII-case-insensitive; the prolog alone is not enough тАФ XHTML starts the same way). `image/svg+xml` added to `supported_mime_types()` (for the `<picture>` type-filter). `decode`/`decode_raw` intentionally still return `UnknownFormat` for SVG тАФ rasterization lives in the shell (`svg_image.rs`) which renders the markup through the normal layout/paint pipeline; lumen-image must not depend on parser/layout/paint (dependency direction). 4 unit tests. Implemented via Laguna M.1 (poolside) under P1 review (mixed programming).
- **Done (sniff-dispatch тАФ `decode(bytes)`):** public `decode(bytes: &[u8]) -> Result<Image, ImageError>` distinguishes PNG / JPEG / WebP by magic bytes. `ImageError` enum: `UnknownFormat` / `Png(DecodeError)` / `Jpeg(JpegError)` / `Webp(WebpError)` + `From`-impls + `Display`.
- **Done (JPEG via zune-jpeg 0.5):** `decode_jpeg(bytes) -> Result<Image, JpegError>`. Two-pass decode: pass 1 reads headers via `decode_headers()` + `input_colorspace()` to detect grayscale; pass 2 decodes with `jpeg_set_out_colorspace(Luma)` for grayscale or default RGB otherwise. Output: `Gray8` (1-component) or `Rgb8` (YCbCr). Browser-accurate bilinear chroma upsampling (vs. nearest-neighbour in old custom decoder). Supports baseline, progressive, restart intervals, ICC profiles. `JpegError(String)` newtype.
- **Done (PNG via zune-png 0.5):** `decode_png(bytes) -> Result<Image, DecodeError>`. Options: `png_set_strip_to_8bit(true)` тАФ 16-bit pixels stripped to 8-bit. BPP detection from `pixels.len() / (w * h)` handles tRNS expansion automatically (GrayтЖТGrayAlpha, RGBтЖТRGBA, PaletteтЖТRGBA). Supports all color types, interlacing, tRNS, sub-byte depths. `DecodeError`: `InvalidSignature` | `Decode(String)`.
- **Done (WebP via image-webp 0.2):** `decode_webp(bytes) -> Result<(u32,u32,Vec<u8>), WebpError>` in `crates/engine/image/src/webp/mod.rs`. RIFF container detection (`is_webp` тАФ bytes 0тАУ3=`RIFF`, 8тАУ11=`WEBP`). VP8 (lossy YUVтЖТRGBA8) and VP8L (lossless) via `WebPDecoder`. RGB8 images (no alpha) expanded to RGBA8 by appending `255` alpha per pixel. `WebpImageDecoder` implements `lumen_core::ext::ImageDecoder` trait for plug-in architecture. `image/webp` included in `supported_mime_types()` for `<picture>` type-filter.
- **Done (resize):** `resize_bilinear(img, w, h)` тАФ bilinear downscale for all `PixelFormat` variants. `resize_area_avg(img, w, h)` тАФ area-averaging (box filter) is the **production downscale kernel** (femtovg `resolve_image_for_rect`, `cpu_raster`, wgpu `renderer`). **Invariant (BUG-219, 2026-07-04):** do NOT swap this for a sharper separable kernel to chase Edge pixel-parity тАФ Lanczos-3 and Mitchell-Netravali bicubic were both implemented and graphic-tested; neither wins uniformly across TEST-18/19 (Mitchell trims TEST-18 but regresses TEST-19; Lanczos worse on both тАФ ringing/halos on high-contrast photos). The remaining тЙИ2% diff vs Edge is inherent resampling+AA parity (Edge's Skia downscale is proprietary/content-adaptive), class rule 2/3 тАФ a KNOWN_DEBTOR, not a fixable defect.
- **Done (ImageDecoder trait):** `lumen_core::ext::ImageDecoder` тАФ plug-in trait for format-specific decoders; `format_name()`, `sniff()`, `mime_types()`, `decode_rgba8()`. Anchor for future AVIF/GIF decoders.
- **Done (ImageDecodeCache тАФ ADR-008 ┬з10E.1+10E.2):** `ImageHandle` (`Arc<Image>`) thin ref; `ImageKey(String)` cache key; `ImageDecodeCache` тАФ LRU decode cache with configurable memory budget (default 256 MB). `insert()` returns handle and triggers `evict_to_budget()` when budget exceeded. `decode_or_get(key, closure)` тАФ cache-aside pattern for decoders. `lru_candidates()` for external eviction control. Callers hold `ImageHandle`; evicted entries keep data alive until all handles are dropped. 9 unit-tests in `decode_cache.rs`. - **Done (ICC-3 matrix-shaper colour management):** `Image::to_rgba8()` now applies the profile's real matrix-shaper transform when the image carries an RGB ICC profile тАФ `lumen_core::icc::IccProfile::parse(...).build_rgb_transform()` (per-channel TRC decode + colorant matrix тЖТ D65 тЖТ sRGB), via the helper `apply_icc_rgb_transform(transform, &mut rgba)`. Falls back to the legacy gamut tone-mapping (`apply_tone_mapping` / `IccGamut`) only when no matrix-shaper can be built (LUT-only / non-RGB profiles тЖТ ICC-4). `correct_rgba_pixels` (wgpu upload path) was upgraded the same way. Effect: P3 / Rec.2020 / Adobe RGB / ProPhoto photos render colour-correct in the femtovg window, CPU snapshot and PDF export, not just the three hard-coded gamuts. CMYK-via-ICC (`A2B0` LUT) is still deferred to ICC-4.
- **Done (ICC-4 CMYK LUT colour management):** the JPEG decoder colour-manages CMYK/YCCK JPEGs that carry an embedded CMYK ICC profile. `try_decode_cmyk_icc` builds a `lumen_core::icc::CmykTransform` from the profile's `A2B0` table, decodes the raw 4-channel samples (zune `out_colorspace == input`), reconstructs device ink coverage (Adobe-APP14-inversion aware via `jpeg_has_adobe_marker`; YCCK first converted to stored CMY through `ycbcr_to_rgb`), and runs each pixel through the profile тЖТ gamma-encoded sRGB `Rgb8` (no residual ICC profile attached). Falls back to zune's built-in CMYKтЖТRGB when no usable CMYK profile is present.
- **Done (ICC-5 decode-time colour management + transform cache):** `decode` now splits into `decode_raw` (native-format decode, profile attached) + `color_manage_in_place`, which applies the matrix-shaper (or gamut fallback) once, in place, on the compact source buffer (stride 3 for `Rgb8`, 4 for `Rgba8`; grayscale deferred to `to_rgba8`) and then clears `icc_profile`. So each image's colour transform runs exactly once at decode тАФ register / resize / paint all operate on already-sRGB pixels тАФ instead of re-running per paint or per re-register. `to_rgba8`, `correct_rgba_pixels` and the JPEG CMYK path build transforms through the process-wide cache (`lumen_core::icc::cached_rgb_transform` / `cached_cmyk_transform`), so a profile is parsed/compiled at most once. Helpers `apply_rgb_transform_packed` / `convert_packed_to_srgb` are stride-generic (legacy RGBA8 wrappers retained). Graceful sRGB fallback unchanged.
- **Done (ICC-6 tests + observability + PNG-iCCP fix):** end-to-end colour-management coverage. **BUG-229:** `parse_png_icc_profile` (`png/mod.rs`) inflated the `iCCP` chunk with a raw `DeflateDecoder`; PNG stores the profile as a *zlib* datastream (RFC 1950), so every real encoder's profile silently failed to inflate and was dropped тАФ no PNG was ever colour-managed.
