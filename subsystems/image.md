# lumen-image ✅ (PNG + JPEG via zune-jpeg + zune-png + WebP via image-webp)

- **Done (sniff-dispatch — `decode(bytes)`):** public `decode(bytes: &[u8]) -> Result<Image, ImageError>` distinguishes PNG / JPEG / WebP by magic bytes. `ImageError` enum: `UnknownFormat` / `Png(DecodeError)` / `Jpeg(JpegError)` / `Webp(WebpError)` + `From`-impls + `Display`.
- **Done (JPEG via zune-jpeg 0.5):** `decode_jpeg(bytes) -> Result<Image, JpegError>`. Two-pass decode: pass 1 reads headers via `decode_headers()` + `input_colorspace()` to detect grayscale; pass 2 decodes with `jpeg_set_out_colorspace(Luma)` for grayscale or default RGB otherwise. Output: `Gray8` (1-component) or `Rgb8` (YCbCr). Browser-accurate bilinear chroma upsampling (vs. nearest-neighbour in old custom decoder). Supports baseline, progressive, restart intervals, ICC profiles. `JpegError(String)` newtype.
- **Done (PNG via zune-png 0.5):** `decode_png(bytes) -> Result<Image, DecodeError>`. Options: `png_set_strip_to_8bit(true)` — 16-bit pixels stripped to 8-bit. BPP detection from `pixels.len() / (w * h)` handles tRNS expansion automatically (Gray→GrayAlpha, RGB→RGBA, Palette→RGBA). Supports all color types, interlacing, tRNS, sub-byte depths. `DecodeError`: `InvalidSignature` | `Decode(String)`.
- **Done (WebP via image-webp 0.2):** `decode_webp(bytes) -> Result<(u32,u32,Vec<u8>), WebpError>` in `crates/engine/image/src/webp/mod.rs`. RIFF container detection (`is_webp` — bytes 0–3=`RIFF`, 8–11=`WEBP`). VP8 (lossy YUV→RGBA8) and VP8L (lossless) via `WebPDecoder`. RGB8 images (no alpha) expanded to RGBA8 by appending `255` alpha per pixel. `WebpImageDecoder` implements `lumen_core::ext::ImageDecoder` trait for plug-in architecture. `image/webp` included in `supported_mime_types()` for `<picture>` type-filter.
- **Done (resize):** `resize_bilinear(img, w, h)` — bilinear downscale for all `PixelFormat` variants.
- **Done (ImageDecoder trait):** `lumen_core::ext::ImageDecoder` — plug-in trait for format-specific decoders; `format_name()`, `sniff()`, `mime_types()`, `decode_rgba8()`. Anchor for future AVIF/GIF decoders.
- **Done (ImageDecodeCache — ADR-008 §10E.1+10E.2):** `ImageHandle` (`Arc<Image>`) thin ref; `ImageKey(String)` cache key; `ImageDecodeCache` — LRU decode cache with configurable memory budget (default 256 MB). `insert()` returns handle and triggers `evict_to_budget()` when budget exceeded. `decode_or_get(key, closure)` — cache-aside pattern for decoders. `lru_candidates()` for external eviction control. Callers hold `ImageHandle`; evicted entries keep data alive until all handles are dropped. 9 unit-tests in `decode_cache.rs`. Exported from `lumen-image` as `pub use decode_cache::{ImageDecodeCache, ImageHandle, ImageKey}`.
- **Deferred:** AVIF / GIF / JXLF (Phase 2+), ICC color pipeline (zune-jpeg ICC exposed; rendering side needs integration), 10E.4 (scroll-discard), 10D.3 (cross-tab cache crate).
- 45 tests (WebP: 7; PNG: updated; decode_cache: 9; sniff-dispatch and lib unit tests).
