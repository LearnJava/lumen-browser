# lumen-a11y

Accessibility tree construction and platform bridge infrastructure.

## Scope

Builds a language-neutral `AXTree` from a Lumen DOM `Document`. Each `AXNode` maps to a
WAI-ARIA role, accessible name, description, and a full set of ARIA state/property flags.
Platform bridge stubs route tree updates to OS accessibility APIs.

**Not in scope:** JavaScript `aria-*` mutation observation (P3 task), live-region
announcement timing (P3 task), actual COM/Objective-C/D-Bus Phase 1 native bindings.

## Done

- **AX tree construction** (`lib.rs`): `build_ax_tree(doc, root_id, flat_tree)` → `AXTree`.
  Shadow DOM composed-tree traversal via `FlatTree`. `aria-hidden` subtrees excluded.
- **67 ARIA roles** (`roles.rs`): `AXRole` enum + `implicit_role(node)` from HTML tag.
  `AXRole::as_str()` for string serialization. Context validation (row requires table parent, etc.).
- **Accessible name/description** (`names.rs`): WAI-ARIA Accessible Name and Description §4.
  aria-labelledby → aria-label → alt → text content → title chain.
- **Full ARIA state set** (`lib.rs`): `AXState` — checked / disabled / expanded / hidden /
  selected / pressed / required / readonly / invalid / busy / tab_index / level / set_size /
  pos_in_set / live / multiline / multiselectable / horizontal / current / modal /
  role_description / value_now / value_min / value_max / value_text.
- **Relationship attributes**: aria-controls / aria-owns / aria-flowto / aria-details —
  stored as `NodeId` references.
- **Constraint-validation integration** (`lib.rs`): `element_validity()` feeds `AXState::invalid`
  when aria-invalid is absent but the form control fails HTML5 validation.
- **Platform bridge infrastructure** (`platform/`): `PlatformBridge` trait + `NullBridge`.
  `platform_bridge()` factory selects `WinUiaBridge` / `MacA11yBridge` / `AtSpiBridge`
  at compile time by `target_os`. Phase 0: all three hold the last `AXTree` clone in memory.
- **Shell wiring** (`shell/src/main.rs`): `Lumen::update_platform_ax_tree()` called after every
  full page load (`apply_loaded_page`) and tab restore (`restore_page_snapshot`).
  `focused_node_changed()` called on click focus change.
- **Tests**: 125 integration tests (`tests/ax_tree.rs`) + 6 platform unit tests (`platform/mod.rs`).
  Total: 131 tests pass.

## Deferred

- **Phase 1 native bindings**: Windows UIA COM (`IRawElementProviderSimple`), macOS
  `NSAccessibilityElement` + notifications, Linux AT-SPI2 D-Bus service via `atspi`+`zbus`.
  Each platform file has detailed Phase 1 comments. No new Cargo deps required for Windows
  (uses `windows` crate already in the dep graph via winit).
- **Live region announcement timing**: `aria-live="polite/assertive"` — state is captured,
  announcement scheduling is shell/P3 territory.
- **`aria-*` mutation observation**: P3 task — hook DOM mutation events to call
  `update_platform_ax_tree()` between full page loads.

## Invariants

- `AXTree` has exactly one root node (the document root).
- `aria-hidden="true"` nodes are excluded entirely from the tree (subtree pruned, not just flagged).
- `NodeId` references (controls/owns/flowto/details) may point to nodes absent from the AX tree
  if the target node is aria-hidden.
- Platform bridges must be `Send + 'static` (box-erased, moved into `Lumen` struct on main thread).
- The `platform_bridge()` factory does not fail — returns `NullBridge` on unsupported platforms.
