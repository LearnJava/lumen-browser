# lumen-paint 🟡 (fill rects + textured text + textured images + FontMeasurer)

- **Done:** `DisplayCommand` enum (FillRect, DrawBorder, DrawText, **DrawOutline**, **DrawImage**), `build_display_list` walk of LayoutBox with painter's order. Per-fragment text-shadow emission (CSS Text Decoration L3 §6): for each `frag.style.text_shadow` element — DrawText copy with offset and shadow color. Multiple shadows emitted in reverse CSS-list order (painter's order: first shadow on top of others). Phase 0 limitation: `blur` ignored (requires off-screen wgpu pass with Gaussian convolution, P2 task 4+). **CSS Backgrounds L3 §4.6 box-shadow** (outset + inset): outset shadows emitted as `FillRect` BEFORE background; inset shadows emitted BETWEEN background and border, **without clip-pipeline**: outer = padding-box, inner = offset+inset by spread; visible shadow = outer minus intersection of outer and inner, expanded to up to 4 FillRect ring segments (top/bottom/left/right). Phase 0: `blur` ignored for both outset and inset. **CSS Backgrounds L3 §3.8 background-clip**: helper `background_clip_rect(b)` determines rect for background FillRect by `style.background_clip`: `BorderBox` (initial) — `b.rect`; `PaddingBox` — shrink by border-widths; `ContentBox` — shrink by border + padding. `Text` (L4) Phase 0 fallback to `BorderBox`. **CSS Display L3 §4 visibility:hidden** in paint emission: `is_paint_visible(b)` gate on self-emission. Block: children always traversed. InlineRun: per-frag visibility check. **CSS Color L3 §3.2 opacity:0** Phase 0 skip: `is_opacity_subtree_painted(b)` — early-return for entire subtree (opacity > 0 && < 1 Phase 0 not composited — requires off-screen pass, P2 task 4+). **CSS Basic UI L4 §5 — outline rendering**: emits `DrawOutline { rect, width, style, color, offset }` after children. Renderer draws outline as 4 fill-quads. Phase 0: `Solid` / `Auto` as single fill-quad, `Dashed` / `Dotted` via `emit_outline_side` with `dash_segments`. wgpu Renderer with **three** pipelines: fill (vertex pos + color), text (vertex pos + uv + color), **image (vertex pos + uv, RGBA-texture per image)**. Three WGSL shaders, common uniform (viewport), bind group for atlas (R8 texture + linear sampler) and `image_bgl` for RGBA images. `GlyphAtlas` 512×512 with shelf packer. `FontMeasurer<'a>` — implementation of `TextMeasurer` based on TTF hmtx/cmap for shell. Per-glyph metadata cache. Atlas uploaded to GPU only when dirty. **`DrawBorder` renderer (CSS Backgrounds L3 §6, dashed/dotted/double):** `DisplayCommand::DrawBorder { rect, widths, colors, styles }` — `styles: [BorderStyle; 4]` per-side. `emit_border_side` per edge: `Solid`/`None` → one fill-quad; `Dashed` → pattern `(2·w, w)`; `Dotted` → pattern `(w, w)`; **`Double` → two parallel rects ~1/3 border-width each, gap ~1/3 (w < 3px falls back to Solid)**. `BorderStyle` extended with `Double` variant; parser updated (`double` keyword). 6 unit tests in `renderer.rs`. Phase 0 without mitre-joints — edges rectangular, overlap at corners.
- **Done (CSS Fonts L4 §3.1 plumbing):** `DisplayCommand::DrawText` carries `font_family: Vec<String>`, `font_weight: FontWeight` and `font_style: FontStyle` from `InlineFrag.style`. Snapshot serializer prints them only for non-default values.
- **Done (real face switch in renderer):** `Renderer` holds `Vec<LoadedFace>`, `face_id_by_path: HashMap<PathBuf, usize>` for dedup, `font_provider: Option<Arc<dyn FontProvider>>`. `cached_glyphs` key is now `(face_id, glyph_id)`. Method `resolve_face_id(families, weight, style)`: iterates names in CSS Fonts L4 §3.1 order, for each calls `provider.pick_face`. Generic CSS-family (`serif`/`sans-serif`/`monospace`/`cursive`/`fantasy`/`system-ui`) skipped — Phase 0 no per-generic tables; ultimate fallback to face 0.
- **Done (codepoint cascade, CSS Fonts L4 §5.3):** at start of `render()` all loaded faces parsed into `Vec<Option<ParsedFace>>` once per frame. `push_text_glyphs` for each char tries primary face: if `cmap.glyph_index(cp)` returns `Some(non-zero)` — primary selected; otherwise iterates other loaded faces; if none have it — renders `.notdef` from primary. Per-DrawText cache `HashMap<char, (face_id, glyph_id)>` avoids repeat lookups for same characters in one line. Cascade only covers already-loaded faces. `pub const CURATED_FALLBACK_FAMILIES: &[&str]` in `lumen-paint::fallback` (24 names: Noto Color Emoji / Apple Color Emoji / Segoe UI Emoji / Twemoji Mozilla / Noto Sans CJK SC|JP|KR|TC / Microsoft YaHei / Yu Gothic / Malgun Gothic / Hiragino Sans / PingFang SC|TC / Noto Sans Arabic|Hebrew|Devanagari|Tamil|Bengali|Telugu|Gujarati|Thai + Segoe UI; emoji listed before CJK). `Renderer::preload_curated_fallbacks()` — shortcut. 7 unit tests on list.
- **Done (image pipeline + register_image API):** new wgpu pipeline for RGBA-texture quad with per-image bind group. `register_image(src, &Image)` loads decoded pixels to GPU; re-registration replaces old entry. `unregister_image(src)`, `clear_images()`, `has_image(src)`. `ImageRegisterError` enum: `EmptyImage` / `TooLarge { w, h, max }`. `convert_to_rgba`: Gray8/GrayAlpha8/Rgb8/Rgba8 conversions. Without cache hit — fallback to light-gray placeholder fill-quad.
- **Done (object-fit / object-position, CSS Images L3 §5.5):** `DisplayCommand::DrawImage` carries `object_fit: ObjectFit` + `object_position: ObjectPosition`. `fit_image_quad(box_rect, intrinsic_size, fit, position) -> Option<(Rect, [f32;2], [f32;2])>` computes visible rect + UV-bounds; for `cover` / `none` with image larger than box — box-clipped quad with clamped UV.
- **Done (text-decoration):** FillRect-s emitted for each fragment with non-empty `TextDecorationLine`: underline — below baseline (+10% fs), line-through — above baseline (-30% fs), overline — near line top (-78% fs). **text-decoration-style (CSS Text Decoration L3 §2.2)**: `emit_decoration_line` expands one line into set of FillRect-s by `frag.style.text_decoration_style`: `Solid` — one rect; `Double` — two parallel; `Dotted` — series of squares `t × t`, step `2·t`; `Dashed` — dashes `2·t × t`, step `3·t`; `Wavy` — series of narrow axis-aligned columns (`emit_wavy_line`) with sin-offset of center. **text-decoration-thickness (CSS Text Decoration L3 §2.3)**: `resolve_decoration_thickness` — `Auto`/`FromFont` ≈ 7% × font-size (min 1px), `Length(px)` — px directly, `Percentage(frac)` — `frac × font_size`.
- **Done:** `serialize_display_list(&[DisplayCommand]) → String` — deterministic text format for snapshot tests. 6 integration golden tests. Mechanism `UPDATE_SNAPSHOTS=1` for regenerating golden files.
- **Done (Compositor scaffolding + two-buffer commit, P2 1B):** new module `lumen-paint::compositor` — traits `Layer`, `LayerTree`, `Compositor`. **Two-buffer model (like Chromium):** `commit(trees, layer_tree)` puts state in pending-buffer without touching active; `flush_pending() -> bool` atomically promotes pending → active. Concrete Phase 0 impls: `BasicLayer` / `BasicLayerTree`, `InProcessCompositor` (synchronous single-thread promotion).
- **Done (Compositor thread, P2 1B step b):** `ThreadedCompositor` + `ThreadedCompositorHandle` — thread-safe variant on `Arc<Mutex<ThreadedState>>`. Multi-thread tests: `cross_thread_commit_and_flush`, `cross_thread_concurrent_commits_last_wins`.
- **Done (DisplayCommand layer-ops definitions, P2 1B):** `BlendMode` enum (17 modes) with `from_keyword(&str)`. Variants `DisplayCommand::PushClipRect { rect } / PopClip`, `PushOpacity { alpha } / PopOpacity`, `PushBlendMode { mode } / PopBlendMode`. Renderer currently ignores via `_` match-arm — real layer-pipeline is P2 1B step (c).
- **Done (DisplayCommand layer-ops emission, P2 1B step a):** `build_display_list_ordered` emits `PushOpacity`/`PopOpacity` for `opacity<1`, `PushBlendMode`/`PopBlendMode` for `mix-blend-mode != Normal`, `PushClipRect`/`PopClip` for `overflow` ∈ {hidden, clip, scroll, auto}. 13 unit tests.
- **Done (DPR / scale_factor support, [P3] 3A step a):** `Renderer` stores `scale_factor: f64`, initialized in `new_async` from `Window::scale_factor()`. Surface stays in physical pixels, but in `render()` viewport uniform recalculated as `(config.w / scale_factor, config.h / scale_factor)` — gives shader CSS-px coordinates. Pub API: `set_scale_factor(f64)` + getter.
- **Done (P2 item 2B — stacking-aware hit testing):** new module `lumen-paint::hit_test` — `hit_test(point: Point, root: &LayoutBox) -> Option<HitTestResult>`. Returns top DOM-node under cursor + `local_point` + `path: Vec<NodeId>`. Algorithm — reverse CSS painting order. **Transform inversion:** for boxes with `style.transform` builds forward-matrix and inverts via `Mat4::invert_2d_affine()`. 14 unit tests.
- **Done (P2 item 2A renderer-side — paint order consumer):** `build_display_list_ordered(root, &StackingTree, &PaintOrder) -> DisplayList`. Bucket-per-SC. Phase 0: phases 3/4/5 lumped into one bucket. Old `build_display_list` kept for backward compat. 4 unit tests.
- **Done (multi-size + variation-aware glyph atlas, P2 quality):** `SIZE_BINS = [8, 12, 16, 20, 24, 32, 48, 64]`. `size_bin_for(font_size)` rounds up to nearest bin. Atlas cache key — `AtlasKey { face_id: u16, glyph_id: u16, size_bin: u16, coords_hash: u64 }`. ATLAS_DIM raised from 512 to 1024. **`DisplayCommand::DrawText` extended with field `font_variation_coords: Vec<f32>`**. Phase 0: all text-emission sites emit empty Vec. 16 unit tests.
- **Done (PushClipRect/PopClip → wgpu scissor rect, P2 1B step c part 1):** renderer transitioned from three flat draw-stages to ordered `Vec<DrawOp>`. `sync_scissor_to_stack` lazily pushes `DrawOp::SetScissor(DeviceScissor)`. DPR-aware. Phase 0: real clipping for `overflow: hidden` / `clip-path: inset` now works.
- **Done (PushOpacity/PopOpacity → alpha-multiply, Phase 0 approximation, P2 1B step c part 2):** renderer holds `opacity_stack: Vec<f32>`. Before each fill/text/image draw-command computes `effective_alpha(opacity_stack)` (product of all). Phase 0 limitation: overlapping elements within one opacity-group rendered with pairwise alpha-blend instead of single-pass off-screen composite. 9 unit tests.
- **Done (DrawOutline dashed/dotted rendering, CSS Basic UI L4 §5.3):** `emit_outline_side(out, side_rect, horizontal, width, color, style)`. Pure function `dash_segments(total_length, dash_len, gap_len) -> Vec<(offset, length)>` centers pattern. `OutlineStyle::Auto` rendered as Solid. 9 unit tests.
- **Deferred:** pixel snapshot tests, GPU-pipeline for rounded corners/gradients/shadows, real compositor-thread tick-loop with separate `JoinHandle` (Mutex-wrapper over state already ready — `ThreadedCompositor`/`Handle`; need thread reading pending every N ms and doing GPU upload), real off-screen-layer pipeline for `PushOpacity` (true single-pass alpha composite) + `PushBlendMode`, exact breakdown of `build_display_list_ordered` into phases BlockBackgrounds / Floats / InlineContent, `BorderStyle::Groove` / `Ridge` / `Inset` / `Outset`.
