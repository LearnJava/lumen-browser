# lumen-knowledge ✅ (FTS5 over history + notes + read-later + omnibox integration)

- **Done (ReadLater):** §12.3 read-later / offline-reading storage layer. `ReadLaterEntry { id, url, title, html_snapshot: Vec<u8>, text, status: ReadStatus, saved_at, last_accessed, tags }`. `ReadStatus { Unread (default), Read, Archived }`. External content FTS5 `read_later_fts(title, text)` with AI/AD/AU triggers for auto-sync. `save(url, title, html_snapshot, text, tags, saved_at) -> id` (upsert, saved_at preserved on conflict; tags overwritten with deduplication); `set_status(id, status)`; `touch(id, now_unix)` updates last_accessed (for LRU eviction); `search(query, limit) -> Vec<ReadLaterSearchHit>` with snippet + bm25 + tags; `delete(id)` (cascades tags). 15 unit tests.
- **Done (Notes):** §12.2 annotations and notes. `Note { id, url, selection, context, comment, created_at }`. External content FTS5 `notes_fts(selection, comment)` with `content = 'notes'`, `content_rowid = 'id'` (FTS5 §4.4.3). FTS index synchronized via 3 triggers (AFTER INSERT / DELETE / UPDATE on `notes`). `add(url, selection, context, comment, created_at) → id`, `update(id, selection, context, comment)`, `search(query, limit) → Vec<NoteSearchHit>` with snippet + bm25-ranking. 13 unit tests.
- **Done (HistoryFts):** §12.1 "Full-text search over history" — basic FTS5 index. Virtual table `history_fts(url, title, text)` with tokenizer `unicode61 remove_diacritics 2` — built-in SQLite Unicode word-boundary tokenizer with lowercase normalization and removal of combining diacritics. Ranking via built-in `bm25(history_fts)` (smaller = more relevant per FTS5 spec). `index(rowid, url, title, text)` — upsert via transaction (DELETE+INSERT; FTS5 doesn't support ON CONFLICT DO UPDATE), `unindex(rowid)`, `search(query, limit) -> Vec<SearchHit>` (rowid + url + title + snippet with markdown-highlight `**...**` + bm25-score), `clear()`. Query syntax — native FTS5: `"phrase"`, implicit AND, `OR`, `^foo`, etc. `rowid` typically matches `lumen-storage::history::HistoryEntry.id` — when sharing a DB file, the two modules work together.
- **Done (fts-omnibox):** §12.1 shell integration — `OmniboxSuggestion { HistoryFts, SearchQuery }` enum in `address_bar.rs`. `parse_omnibox_prefix()` detects `@history <query>` vs. plain input. `AddressBarState::set_suggestions / select_next / select_prev / selected_idx()`. Dropdown renders up to 7 rows (label + sub-label for HistoryFts, label for SearchQuery, frequency tag). `query_omnibox_suggestions()` in main.rs: plain → `SearchHistory::prefix_match` (4 hits) + `HistoryFts::search` (4 hits); `@history` → `HistoryFts::search` only (7 hits). ArrowDown/Up navigate dropdown; Enter on selected suggestion commits its URL. Plain text queries recorded to `SearchHistory`. HistoryFts + SearchHistory opened in-memory (Phase 0); Phase 2: profile DB.
- **Done (KnowledgeStore + @notes omnibox):** §12.1–12.4 `KnowledgeStore` trait in `lumen-core::ext` — pluggable backend facade over HistoryFts, Notes, ReadLater, OpenTabsIndex. `DefaultKnowledgeStore` in `lumen-knowledge::store` wraps all four. `@notes <query>` prefix in omnibox (`OmniboxPrefix::Notes`): `parse_omnibox_prefix()` detects prefix, `query_omnibox_suggestions()` calls `KnowledgeStore::search_notes(query, 7)` → up to 7 `NoteSearchHit` suggestions with url/snippet sub-label. `notes_store: Box<dyn KnowledgeStore>` in `App` struct; initialized to `DefaultKnowledgeStore::new_in_memory()` in Phase 0.
- **Done (OpenTabsIndex):** §12.4 "Search over open tabs" — live, in-memory-only FTS5 index over currently-open tabs. Distinct from `HistoryFts` (disk mirror keyed by `History.id`): the key is the live shell `tab_id`, entries never persist to disk (closing the app = empty index on next start), and shell re-indexes on the fly. Virtual table `open_tabs_fts(url, title, text)`, tokenizer `unicode61 remove_diacritics 2`, bm25 ranking. `new()` (in-memory only — no `open(path)` by design), `index_tab(tab_id, url, title, text)` — upsert via DELETE+INSERT transaction keyed by `tab_id`, `remove_tab(tab_id)` (no-op if absent, called on tab close), `search(query, limit) -> Vec<OpenTabHit>` (tab_id + url + title + snippet `**...**` + bm25-score), `count()`, `clear()`. Purpose: omnibox `@tabs <query>` / tab switcher — "find among open tabs". 13 unit tests. **Shell-wiring handoff (P3):** one process-global `OpenTabsIndex`; `index_tab` after each tab load/navigation, `remove_tab` on close; `search` for omnibox `@tabs`.
- **Deferred:** §12.2 Range API (for restoring highlight overlays on page re-open via DOM range coordinates), §12.3 background resource loading on save (downloader for `<img>` / `<link>`), §12.5 local AI (HNSW vector index, optional), §12.7 session export/import, **Russian Porter-stemmer** via external FTS5 tokenizer (C-callback), ё↔е equivalence (custom tokenizer pre-pass), export of notes to Markdown / JSON, `@read-later` / `@tabs` prefixes in omnibox.
- 75 unit tests: 11 HistoryFts (index+search, no-hits, implicit AND, OR query, BM25 ranking, cyrillic text, index overwrites, unindex, search limit, clear, case-insensitive) + 13 Notes + 15 ReadLater + 13 OpenTabsIndex + 23 KnowledgeStore/DefaultKnowledgeStore (search_notes: found, not-found, limit; @notes omnibox prefix parsing: with/without space, plain text, @history, empty).
