- **Done (HistoryFts):** ¬ß12.1 "Full-text search over history" ‚Äî basic FTS5 index. Virtual table `history_fts(url, title, text)` with tokenizer `unicode61 remove_diacritics 2` ‚Äî built-in SQLite Unicode word-boundary tokenizer with lowercase normalization and removal of combining diacritics. Ranking via built-in `bm25(history_fts)` (smaller = more relevant per FTS5 spec). `index(rowid, url, title, text)` ‚Äî upsert via transaction (DELETE+INSERT; FTS5 doesn't support ON CONFLICT DO UPDATE), `unindex(rowid)`, `search(query, limit) -> Vec<SearchHit>` (rowid + url + title + snippet with markdown-highlight `**...**` + bm25-score), `clear()`. Query syntax ‚Äî native FTS5: `"phrase"`, implicit AND, `OR`, `^foo`, etc. `rowid` typically matches `lumen-storage::history::HistoryEntry.id` ‚Äî when sharing a DB file, the two modules work together.
- **Done (fts-omnibox):** ¬ß12.1 shell integration ‚Äî `OmniboxSuggestion { HistoryFts, SearchQuery }` enum in `address_bar.rs`. `parse_omnibox_prefix()` detects `@history <query>` vs. plain input. `AddressBarState::set_suggestions / select_next / select_prev / selected_idx()`. Dropdown renders up to 7 rows (label + sub-label for HistoryFts, label for SearchQuery, frequency tag). `query_omnibox_suggestions()` in main.rs: plain ‚Üí `SearchHistory::prefix_match` (4 hits) + `HistoryFts::search` (4 hits); `@history` ‚Üí `HistoryFts::search` only (7 hits). ArrowDown/Up navigate dropdown; Enter on selected suggestion commits its URL. Plain text queries recorded to `SearchHistory`. HistoryFts + SearchHistory opened in-memory (Phase 0); Phase 2: profile DB.
- **Done (OpenTabsIndex):** ¬ß12.4 "Search over open tabs" ‚Äî live, in-memory-only FTS5 index over currently-open tabs. Distinct from `HistoryFts` (disk mirror keyed by `History.id`): the key is the live shell `tab_id`, entries never persist to disk (closing the app = empty index on next start), and shell re-indexes on the fly. Virtual table `open_tabs_fts(url, title, text)`, tokenizer `unicode61 remove_diacritics 2`, bm25 ranking. `new()` (in-memory only ‚Äî no `open(path)` by design), `index_tab(tab_id, url, title, text)` ‚Äî upsert via DELETE+INSERT transaction keyed by `tab_id`, `remove_tab(tab_id)` (no-op if absent, called on tab close), `search(query, limit) -> Vec<OpenTabHit>` (tab_id + url + title + snippet `**...**` + bm25-score), `count()`, `clear()`. Purpose: omnibox `@tabs <query>` / tab switcher ‚Äî "find among open tabs". 13 unit tests. **Shell-wiring handoff (P3):** one process-global `OpenTabsIndex`; `index_tab` after each tab load/navigation, `remove_tab` on close; `search` for omnibox `@tabs`.
