# lumen-core ✅ (foundation is stable)

- Types: `Error`, `Result<T>`, `Url`, `Event` (TabCreated/Closed, Navigation, PageLoaded, RequestStarted/Completed/Blocked), `TabId`, `Capability`, `CapabilityToken`, `Module` trait.
- Geometry: `Rect`, `Point`, `Size`.
- `lumen-core::ext` — extension trait points: `NetworkTransport`, `StorageBackend` (with origin-partitioning + `list_keys`), `SearchProvider`, `FilterListSource` (rule loader), **`RequestFilter`** (per-URL `should_block(&Url) -> Option<String>`; separated from `FilterListSource` because `HttpClient` depends only on per-URL decisions and is unaware of rule format), `EncodingDetector`, **`EventSink`** (`fn emit(&self, event: &Event)`, `Send + Sync`, sink accepts `&Event` without `clone`; no "silent default" — consumers hold `Option<Arc<dyn EventSink>>` and `None` means "nobody needs events"), **`DnsResolver`** (`fn resolve(&self, hostname: &str, port: u16) -> Result<Vec<SocketAddr>>`; implementations: `lumen-network::SystemDnsResolver` over `to_socket_addrs`, `lumen-storage::CachedDnsResolver` with TTL cache over `DnsCache`, `lumen-network::DohResolver` (RFC 8484, DNS-over-HTTPS), `lumen-network::DotResolver` (RFC 7858, DNS-over-TLS)), **`HstsEnforcement`** (`is_https_only(host, now_unix) -> bool`, `record_sts(host, max_age, include_subdomains, preload, now_unix)`; no `Result`, fail-open; implementation in `lumen-storage::hsts::HstsStore`, consumer — `lumen-network::HttpClient::with_hsts(...)` for http→https upgrade and persisting `Strict-Transport-Security` policy from responses per RFC 6797), **`HttpCredentialProvider`** (`credentials(&HttpAuthChallenge) -> Option<HttpCredentials>`; HTTP Basic + Digest on 401, provider looks up creds by `(origin, realm, scheme)`; implementation — `lumen-network::StaticCredentialProvider` for tests / curl-style `-u`, UI-popup and keyring — future P4 / platform layer tasks).
- **Sprint 0 P3 trait-anchors (lumen-plan.md §"Sprint 0 — Contracts"):** 7 trait points under provisional accelerators from §5, each with a `Null*` stub implementation ("not supported") — allows consumers to code against the trait before the real provisional crate is connected. `UnicodeProvider` (UAX #14 line-break / #29 grapheme+word segmentation / #9 bidi runs — under `icu4x.segmenter`+`icu4x.linebreak`; `NullUnicodeProvider` returns empty vectors). `IdnaProvider` (UTS #46 `to_ascii`/`to_unicode` — under `idna` crate; `NullIdnaProvider::to_ascii` returns None, consumer falls back to own Punycode from `lumen_core::idn`). `PublicSuffixList` (`public_suffix`/`registrable_domain`/`is_public_suffix` — under `publicsuffix` crate, P3 item 2B; `NullPublicSuffixList` — all None/false, RFC 6265 §5.3 step 5 gives strict-host fallback). `ContentDecoder` (HTTP `Content-Encoding`: `encoding()` + `decode(input) -> Result<Vec<u8>>`; per-codec implementations — `gzip`/`deflate` own via DEFLATE from `lumen-image`, `br`/`zstd` — `brotli-decompressor`/`ruzstd` provisional). **`UnsupportedContentDecoder { encoding: &'static str }`** — stub, `decode` always returns `Error::Other("content-encoding 'X' not supported")` (not identity passthrough — that's not a safe fallback). `FontFormat` (`format_name`/`can_decode`/`decode_to_sfnt` — under `woff2` crate, P2 Phase 2 WebFonts; `NullFontFormat` rejects everything). `SpellChecker` (`check`/`suggest`/`locale` — under `hunspell-rs`/`spellbook`, Phase 3; `NullSpellChecker::check` always true so UI doesn't underline everything). `HyphenationProvider` (`hyphenate(word, locale)`/`locales` — under `hyphenation` with TeX dictionaries; `NullHyphenationProvider` returns empty vectors). All Null-stubs are dyn-safe + Send+Sync, covered by unit tests on dyn-cast/return-values.
- **`JsRuntime` (lumen-core::ext)** — JS runtime: `eval`/`set_global`/`get_global`/`call_function` via `JsValue` (JSON-compatible types: Null/Undefined/Bool/Number/String/Array/Object with sorted-by-key entries). `JsError { Parse(String), Runtime(String), NotImplemented }` + `JsResult<T>`. `NullJsRuntime` — all methods return `JsError::NotImplemented`; first real runtime — QuickJS via `rquickjs` (Phase 2-3, P3 item 1B). Trait is dyn-safe + Send+Sync.
- Comments document remaining future trait points: `WindowingBackend`, `RenderBackend`, `TlsBackend`, `Hasher`. Trait body will be added at first implementation. (`FontProvider` is implemented — see `lumen-font::SystemFontIndex`.)
- **`lumen_core::punycode::encode`** — RFC 3492 Punycode encode (bootstring base=36, tmin=1, tmax=26, skew=38, damp=700, initial_bias=72, initial_n=128). 8 unit tests on known IDN labels (example → e1afmkfd, рф → p1ai, президент → d1abbgf6aiiy, тест → e1aybc, рус → p1acf, CJK 你好 → 6qq79v).
- **`lumen_core::idn::domain_to_ascii`** — IDNA `ToASCII` (simplified subset): lowercase → split by `.` → label-by-label (ASCII passthrough, otherwise `xn--<punycode>`). Does not do NFC normalization and UTS #46 mapping (for Russian domains already in NFC — practically sufficient). 10 unit tests (including idempotency for already `xn--…`, mixed ASCII+IDN subdomains, trailing dot for FQDN, case normalization).
- **`Url` — structured type** (`scheme`, `host`, `port`, `path`, `query`, `fragment`, `serialized` cache). Parser per simplified RFC 3986 §3 within scope: http/https/file (require `//` authority), data: and unknown schemes (without authority). API: `scheme()`, `host()` (Unicode for address bar), `port()` (None if not specified), `effective_port()` (with defaults http=80/https=443), `path()`, `query()`, `fragment()`, `as_str()` (serialized round-trip), `host_ascii()` (Punycode for DNS/TLS/Host header — RFC 7230 §5.4, RFC 6066 §3; empty host in data: gives empty string without error), `path_and_query()` (for HTTP request line), `resolve(reference)` (simplified RFC 3986 §5.3: absolute / `//` protocol-relative / `/` absolute path / `#` fragment-only / `?` query-only / relative — without `.`/`..` normalization). userinfo (`user:pass@`) is discarded on parsing; ASCII case-insensitive scheme; invalid port → InvalidUrl. This type is now used by `lumen-network` (via `host_ascii`/`effective_port`/`path_and_query`/`resolve` for redirects) and `lumen-shell` (via `Url::parse(base).resolve(href)` for relative `<link href>`). No more ad-hoc URL parsing in consumers.
- **`lumen_core::hash`** — common module for protocol hashes. `sha256(&[u8]) -> [u8; 32]` per FIPS 180-4, `hex_lower(&[u8]) -> String` (lowercase, no prefixes/separators), `sha256_hex(&[u8]) -> String` (sha256 + hex). Not security-critical crypto (for KDF / TLS / X.509 — rustls, exception #3); SHA-256 as protocol hash: HTTP Digest auth response (RFC 7616 §3.4.3), Safe Browsing canonical-expression lookup, SRI digest. MD5 (RFC 1321) lives locally in `lumen-network::auth` — needed only for legacy Digest, no reason to move it out. 6 unit tests (FIPS 180-4 vectors: empty / abc / 56-byte two-block + hex_lower padding/empty).
- 127 tests (24 url-parse/resolve/host_ascii + 8 Punycode + 10 IDN + 6 hash + ~12 JsRuntime/HttpCredentialProvider/JsValue + **13 Sprint 0 P3 stubs** (null_unicode/idna/public_suffix/font_format/spell_checker/hyphenation + unsupported_content_decoder + sprint0_send_sync) + remaining basics).
