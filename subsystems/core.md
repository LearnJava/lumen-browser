# lumen-core ✅ (foundation is stable)

- Types: `Error`, `Result<T>`, `Url`, `Event` (TabCreated/Closed, Navigation, PageLoaded, RequestStarted/Completed/Blocked), `TabId`, `Capability`, `CapabilityToken`, `Module` trait.
- Geometry: `Rect`, `Point`, `Size`.
- `lumen-core::ext` — extension trait points: `NetworkTransport`, `StorageBackend` (with origin-partitioning + `list_keys`), `SearchProvider`, `FilterListSource` (rule loader), **`RequestFilter`** (per-URL `should_block(&Url) -> Option<String>`; separated from `FilterListSource` because `HttpClient` depends only on per-URL decisions and is unaware of rule format), `EncodingDetector`, **`EventSink`** (`fn emit(&self, event: &Event)`, `Send + Sync`, sink accepts `&Event` without `clone`; no "silent default" — consumers hold `Option<Arc<dyn EventSink>>` and `None` means "nobody needs events"), **`DnsResolver`** (`fn resolve(&self, hostname: &str, port: u16) -> Result<Vec<SocketAddr>>`; implementations: `lumen-network::SystemDnsResolver` over `to_socket_addrs`, `lumen-storage::CachedDnsResolver` with TTL cache over `DnsCache`, `lumen-network::DohResolver` (RFC 8484, DNS-over-HTTPS), `lumen-network::DotResolver` (RFC 7858, DNS-over-TLS)), **`HstsEnforcement`** (`is_https_only(host, now_unix) -> bool`, `record_sts(host, max_age, include_subdomains, preload, now_unix)`; no `Result`, fail-open; implementation in `lumen-storage::hsts::HstsStore`, consumer — `lumen-network::HttpClient::with_hsts(...)` for http→https upgrade and persisting `Strict-Transport-Security` policy from responses per RFC 6797), **`HttpCredentialProvider`** (`credentials(&HttpAuthChallenge) -> Option<HttpCredentials>`; HTTP Basic + Digest on 401, provider looks up creds by `(origin, realm, scheme)`; implementation — `lumen-network::StaticCredentialProvider` for tests / curl-style `-u`, UI-popup and keyring — future P4 / platform layer tasks), **`CredentialProvider`** (WebAuthn / passkeys backing `navigator.credentials`; `create(&WebAuthnCreateRequest) -> Result<WebAuthnCreateResponse, WebAuthnError>`, `get(&WebAuthnGetRequest) -> Result<WebAuthnGetResponse, WebAuthnError>`, `is_user_verifying_platform_authenticator_available()`; `WebAuthnError` maps to DOMException names; implementation — `lumen-network::VirtualAuthenticator` (software ES256 passkey store), installed process-globally via `lumen_js::set_credential_provider`; `lumen-js` references only the trait, keeping the graph acyclic like `JsFetchProvider`).
- **Sprint 0 P3 trait-anchors (lumen-plan.md §"Sprint 0 — Contracts"):** 7 trait points under provisional accelerators from §5, each with a `Null*` stub implementation ("not supported") — allows consumers to code against the trait before the real provisional crate is connected. `UnicodeProvider` (UAX #14 line-break / #29 grapheme+word segmentation / #9 bidi runs — under `icu4x.segmenter`+`icu4x.linebreak`; `NullUnicodeProvider` returns empty vectors). `IdnaProvider` (UTS #46 `to_ascii`/`to_unicode` — under `idna` crate; `NullIdnaProvider::to_ascii` returns None, consumer falls back to own Punycode from `lumen_core::idn`). `PublicSuffixList` (`public_suffix`/`registrable_domain`/`is_public_suffix` — under `publicsuffix` crate, P3 item 2B; `NullPublicSuffixList` — all None/false, RFC 6265 §5.3 step 5 gives strict-host fallback). `ContentDecoder` (HTTP `Content-Encoding`: `encoding()` + `decode(input) -> Result<Vec<u8>>`; per-codec implementations — `gzip`/`deflate` own via DEFLATE from `lumen-image`, `br`/`zstd` — `brotli-decompressor`/`ruzstd` provisional). **`UnsupportedContentDecoder { encoding: &'static str }`** — stub, `decode` always returns `Error::Other("content-encoding 'X' not supported")` (not identity passthrough — that's not a safe fallback). `FontFormat` (`format_name`/`can_decode`/`decode_to_sfnt` — under `woff2` crate, P2 Phase 2 WebFonts; `NullFontFormat` rejects everything). `SpellChecker` (`check`/`suggest`/`locale` — under `hunspell-rs`/`spellbook`, Phase 3; `NullSpellChecker::check` always true so UI doesn't underline everything). `HyphenationProvider` (`hyphenate(word, locale)`/`locales` — under `hyphenation` with TeX dictionaries; `NullHyphenationProvider` returns empty vectors). All Null-stubs are dyn-safe + Send+Sync, covered by unit tests on dyn-cast/return-values.
- **`JsRuntime` (lumen-core::ext)** — JS runtime: `eval`/`set_global`/`get_global`/`call_function` via `JsValue` (JSON-compatible types: Null/Undefined/Bool/Number/String/Array/Object with sorted-by-key entries). `JsError { Parse(String), Runtime(String), NotImplemented }` + `JsResult<T>`. `NullJsRuntime` — all methods return `JsError::NotImplemented`; first real runtime — QuickJS via `rquickjs` (Phase 2-3, P3 item 1B). Trait is dyn-safe + Send+Sync.
- **`JsFetchProvider` (lumen-core::ext)** — Fetch API bridge: `fetch_sync(url: &str, method: &str) -> Result<JsFetchResult>`. `JsFetchResult { status: u16, status_text: String, headers: Vec<(String,String)>, body: Vec<u8> }`. Implemented by `lumen-network::HttpClient`. Used by `lumen-js` via `install_dom_api(ctx, doc, Option<Arc<dyn JsFetchProvider>>)` — `None` makes `window.fetch()` reject immediately. Phase 0: GET/HEAD only.
- Comments document remaining future trait points: `WindowingBackend`, `RenderBackend`, `TlsBackend`, `Hasher`. Trait body will be added at first implementation. (`FontProvider` is implemented — see `lumen-font::SystemFontIndex`.)
- **`lumen_core::punycode::encode`** — RFC 3492 Punycode encode (bootstring base=36, tmin=1, tmax=26, skew=38, damp=700, initial_bias=72, initial_n=128). 8 unit tests on known IDN labels (example → e1afmkfd, рф → p1ai, президент → d1abbgf6aiiy, тест → e1aybc, рус → p1acf, CJK 你好 → 6qq79v).
- **`lumen_core::idn::domain_to_ascii`** — IDNA `ToASCII` (simplified subset): lowercase → split by `.` → label-by-label (ASCII passthrough, otherwise `xn--<punycode>`). Does not do NFC normalization and UTS #46 mapping (for Russian domains already in NFC — practically sufficient). 10 unit tests (including idempotency for already `xn--…`, mixed ASCII+IDN subdomains, trailing dot for FQDN, case normalization).
- **`Url` — structured type** (`scheme`, `host`, `port`, `path`, `query`, `fragment`, `serialized` cache). Parser per simplified RFC 3986 §3 within scope: http/https/file (require `//` authority), data: and unknown schemes (without authority). API: `scheme()`, `host()` (Unicode for address bar), `port()` (None if not specified), `effective_port()` (with defaults http=80/https=443), `path()`, `query()`, `fragment()`, `as_str()` (serialized round-trip), `host_ascii()` (Punycode for DNS/TLS/Host header — RFC 7230 §5.4, RFC 6066 §3; empty host in data: gives empty string without error), `path_and_query()` (for HTTP request line), `resolve(reference)` (simplified RFC 3986 §5.3: absolute / `//` protocol-relative / `/` absolute path / `#` fragment-only / `?` query-only / relative — without `.`/`..` normalization). userinfo (`user:pass@`) is discarded on parsing; ASCII case-insensitive scheme; invalid port → InvalidUrl. This type is now used by `lumen-network` (via `host_ascii`/`effective_port`/`path_and_query`/`resolve` for redirects) and `lumen-shell` (via `Url::parse(base).resolve(href)` for relative `<link href>`). No more ad-hoc URL parsing in consumers.
- **`lumen_core::hash`** — common module for protocol hashes. `sha256(&[u8]) -> [u8; 32]` per FIPS 180-4, `hex_lower(&[u8]) -> String`, `sha256_hex`. `sha1(&[u8]) -> [u8; 20]` per FIPS 180-3 — used for WebSocket Sec-WebSocket-Accept (RFC 6455). `base64_encode(&[u8]) -> String` per RFC 4648 §4. `ws_accept_key(client_key) -> String` — base64(sha1(key+GUID)). Not security-critical crypto (for KDF / TLS / X.509 — rustls). 12 unit tests (FIPS 180-4 vectors, FIPS 180-3 vectors, RFC 4648 base64 vectors, RFC 6455 §1.3 WS key example).
- **`lumen_core::json::JsonValue`** — implements `Display` (JSON serializer). RFC 8259 compliant: NaN/Infinity → null, control chars as `\uXXXX`, integer f64 without decimal point. Used by lumen-devtools CDP response serialization.
- **`WebStorage`** — shared in-memory Web Storage type (HTML Living Standard §8.1). Maintains insertion order via `Vec<String>` keys + `HashMap<String, String>` data. API: `len()`, `is_empty()`, `key(index)`, `get_item(key)`, `set_item(key, value)`, `remove_item(key)`, `clear()`. Placed in `lumen-core` (not `lumen-js`) so `lumen-shell` can store `Arc<Mutex<WebStorage>>` without enabling the `quickjs` feature.
- **`MemoryPressureSource` + `MemoryPressureLevel` (lumen-core::ext, ADR-008 §10H)** — OS memory pressure polling trait. `MemoryPressureLevel { Low, Medium, High }` (Ord). Platform impls in `lumen-core::memory_pressure`: `Win32MemoryPressureSource` (`GlobalMemoryStatusEx`, `dwMemoryLoad` %), `LinuxMemoryPressureSource` (`/proc/pressure/memory` PSI `some avg10`), `MacosMemoryPressureSource` (`host_statistics64(HOST_VM_INFO64)`, `(active+wire)/total` ratio). `NullMemoryPressureSource` always `Low`. All three thresholds: 75%→Medium, 90%→High. Caches subscribe via `on_memory_pressure(level)` on `EvictableCache` trait. 191 total tests in lumen-core.
- **`EvictableCache` + `CacheRegistry` (lumen-core::ext, ADR-008 §10D.3)** — unified cache management interface. `EvictableCache`: `on_memory_pressure`, `used_bytes`, `budget_bytes`, `clear`, `cache_name`. `CacheRegistry`: `register`, `broadcast_pressure`, `total_used_bytes`, `total_budget_bytes`, `clear_all`. Implemented by `GlyphAtlas`, `ImageDecodeCache`, `LayerCache`. P3 shell: create registry in shell, register all three caches, call `broadcast_pressure()` from `MemoryPressureSource` poll loop.
- **`CrashRecorder` (lumen-core::crash, lumen-plan.md §9.6)** — `EventSink` decorator with an in-memory ring buffer of the last N events (default 50, configurable; optional downstream `Arc<dyn EventSink>` so it slots into an existing delivery chain). `recent_events() -> Vec<String>` snapshots the buffer as dump lines (`[+<ms> ms] #<seq> <event-debug>`); `total_recorded()` counts all events including evicted. `install_panic_hook(dir)` installs a process-global `std::panic::set_hook` that, on panic, writes `format_crash_dump(events, panic_msg)` to `lumen-crash-<unix_ms>.log` via `write_crash_dump`, then chains the previous hook. Pure pieces (`format_crash_dump`, `write_crash_dump`) are unit-tested independently of an actual panic. Poisoned-mutex tolerant (a crash matters more than lock hygiene). Shell wiring (build recorder, put in `EventSink` chain, call `install_panic_hook` at startup) — handoff to P3.
