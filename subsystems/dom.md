# lumen-dom ‚úÖ (complete API for current scope)

- Arena-based: `Vec<Node>` + `NodeId(u32)`. No `Rc/RefCell`, no cycles. **Invariant enforced** via `#[deny(clippy::rc_buffer)]` + `INVARIANT(10B/ADR-008)` doc comment in lib.rs.
- Types: `Document`, `Node` (parent + children + data), `NodeData` (Document / Doctype / Element / Text / Comment), `QualName`, `Namespace` (HTML/SVG/MathML/Xml/XmlNs/XLink), `Attribute`.
- API: `create_element / create_text / create_comment / create_doctype`, `append_child`, `detach`, `get / get_mut`, `root`, `len`, **`base_href()`** (HTML5 ¬ß4.2.3 ‚Äî extraction of `<base href>` for resolving relative URLs), **`find_first_element(predicate)`** (pre-order search for first element satisfying predicate).
- `Display` impl prints tree with indentation ‚Äî for debugging.
- **InputType (HTML5 ¬ß4.10.5):** enum for parsing `<input type="...">` attribute. 22 standard types: `Text` (default), `Password`, `Email`, `Tel`, `Url`, `Number`, `Search`, `Date`, `DateTimeLocal`, `Time`, `Month`, `Week`, `Color`, `Range`, `Checkbox`, `Radio`, `File`, `Submit`, `Reset`, `Button`, `Image`, `Hidden` + `Other(String)` for forward-compat. `InputType::parse(s)` case-insensitive, trim whitespace. Method `Node::input_type() -> Option<InputType>` returns `None` for non-`input` elements, `Some(Text)` for `<input>` without `type` (HTML5 default). Helpers: `is_textual()` (text/password/email/tel/url/number/search ‚Äî fields with text content), `is_button_like()` (submit/reset/button/image). `as_str()` + `parse()` ‚Äî round-trip.
- **DocumentMode (HTML5 ¬ß13.2.6.2):** enum `NoQuirks` (default) / `Quirks` / `LimitedQuirks` + field `Document.mode` with `mode()`/`set_mode()` getter/setter. Tree builder sets mode when processing the first DOCTYPE token (see `lumen-html-parser::quirks_mode::detect_document_mode`); when DOCTYPE token is completely absent ‚Äî `Quirks` per ¬ß13.2.6.4.1. `Document::new()` gives `NoQuirks` for programmatic use and unit tests. Layout/cascade currently **do not read** `Document.mode` ‚Äî this is groundwork for dozens of legacy CSS switches (table sizing, body-background propagation, font-size in `<table>`, etc.).
- **target_id (CSS Selectors L4 ¬ß9.6, HTML LS ¬ß7.10.6):** field `Document.target_id: Option<String>` with `target() -> Option<&str>` / `set_target(Option<S>)` getter/setter. Stores the id from the URL fragment (without `#`) to which the `:target` pseudo-class is bound in the layout matcher. Empty string ‚Üí `None` (filtered in setter). Phase 0: value is not set automatically by the tree builder or DOM API ‚Äî this is shell integration (P3): when loading a URL, parse the fragment and call `set_target(...)` before style cascade.
- **Shadow DOM API (DOM Living Standard ¬ß4.2, ¬ß8.2):** `ShadowRootMode` enum (Open/Closed). `NodeData::ShadowRoot { mode }` variant ‚Äî infrastructure node, never rendered directly. `Document.shadow_roots: HashMap<NodeId, NodeId>` (private). `attach_shadow(host, mode) -> NodeId` ‚Äî allocates shadow root in arena, not a DOM child of host. `shadow_root_of(host) -> Option<NodeId>`, `is_shadow_host(id) -> bool`. `write_tree` prints `#shadow-root (open/closed)` as pseudo-child after light-tree children. **`FlatTree`** struct (`overrides: HashMap<NodeId, Vec<NodeId>>`): `children_of(doc, id) -> &[NodeId]` ‚Äî returns DOM children for non-shadowed nodes (zero-allocation fast path). **`build_flat_tree(doc) -> FlatTree`** ‚Äî fast path when no shadow hosts (empty overrides). For each shadow host: host's composed children ‚Üí shadow root's DOM children; `<slot name="‚Ä¶">` elements receive assigned light-tree nodes matching `slot="‚Ä¶"` attribute; unmatched light-tree children discarded; empty slot assignment ‚Üí no override (fallback content kept). `compute_slot_assignments` + `wire_slot_overrides` internal helpers. lumen-layout calls `build_flat_tree` once before `build_box` and passes `&FlatTree` through all recursive calls. Comment `// CSS: :host, ::slotted` marks P4 wiring point in `build_box`. 9 new tests.
- **T3 Hibernation Snapshot (ADR-008 ¬ß10B):** `Document::to_bytes() -> Result<Vec<u8>, DomSnapshotError>` and `Document::from_bytes(&[u8]) -> Result<Document, DomSnapshotError>` using bincode 1.x. All arena types derive `serde::Serialize + Deserialize`: `NodeId`, `Namespace`, `QualName`, `Attribute`, `ShadowRootMode`, `NodeData`, `Node`, `DocumentMode`, `DomPosition`, `Range`, `Selection`, `Document`. The blob is self-contained ‚Äî no pointer fixups needed because all node references are `NodeId(u32)` offsets. `DomSnapshotError::Encode / Decode` with `Display + std::error::Error`. Dependencies: `serde = "1"` + `bincode = "1.3.3"` (workspace, permanent).
- **GC integration ‚Äî JS wrapper reference tracking (Phase 2, ADR-008 ¬ß6+ / lumen-plan.md line 518):** `Document.js_refs: HashMap<NodeId, u32>` (not serialised ‚Äî `#[serde(skip)]` ‚Äî JS objects do not survive tab hibernation). `acquire_js_ref(NodeId) -> u32` / `release_js_ref(NodeId) -> u32` / `js_ref_count(NodeId) -> u32` ‚Äî reference counting for live JS wrappers. `is_detached(NodeId) -> bool` ‚Äî returns `true` if a node has no parent, is not root, shadow root, or template fragment. `dead_node_ids() -> Vec<NodeId>` ‚Äî computes the set of nodes that are both detached and unreferenced; P3 idle GC tick drains this list and drops external resources (image handles, layout boxes). Nodes are identified as collectable but the arena remains append-only in Phase 2 (free-list compaction is Phase 3). `detach()` docs updated with P3 integration note. **P3 integration:** call `acquire_js_ref` when allocating a rquickjs class instance for a DOM node; call `release_js_ref` in the QuickJS class finalizer. 11 new tests.
