# lumen-dom ✅ (complete API for current scope)

- Arena-based: `Vec<Node>` + `NodeId(u32)`. No `Rc/RefCell`, no cycles.
- Types: `Document`, `Node` (parent + children + data), `NodeData` (Document / Doctype / Element / Text / Comment), `QualName`, `Namespace` (HTML/SVG/MathML/Xml/XmlNs/XLink), `Attribute`.
- API: `create_element / create_text / create_comment / create_doctype`, `append_child`, `detach`, `get / get_mut`, `root`, `len`, **`base_href()`** (HTML5 §4.2.3 — extraction of `<base href>` for resolving relative URLs), **`find_first_element(predicate)`** (pre-order search for first element satisfying predicate).
- `Display` impl prints tree with indentation — for debugging.
- **InputType (HTML5 §4.10.5):** enum for parsing `<input type="...">` attribute. 22 standard types: `Text` (default), `Password`, `Email`, `Tel`, `Url`, `Number`, `Search`, `Date`, `DateTimeLocal`, `Time`, `Month`, `Week`, `Color`, `Range`, `Checkbox`, `Radio`, `File`, `Submit`, `Reset`, `Button`, `Image`, `Hidden` + `Other(String)` for forward-compat. `InputType::parse(s)` case-insensitive, trim whitespace. Method `Node::input_type() -> Option<InputType>` returns `None` for non-`input` elements, `Some(Text)` for `<input>` without `type` (HTML5 default). Helpers: `is_textual()` (text/password/email/tel/url/number/search — fields with text content), `is_button_like()` (submit/reset/button/image). `as_str()` + `parse()` — round-trip.
- **DocumentMode (HTML5 §13.2.6.2):** enum `NoQuirks` (default) / `Quirks` / `LimitedQuirks` + field `Document.mode` with `mode()`/`set_mode()` getter/setter. Tree builder sets mode when processing the first DOCTYPE token (see `lumen-html-parser::quirks_mode::detect_document_mode`); when DOCTYPE token is completely absent — `Quirks` per §13.2.6.4.1. `Document::new()` gives `NoQuirks` for programmatic use and unit tests. Layout/cascade currently **do not read** `Document.mode` — this is groundwork for dozens of legacy CSS switches (table sizing, body-background propagation, font-size in `<table>`, etc.).
- **target_id (CSS Selectors L4 §9.6, HTML LS §7.10.6):** field `Document.target_id: Option<String>` with `target() -> Option<&str>` / `set_target(Option<S>)` getter/setter. Stores the id from the URL fragment (without `#`) to which the `:target` pseudo-class is bound in the layout matcher. Empty string → `None` (filtered in setter). Phase 0: value is not set automatically by the tree builder or DOM API — this is shell integration (P3): when loading a URL, parse the fragment and call `set_target(...)` before style cascade.
- **Shadow DOM API (DOM Living Standard §4.2, §8.2):** `ShadowRootMode` enum (Open/Closed). `NodeData::ShadowRoot { mode }` variant — infrastructure node, never rendered directly. `Document.shadow_roots: HashMap<NodeId, NodeId>` (private). `attach_shadow(host, mode) -> NodeId` — allocates shadow root in arena, not a DOM child of host. `shadow_root_of(host) -> Option<NodeId>`, `is_shadow_host(id) -> bool`. `write_tree` prints `#shadow-root (open/closed)` as pseudo-child after light-tree children. **`FlatTree`** struct (`overrides: HashMap<NodeId, Vec<NodeId>>`): `children_of(doc, id) -> &[NodeId]` — returns DOM children for non-shadowed nodes (zero-allocation fast path). **`build_flat_tree(doc) -> FlatTree`** — fast path when no shadow hosts (empty overrides). For each shadow host: host's composed children → shadow root's DOM children; `<slot name="…">` elements receive assigned light-tree nodes matching `slot="…"` attribute; unmatched light-tree children discarded; empty slot assignment → no override (fallback content kept). `compute_slot_assignments` + `wire_slot_overrides` internal helpers. lumen-layout calls `build_flat_tree` once before `build_box` and passes `&FlatTree` through all recursive calls. Comment `// CSS: :host, ::slotted` marks P4 wiring point in `build_box`. 9 new tests.
- 57 tests: 7 basic + cyrillic invariants + 6 for base_href + 12 for InputType + **2 for DocumentMode** (default NoQuirks, set/get round-trip) + **3 for target_id** (default None, round-trip, empty→None) + **9 for Shadow DOM** (attach/query, node variant, mode display, flat tree zero-alloc fast path, host children = shadow root children, slot distribution, slot fallback, display format).
