# lumen-font 🟡 (TTF read + raster + name + system index)

- **Done:** parsers for tables head, maxp, cmap (format 4 + **format 12**), hhea, hmtx, loca, glyf, **`name` (HTML5 / OpenType §6 name)** — format 0 + format 1 (language tags), rank-based record selection (typographic family / family / full name; Windows Unicode preferred), UTF-16BE and MacRoman decode, `best_family()` helper. Glyf handles simple glyphs (contours with on-curve / off-curve, quadratic Bezier) and **composite glyphs** (references to other glyphs with 2×2 transform + offset). `Font::glyph_resolved` recursively expands composite to Simple with max-depth 8. Scanline rasterizer with 4×4 supersampling, even-odd fill, 1px padding. `Bitmap` with metrics left/top for placement. Bundled Inter v4.1 Regular. **cmap format 12** — Sequential Groups, full Unicode U+0000..U+10FFFF, including SMP (emoji U+1F600+, math U+1D400+, historical scripts); `CmapSubtable` enum with rank-based selection of best entry; binary search over groups O(log n).
- **Done (`SystemFontIndex` — implementation of `lumen_core::ext::FontProvider`):** lazy index of system fonts (Linux `/usr/share/fonts` + `~/.local/share/fonts` + `~/.fonts`; Windows `%WINDIR%\Fonts` + `%LOCALAPPDATA%\Microsoft\Windows\Fonts`; macOS `/System/Library/Fonts` + `/Library/Fonts` + `~/Library/Fonts`). Parses `name` and **`OS/2`** tables of each `.ttf`/`.otf`, stores in `HashMap<lowercase family, Vec<FaceRecord { family, weight, style, path }>>`. `OnceLock` — constructor does no I/O, scan starts at first `lookup_faces` / `lookup_family` / `list_families`. Falls back to subfamily-string parsing when `OS/2` is absent. `SystemFontIndex::with_dirs(...)` for tests and headless modes.
- **Done (OS/2 extended parser, OpenType §6 "OS/2 and Windows Metrics"):** `Font::os2() → Os2 { version, weight_class, fs_selection, sub/super-script x/y size+offset, strikeout size+position, typo ascender/descender/line-gap, win ascent/descent, x_height: Option<i16> (v2+), cap_height: Option<i16> (v2+) }`. v2+ fields parsed conditionally with graceful degrade on truncation. 19 unit tests.
- **Done (`post` parser — PostScript Information Table):** `Font::post() -> Result<Post, FontError>`. `Post { version, italic_angle, underline_position, underline_thickness, is_fixed_pitch }`. Phase 0 — header only (same for all versions); glyph name table (v2.0+) deferred. 12 unit tests.
- **Done (CSS Fonts L4 §5.2 matcher):** free function `lumen_core::ext::match_face(faces, weight, style) → Option<&FaceRecord>` and default-impl `FontProvider::pick_face(family, weight, style)`. Style filtered with priority strictly above weight; weight by §5.2 rules with special 400↔500 swap logic.
- **Done (`fvar` parser, OpenType Font Variations table — Variable Fonts L1 enabler):** `Font::fvar() -> Result<Fvar, FontError>`. `Fvar { axes: Vec<VariationAxis>, instances: Vec<NamedInstance> }`. `VariationAxis { tag, min, default, max, flags, axis_name_id }`. `NamedInstance { subfamily_name_id, flags, coordinates: Vec<f32>, post_script_name_id: Option<u16> }`. `axis.clamp(value)`, `axis.is_hidden()`. F16Dot16 (fixed-point 16.16) sign-extended big-endian. 21 unit tests.
- **Done (`avar` parser, OpenType Axis Variations Table — Variable Fonts L1 normalization):** `Font::avar() -> Result<Avar, FontError>`. `Avar::normalize(axis_index, coord)` applies piecewise-linear renormalization. Phase 0 — v1.0 only; v2.0 deferred. 13 unit tests.
- **Done (composite glyphs ARGS_ARE_XY_VALUES=0 point alignment):** `CompositeComponent` now holds `anchor: Anchor` (enum) instead of flat `offset: (f32, f32)`. `Anchor::Offset(dx, dy)` — modern path; `Anchor::Points { parent: u16, child: u16 }` — legacy TrueType alignment. 5 unit tests. **Phase 0 limitation:** composite flags USE_MY_METRICS, OVERLAP_COMPOUND, UNSCALED_COMPONENT_OFFSET, SCALED_COMPONENT_OFFSET not accounted for — added per request when a specific font needs it.
- **Done (`ItemVariationStore` parser + runtime — common enabler for HVAR/MVAR/gvar):** module `lumen-font::item_variation`. `RegionAxisCoordinates::scalar(coord)` — per-axis tent function. `VariationRegion::scalar(coords)` — product of per-axis scalars. `ItemVariationStore::evaluate(outer, inner, coords) -> Option<f32>` — sum-product deltas with regional scalars. 27 unit tests.
- **Done (`DeltaSetIndexMap` parser — HVAR/VVAR/MVAR glyph→indices mapper):** module `lumen-font::delta_set_index_map`. `DeltaSetIndex { outer: u16, inner: u16 }`. `DeltaSetIndexMap::get(index)` — O(1) lookup; per spec for index ≥ len() returns **last** entry. 13 unit tests.
- **Done (`HVAR` parser — Horizontal Metrics Variations, first real consumer of IVS+DSIM):** module `lumen-font::hvar`. `Hvar { store, advance_width_map, lsb_map, rsb_map }`. API lookup: `advance_width_index(glyph_id)` — via `advance_width_map.get()` if present, otherwise per-spec identity fallback `(outer=0, inner=glyph_id)`. Phase 0 — parser only, without `evaluate(coords)`. 12 unit tests.
- **Done (`VVAR` parser — Vertical Metrics Variations, mirror of HVAR for vertical text):** module `lumen-font::vvar`. 5 optional `DeltaSetIndexMap`s over shared `ItemVariationStore`. Phase 0 — parser only. 16 unit tests.
- **Done (`MVAR` parser — Metrics Variations for global metrics):** module `lumen-font::mvar`. `Mvar { store, records: Vec<ValueRecord> }`. Standard tags: `xhgt`, `cpht`, `undo`/`unds`, `strs`/`stro`, sub/super-script tags, horizontal/vertical metrics tags. `Mvar::lookup(tag)` — O(log n) binary search. 13 unit tests.
- **Done (`gvar` parser — Glyph Variations, last major variable-fonts runtime consumer):** module `lumen-font::gvar`. `Gvar<'a>` parses per-glyph variation data. `TupleVariation { peak, intermediate, points: PointNumbers, x_deltas, y_deltas }`. `PointNumbers::{All, Explicit(Vec<u16>)}`. Packed point numbers and packed deltas decoding. `tuple_axis_scalar(coord, peak, intermediate)` runtime — tent function. IUP (Interpolation of Untouched Points) and applying deltas to outline — separate task in rasterizer. 29 unit tests.
- **Done (Variable fonts runtime — IUP + apply_glyph_deltas, P2):** module `lumen-font::variation` — `apply_variations_to_simple_outline(contours, variations, coords) -> usize`. IUP per contour per axis: untouched-point `u` between `prev_touched` and `next_touched` (cyclically): linear interpolation. IUP applied **independently** by X and Y, always looking at **base orig** (not accumulated state). 19 unit tests.
- **Done (Variable fonts runtime — `Font::glyph_resolved_with_coords`, P2):** `Font::glyph_resolved_with_coords(glyph_id, coords) -> Result<Option<Glyph>, FontError>` — variable-fonts entry over `glyph_resolved` pipeline. For composite glyph: single internal method `glyph_resolved_inner(glyph_id, coords, gvar, depth)` traverses components recursively with same `coords`. `gvar` parsed once before descent and shared via `Option<&Gvar<'a>>`. **Phase 0 limitation:** component-level gvar variations (deltas on `CompositeComponent.anchor`) **not applied** — requires varying `Anchor::Offset(dx, dy)` through delta for component-index in TupleVariation, deferred. 8 integration tests.
- **Done (WOFF2 + WOFF1 decoder — CSS Fonts L4 §4.3):** module `lumen-font::woff2`. `is_woff2/is_woff1(data) -> bool`. `decode_woff2(data) -> Result<Vec<u8>, FontError>` — Brotli decompression (via `brotli-decompressor`), table directory parsing (base-128 variable-length), sfnt reconstruction. Transformed `glyf`/`loca` reconstructed per WOFF2 spec §5.2: flags from `flagStream` (1 byte/point), coordinate triplets from `glyphStream` (reference `with_sign` algorithm, 6 flag ranges), instructionLength as 255UInt16 from `glyphStream` (bytes from `instructionStream`), composite rebuilding, `overlapSimpleBitmap` support. Output is canonical TrueType (one flag byte/point, int16 deltas); `loca` always emitted long-form and `head.indexToLocFormat` patched to match; missing simple-glyph bbox computed from points. `decode_woff1(data) -> Result<Vec<u8>, FontError>` — per-table zlib decompression (via `zune-inflate`), sfnt reconstruction. `maybe_decode_font(data) -> Result<Option<Vec<u8>>, FontError>` — auto-detect and convert if WOFF/WOFF2; returns `None` for raw TTF/OTF. Integrated in `renderer.rs` `resolve_face_id`: WOFF/WOFF2 font files are decoded transparently before `Font::parse`. 17 unit tests + 2 real-font integration tests (BUG-160: the §5.2 reconstruction was previously wrong and every real WOFF2 failed with "unexpected end of font data").
- **Done (GSUB/GPOS shaping — U-2 stage 1, Latin/Cyrillic ligatures + kerning):** modules `lumen-font::{otlayout, gsub, gpos, shape}`. `Shaper::new(&font)` parses `GSUB`/`GPOS` and pre-selects active lookups; `Shaper::shape(glyph_ids, hmtx) -> Vec<ShapedGlyph>` runs GSUB then seeds base advances then GPOS. `otlayout` is the shared layer: `LayoutTable` (header + script→langsys→feature→lookup navigation via `enabled_lookups(tags)`, default-script policy DFLT→latn→cyrl), `Coverage` (fmt 1/2), `ClassDef` (fmt 1/2), `ValueRecord`/`read_value_record`, `resolve_extension`. **GSUB:** Type 1 single (fmt 1/2), Type 4 ligature (fmt 1, greedy, ligature cluster = min component), Type 7 extension; default features `liga`/`clig`/`calt`/`rlig`/`ccmp`. **GPOS:** Type 1 single (fmt 1/2), Type 2 pair (fmt 1 glyph-pairs + fmt 2 class-pairs), Type 9 extension; feature `kern`. Adjustments accumulate `x_advance`/`x_offset`/`y_offset` in font units. Wired into `lumen-paint::cpu_raster::rasterize_text` (CPU snapshot / `--screenshot` path; shapes per tab-delimited segment). Validated on bundled Inter: GPOS kerns AV/To/Wa/Type; GSUB ligates `->`→arrow (Inter ships no `liga` — its f-handling is type-6 contextual, out of scope). 14 unit tests + 7 integration tests (`inter_shaping.rs`). **Not** wired into the live femtovg window (femtovg shapes via its own `fill_text`) nor per-char layout measurement.
- **Done (CFF outlines — U-2 stage 2, PostScript-OpenType `.otf`):** module `lumen-font::cff`. `Cff::parse(data) -> Result<Cff, FontError>` parses the `CFF ` table: header → Name/TopDICT/String/GlobalSubr INDEX → CharStrings INDEX, Private DICT (default/nominal width parsed only for operand parity) + local Subr INDEX. `Cff::glyph(gid) -> Result<Option<Glyph>, FontError>` interprets the Type 2 charstring into an `Outline::Simple` (cubics flattened to `CUBIC_STEPS=10` on-curve segments so the quadratic rasterizer is reused unchanged; bbox computed from flattened points). Interpreter covers all path operators (rmoveto/hmoveto/vmoveto, rlineto, hlineto/vlineto, rrcurveto, rcurveline/rlinecurve, vvcurveto/hhcurveto, vhcurveto/hvcurveto), hint operators (hstem/vstem/-hm counted to skip `hintmask`/`cntrmask` mask bytes), `callsubr`/`callgsubr`/`return` with the standard subr bias (107/1131/32768, depth-capped at 10), the four flex ops (`flex`/`hflex`/`hflex1`/`flex1`), the optional leading-width operand, and `endchar` incl. `seac`-style accented composites. **CID-keyed CFF:** `ROS`/`FDArray`/`FDSelect` (formats 0+3) → per-FD local subrs. Routed transparently through `Font::cff()`/`has_cff()` + an early branch in `Font::glyph_resolved`/`glyph_resolved_with_coords`, so the CPU rasterizer, the wgpu renderer (`renderer.rs`), and Canvas 2D all draw `.otf` text without changes. 16 unit tests on a hand-built spec-valid CFF table (INDEX/DICT/real/bias/square/curve/subr/hintmask/width/hv-lineto/FDSelect); dev-validated against real system OTFs (AdobeClean non-CID, MyriadCAD CID-keyed). Deferred: CFF2 (variable PostScript outlines), charstring arithmetic/storage operators, CFF FontMatrix scaling other than 1/1000 em.
- **Deferred:** hinting (TT instructions), CFF2 (variable PostScript outlines), GSUB/GPOS contextual lookups (GSUB 5/6, GPOS 7/8), mark positioning (GPOS 3–6), complex scripts (Arabic/Indic), LookupFlag mark filtering, shaping in the femtovg live path + layout measurement, integration with CSS `font-variation-settings` cascade, component-level gvar deltas in composite-glyph anchors, color glyphs (COLR/CPAL, sbix), bitmap strikes (EBDT/EBLC), composite flags USE_MY_METRICS / OVERLAP_COMPOUND / SCALED_COMPONENT_OFFSET.

## Gotchas

- **Composite glyphs** are supported via `Font::glyph_resolved` (max recursion depth 8). Both alignment variants: `Anchor::Offset(dx, dy)` (modern, ARGS_ARE_XY_VALUES=1) and `Anchor::Points { parent, child }` (legacy TrueType pre-1996). Cyrillic uppercase А/В/Е/К/М/Н/О/Р/С/Т/Х (composite via Latin equivalents in Inter) render correctly. Always call `glyph_resolved`, not `glyph` directly.
- **Multi-size + variation-aware glyph atlas.** Glyphs rasterized at a bin-matched size (`SIZE_BINS = [8, 12, 16, 20, 24, 32, 48, 64]`), display scale = `font_size / size_bin`. Exact bin match → no scaling, crisp text. Cache key: `AtlasKey { face_id, glyph_id, size_bin, coords_hash }` where `coords_hash` is from normalized variation coords. Empty coords → hash=0 (backward-compatible with non-VF pages and snapshot tests).
- **Font fallback / matcher fully implemented** (CSS Fonts L4 §3.1+§5.2+§5.3). Per-char codepoint cascade: if primary face lacks a glyph, iterate other loaded faces; if none have it, render `.notdef`. **Phase 0 limitation:** cascade only covers already-loaded faces — CJK/emoji won't render unless explicitly listed in CSS. Generic families (`serif`/`sans-serif`/`monospace`) currently skipped during face resolution.
