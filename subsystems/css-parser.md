# lumen-css-parser 🟡 (complete CSS3 selector set)

- **Done:** `selector_list { decl_list }` parsing. Selectors Level 3: simple selectors (`Type`, `Class`, `Id`, `Universal`, `Attribute`, `PseudoClass`, `PseudoElement`), compound (`p.foo#bar:first-child`), complex with combinators (descendant ` `, child `>`, next-sibling `+`, later-sibling `~`). Attribute operators: `=`, `~=`, `|=` (for `lang`), `^=`, `$=`, `*=`. **Case-insensitive flag `[attr=val i]`** (CSS Selectors L4 §6.3.6) — after value recognizes `i` / `I` (ASCII case-insensitive) or `s` / `S` (explicitly case-sensitive, default); stored in `AttrSelector.case_insensitive: bool`; applicable to all six operators. **Structural pseudo:** `:first-child`, `:last-child`, `:only-child`, `:empty`, `:root`, `:first-of-type`, `:last-of-type`, `:only-of-type`, **`:placeholder-shown`** (CSS Selectors L4 §15.1 — typed variant, matcher in layout). **Form-state pseudo (CSS Selectors L4 §14.2 / §15.4 / §15.5; HTML5 §4.10.3 / §4.10.19 / §4.16.4):** `:required`, `:optional`, `:read-only`, `:read-write`, `:disabled`, `:enabled` — typed variants in `PseudoClass` enum, matcher in layout. All pure attribute-based, without runtime form-state. **UI-state pseudo (CSS Selectors L4 §10.1 / §10.2 / §10.4; HTML5 §4.16.3):** `:checked`, `:indeterminate`, `:default` — variants of same enum (matchers in layout). Also pure DOM-based: `:checked` ↔ `checked` attribute on checkbox/radio or `selected` attribute on `<option>`; `:indeterminate` for radio in group by `name` without single-checked sibling (+ `<progress>` without `value`); `:default` for checked-checkbox/radio, selected-option and first submit-button inside `<form>` ancestor. **`:lang(<language-tag>#)`** (CSS Selectors L4 §11) — functional variant `Lang(Vec<String>)`, parser accepts comma-list BCP 47 language tags (ident-like: ASCII alphanumeric + `-`), normalizes to lowercase; empty list → `Unsupported(name)`. Matcher in layout — RFC 4647 §3.3.1 basic filtering against element content-language. **`:dir(ltr|rtl)`** (CSS Selectors L4 §13.2) — functional variant `Dir(DirArg)` with keyword argument `ltr` / `rtl` (ASCII case-insensitive); `auto` and other invalid values → `Unsupported(name)`. Matcher in layout — element directionality via `dir` attribute with inherited fallback (HTML5 §3.2.6.1); `dir="auto"` in Phase 0 treated as `ltr`. **Link pseudo (CSS Selectors L4 §6.2):** `:link` / `:visited` / `:any-link` — variants of same enum (matchers in layout). Pure DOM-based in Phase 0 without runtime visited-state: `:any-link` ↔ `<a>` / `<area>` / `<link>` with `href`; `:link` equivalent to `:any-link` (all links considered unvisited); `:visited` always `false` (safe default per privacy-by-default — no history leaks via styling). **`:scope`** (CSS Selectors L4 §4.2) — variant `PseudoClass::Scope`; root of selector matching context. In author-CSS without runtime querySelector matches document root element (equivalent to `:root`). Real difference will appear when integrating with DOM querySelector API (P3 + JS-runtime). **`:target`** (CSS Selectors L4 §9.6) — variant `PseudoClass::Target`; matches element with `id` attribute equal to URL fragment. Comparison case-sensitive (HTML LS §3.2.6). Not functional pseudo: `:target(x)` discarded to `Unsupported`. Matcher in layout reads `Document::target() -> Option<&str>`; shell integration (set target_id from URL fragment on load) — separate P3 task, until then `:target` silently returns `false` (privacy-safe default). **`:target-within`** (CSS Selectors L4 §9.7) — variant `PseudoClass::TargetWithin`; matches element that is itself `:target` or has descendant with `:target`. Equivalent to `:has(:target), :target` — separate matcher (`matches_target_within` via `any_descendant`), without dependency on `:has`. Not functional. Phase 0 same limitation as `:target` (short-circuit when `Document::target() == None` — without fragment we don't traverse subtree). **`:defined`** (CSS Selectors L4 §6.4.1, HTML LS §4.13.5) — variant `PseudoClass::Defined`; matches built-in HTML/SVG/MathML elements and registered custom elements. Not functional pseudo: `:defined(x)` → `Unsupported`. Matcher in layout: custom element name per HTML LS §4.13.2 must have ASCII `-` — in Phase 0 without `CustomElementRegistry` matcher uses this as approximation (`!name.contains('-')` = defined). When P3 raises registry, check becomes `built-in || registry.has(name)`. Useful for FOUC-protection idiom `:not(:defined) { display: none }` (hides unloaded custom-elements). **Open-state pseudo (Fullscreen API §4.2, CSS Selectors L4 §16.5.2, HTML LS §6.12.2):** `:fullscreen` / `:modal` / `:popover-open` — variants `PseudoClass::Fullscreen` / `Modal` / `PopoverOpen`; matchers in layout always return `false` in Phase 0. Runtime-only state, Phase 0 safe default = `false`. Variants provide correct specificity-count and `:not(:fullscreen)` / `:not(:modal)` / `:not(:popover-open)` idioms (always match in Phase 0) — groundwork for Phase 2/3 runtime. **Time-dimensional pseudo (CSS Selectors L4 §11.4):** `:current` / `:past` / `:future` — variants `PseudoClass::Current` / `Past` / `Future`; matchers in layout always return `false` in Phase 0. All match momentum in timed-text stream (e.g., WebVTT cue rendering during video/audio playback). Runtime-only state. Phase 0 without timed-text runtime — privacy-/UX-safe default = `false`. Variants provide correct specificity-count — groundwork for Phase 3+ WebVTT/captions runtime. **Functional pseudo:** `:nth-child(an+b [of <selector-list>])`, `:nth-last-child(an+b [of <selector-list>])`, `:nth-of-type(an+b)`, `:nth-last-of-type(an+b)` with keywords `odd` / `even` (stored as `NthSpec { a, b }`, `.matches(index)` solves equation `i = a*n + b` at `n ≥ 0`). `:nth-child` / `:nth-last-child` support optional **`of <selector-list>`** (CSS Selectors L4 §6.6.5.1) — variant `NthChild(NthSpec, Option<Vec<ComplexSelector>>)`; filter sibling-pool before nth-indexing. Specificity (§17): of-list adds max specificity of its selectors on top of 1. `:nth-of-type` per spec does not accept `of`. **`:not(selector-list)`** (CSS Selectors L4 §5.4) — negation of selector-list; inside complex selectors (combinators `>`, `+`, `~`, descendant) and nested `:not(:not(...))` are allowed; matches if none of the selectors in list match the element. Empty `:not()` → `Unsupported`. **CSS4 `:is(selector-list)` and `:where(selector-list)`** — match if any selector in list matches; inside any complex selectors allowed. Empty list `:is()` / `:where()` → `Unsupported(name)`. Interactive (`:hover`, `:focus`, …) stored as `Unsupported(name)` and at matching always return false. Pseudo-elements `::name` parsed as separate node, never match. **Specificity** per CSS Selectors L4 §16-§17: `:not(...)` and `:is(...)` don't count themselves, contribute max specificity of their selector-list. `:where(...)` always 0. Other pseudo-classes count as class. Declarations — as string pairs. Lenient recovery, comments `/* */`, skip `@`-rules. Parser `parse_complex_selector` also breaks on `)`, to work correctly inside functional pseudo.
- **`!important` flag (CSS Cascade L4 §8.1):** parser `extract_important` separates `!important` from value (with optional whitespace between `!` and word, ASCII case-insensitive). Does not touch `!important` inside string literals: `content: "!important"` remains value=`"!important"`, important=false. Stored in `Declaration.important: bool`.
- **CSS-wide keywords (CSS Cascade L4 §7):** `inherit` / `initial` / `unset` / `revert` parsed as ordinary value-strings in css-parser (without special grammar); recognized at `apply_declaration` stage via `parse_css_wide_keyword(value) -> Option<CssWideKeyword>` (ASCII case-insensitive, trim whitespace). Semantics: `Inherit` → computed value of parent; `Initial` → initial value from spec (taken from `ComputedStyle::root()`); `Unset` → inherit for inherited-properties, initial for non-inherited; `Revert` in Phase 0 ≡ `Unset` (UA / User origins not clearly separated). Applied in `apply_css_wide_keyword(style, prop, kw, inherited)` — separate function with per-property branches for all ~40 supported properties (including shorthands: margin / padding / border / border-* / border-radius / outline / overflow + counter-reset/-increment). Unknown names — silent no-op (same as for regular declarations). `inherited: &ComputedStyle` parameter is threaded through `apply_declaration` from `compute_style`. `font-size: inherit/initial/unset/revert` handled separately in `apply_font_size` pre-pass — before main-pass, because em-basis depends on font-size.
- **`display: flex / inline-flex / grid / inline-grid`:** parsing and storage in `Display` enum (CSS Display L3 §2; CSS Flexbox L1 §3; CSS Grid L1). **Phase 0: parsing+storage only, without real flex/grid algorithm** — `Flex` / `Grid` treated as Block for outer-display (vertical layout); `InlineFlex` / `InlineGrid` — as Inline (placed in `InlineRun` of neighboring text-nodes). Real flex algorithm (main-axis / cross-axis / `justify-content` / `align-items` / `gap` / `flex-grow|shrink|basis`) — separate task in Phase 1+ (requires significant rewrite of `lay_out` function). Snapshot writes `display=flex` / `display=inline-flex` / `display=grid` / `display=inline-grid` if value differs from Block.
- **@media queries (Media Queries L4):** `Stylesheet.media_rules: Vec<MediaRule>` stores `@media` blocks alongside regular `rules`. `MediaRule { query: MediaQuery, rules: Vec<Rule> }`. `MediaQuery.clauses: Vec<MediaQueryClause>` — OR-list (comma-separated). `MediaQueryClause { negated: bool, conditions: Vec<MediaCondition> }` — AND-list inside one clause with optional leading `not`/`only` modifier. `MediaCondition` enum: `MediaType("screen"/"print"/"all"/...)`, `Feature(MediaFeature)`, `Unsupported` (forward-compat for unknown feature-names). `MediaFeature` Phase 0: `MinWidth(f32)`, `MaxWidth(f32)`, `MinHeight(f32)`, `MaxHeight(f32)`, `Orientation(Portrait/Landscape)`, `PrefersColorScheme(Light/Dark)`. In layout cascade: after regular `sheet.rules` iterates `sheet.media_rules`; for each `media.query.matches(ctx)` rules are added with `rule_idx` continuing from regular rules — gives correct "source-position later wins" rule for @media-rules, but **with simplification**: all @media treated as coming after regular rules, not strictly by source-position (Phase 0 limitation). 23 tests (+11).
- **CSS Masking L1 §4 — `mask-image` / `-repeat` / `-size`:** parallel properties to background-*. `mask_image: BackgroundImage` (None/Url/Gradient — type reused), `mask_repeat: BackgroundRepeat`, `mask_size: BackgroundSize`. Not inherited. Phase 0: parsing only — real masking in paint pipeline deferred.
- **CSS Scrollbars 1 + CSS Overflow L3 — `scrollbar-width` / `-color` / `-gutter`:** `ScrollbarWidth` enum (Auto default/Thin/None) — **inherited**. `scrollbar_color: Option<(Color, Color)>` (thumb + track) — `auto` = None; parser of two colors with balancing `(`/`)` for `rgb(...)`-functions — **inherited**. `ScrollbarGutter` enum (Auto default/Stable/StableBothEdges — `stable both-edges` parsed as two-token form) — not inherited. Phase 0: parsing only — real scrollbar styling in paint pipeline deferred.
- **CSS Transforms L1 §6 — `transform-origin` + `perspective`:** `transform_origin: (f32, f32, f32)` (x, y, z in px; default `(0, 0, 0)`; not inherited; keywords center/top/etc. deferred — only length values for now). `perspective: Option<f32>` (None or px > 0; perspective=0 treated as None). Phase 0: parsing.
- **CSS Lists L3 §2 — `list-style-type` / `-position` / `-image` + `list-style` shorthand:** `ListStyleType` enum (None/Disc default/Circle/Square/Decimal/DecimalLeadingZero/LowerRoman/UpperRoman/LowerAlpha/UpperAlpha/LowerGreek — last with aliases `lower-latin`/`upper-latin`). `ListStylePosition` enum (Outside default/Inside). `list_style_image: Option<String>` (URL from `url(...)` or None for `none`). All three **inherited**. Shorthand `list-style: <type> <position> <image>` — tokens recognized position-independently; `none` treated first as type, then image if repeated. Real rendering of markers — separate task.
- **CSS Transitions L1 §3 — `transition-property` / `-duration` / `-delay` / `-timing-function` + `transition` shorthand:** all four not inherited. `transition` shorthand parsing implemented. All consumer plumbing: P1 item 3A — Web Animations interpolation, P2 item 3B — compositor offload for `transform`/`opacity`, P3 — scheduling in rendering steps stage. 17 unit tests. Phase 0: parsing only — runtime for property interpolation — separate animation pass (P1 item 3A).
- **CSS Easing L1 §2 — `TimingFunction` enum:** `Linear` / `CubicBezier(x1, y1, x2, y2)` / `Steps(n, StepPosition)`. Default = `ease` ≡ `CubicBezier(0.25, 0.1, 0.25, 1.0)`. Keyword shortcuts `ease`/`ease-in`/`ease-out`/`ease-in-out` normalized to `CubicBezier` variants per spec-table coefficients. `step-start` ≡ `Steps(1, JumpStart)`, `step-end` ≡ `Steps(1, JumpEnd)`. Parser requires x1, x2 ∈ [0, 1] (spec invariant); y1, y2 — unbounded (overshoot easings). `StepPosition` enum (JumpStart alias `start` / `JumpEnd` default alias `end` / `JumpNone` requires n ≥ 2 / `JumpBoth`). `TimingFunction::parse` for single value, `::parse_list` for comma-separated. Used in `transition_timing_functions` and `animation_timing_functions`.

No fetch/loading yet (shell task). 6 new tests added (229 total).
