# Оркестратор задач Lumen

Автоматический запуск Claude Code сессий для разработчиков P1–P4.
Каждая задача выполняется в отдельной сессии с чистым контекстом.
После завершения задачи автоматически запускается следующая.

## Требования

- Windows 10, PowerShell 5.1+
- Claude Code CLI (`claude`) в PATH
- Файлы `STATUS-P1.md` ... `STATUS-P4.md` в корне проекта

## Скрипты

| Скрипт | Назначение |
|---|---|
| `run-dev.ps1` | Цикл задач для одного разработчика |
| `run-all.ps1` | Запуск нескольких разработчиков параллельно (каждый в своём окне) |
| `stop-dev.ps1` | Остановка разработчика после текущей задачи |

## Запуск

```powershell
# Один разработчик
.\scripts\run-dev.ps1 -Developer P1

# С лимитом — максимум 3 задачи, потом стоп
.\scripts\run-dev.ps1 -Developer P2 -MaxTasks 3

# Все четверо параллельно (каждый в отдельном окне PowerShell)
.\scripts\run-all.ps1

# Только выбранные
.\scripts\run-all.ps1 -Developers P1,P3

# Все, но не больше 5 задач на каждого
.\scripts\run-all.ps1 -MaxTasks 5
```

## Остановка

```powershell
# Остановить P1 после текущей задачи (мягкая остановка)
.\scripts\stop-dev.ps1 -Developer P1

# Остановить всех
.\scripts\stop-dev.ps1 -All

# Немедленная остановка — Ctrl+C в окне разработчика
```

### Как работает мягкая остановка

1. `stop-dev.ps1` создаёт файл `.stop-P1` в корне проекта
2. `run-dev.ps1` проверяет этот файл **между задачами**
3. Текущая задача всегда доработает до конца
4. После завершения цикл увидит стоп-файл, удалит его и остановится

## Жизненный цикл задачи

```
run-dev.ps1 запускается
    │
    ├─► Проверка .stop-P1 → найден? → СТОП
    │
    ├─► Проверка STATUS-P1.md → нет задач? → СТОП
    │
    ├─► Запуск: claude -p "..." --dangerously-skip-permissions
    │       Claude читает STATUS-P1.md
    │       Берёт задачу (In progress или первую из Next)
    │       Работает...
    │       Вызывает /lumen-task-finish
    │       Сессия завершается
    │
    ├─► Проверка кода возврата
    │       0 → задача выполнена
    │       ≠0 → пауза 10 сек, повтор
    │
    └─► Следующая итерация цикла
```

## Частые вопросы

**Что если Claude упадёт посередине задачи?**
Скрипт получит ненулевой код возврата, подождёт 10 секунд и запустит новую сессию.
Новая сессия прочитает STATUS-файл и продолжит незавершённую задачу (поле "In progress").

**Что если STATUS-файл пустой?**
Цикл завершится с сообщением "Нет задач".

**Можно ли запустить два экземпляра для одного разработчика?**
Нет, будут конфликты — оба возьмут одну задачу. Один разработчик = одно окно.

**Как посмотреть что делает разработчик прямо сейчас?**
Вывод Claude виден в окне PowerShell в реальном времени.
