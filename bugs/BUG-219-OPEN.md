# BUG-219

**Статус:** OPEN
**Компонент:** image/paint
**Тест:** TEST-18 (остаток 2.11% после BUG-180)

## Описание

Тонкое расхождение пикселей при downscale растровых картинок относительно Edge.
После фикса геометрии (BUG-180, «image bottom gap») TEST-18 даёт 2.11% — diff
размазан тонким AA по всем фото (детальные мозги, JPEG `ai_wikipedia`); ни одна
ячейка не разъезжается структурно (порог `>60` всего 0.43%).

## Корень (гипотеза)

Ядро ресемплинга у Lumen (area-averaging, см. BUG-032/BUG-077) не совпадает
по точным значениям с downscale-фильтром Edge/Blink (Lanczos-подобный). Это
пиксель-парность, аналог text-AA (rule 3), а не геометрический/логический дефект.

## Как чинить

Сравнить выход `resize`/area-avg на фиксированном входе с Edge; при необходимости
перейти на более близкий к браузеру kernel (bilinear+prefilter / Lanczos).
Отдельная задача по качеству ресемплинга, не блокирует.

## Статус теста

TEST-18 → `KNOWN_DEBTORS` (baseline 2.11%). Цель — снять при достижении 0.5%.
