# BUG-165

**Статус:** FIXED 2026-06-15
**Компонент:** layout
**Файл:** `crates/engine/layout/src/box_tree.rs` (`lay_out_flex`, align-content loop ~7090)

## Описание

flex `align-content` (TEST-65: 16.40%) сдвигал flex-строку по cross-оси, прибавляя
смещение только к `children[i].rect.y`, но НЕ к поддереву item-а. Потомки flex-item-а
уже разложены в абсолютных координатах на момент применения align-content, поэтому при
сдвиге строки они оставались на месте, отставая от своего бокса на `offset`.

Наиболее заметно, когда внешний flex-wrap контейнер с дефолтным align-content (`stretch`)
держал flex-item-ы, которые сами были контейнерами: grandparent `__f` растягивал строки
контейнеров (`offset = per * line_index`), двигал боксы контейнеров, но их items
оставались на не-растянутой позиции → визуально вылезали выше своих контейнеров.

Пример из TEST-65 (до фикса): контейнер `space-between` нарисован на y=182.5, а его items
на y=168 (выше контейнера); контейнер `stretch` на y=545.5, items на y=484.

## Корень

`box_tree.rs` align-content loop:

```rust
children[i].rect.y += offset;   // двигает только бокс item-а
```

Сдвигался только собственный `rect.y`, без рекурсии по потомкам.

## Фикс

```rust
shift_y_box(&mut children[i], offset);  // рекурсивный сдвиг всего поддерева по Y
```

`shift_y_box` (уже существовал, `box_tree.rs:3783`) рекурсивно прибавляет `dy` к rect-ам
всего поддерева — тот же паттерн, что `shift_tree` для абсолютного позиционирования
(строка 6660).

## Регресс-тест

`flex_align_content_shifts_item_subtree` — single-line flex-end контейнер с item-ом,
содержащим вложенный блок; проверяет, что вложенный `#inner` следует за `#a` на y=120.

## Проверка

`--dump-layout graphic_tests/65-flex-align-content.html` после фикса: все items сидят
внутри своих контейнеров (center→центр, space-around/evenly→центр, stretch/space-between→верх),
совпадает с рендером Edge.

Визуальный прогон TEST-65 против Edge: **16.40% → 1.33%**. Боксы позиционно совпадают
с Edge во всех 7 контейнерах (на diff видны только тонкие 1px-контуры боксов, без залитых
прямоугольных пятен). Остаток 1.33% — текст (цифры 1–6 в каждом боксе + подписи `::before`
«center»/«space-between»/…) и edge-AA: глобальное расхождение font-AA с Edge (правило graphic
tests №3 «Ignore text»), не геометрия. Дальнейшее снижение требует font-parity, не относится
к этому багу.
