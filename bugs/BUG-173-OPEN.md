# BUG-173

**Статус:** OPEN
**Компонент:** paint
**Файл:** `crates/engine/paint/src/svg_path.rs`, `crates/engine/paint/src/backends/femtovg_backend.rs`

## Описание

Остаточная дельта SVG `<path>` vs Edge после фикса BUG-102 (advanced stroke
attributes). TEST-54: 2.30%, TEST-60: 1.41% (порог 0.5%) — оба заведены в
`KNOWN_DEBTORS` (`run.py`). Геометрия штриха теперь корректна (ширина, dash,
caps, joins, гладкие кривые), остаток складывается из четырёх независимых
причин, каждая требует отдельной более крупной работы:

1. **Triangle-soup AA-швы.** `DrawSvgPath` несёт плоский список треугольников;
   `femtovg_backend` заливает каждый треугольник отдельным закрытым сабпасом
   (`move_to/line_to/line_to/close` + `fill_path`). Антиалиасинг по общим
   внутренним рёбрам соседних треугольников не сокращается → видна слабая
   диагональ внутри каждого quad'а. Исправление: рисовать обводку/заливку
   выпуклыми полигонами (а не triangle soup) — меняет формат команды
   `DrawSvgPath` и три бэкенда (femtovg/wgpu/cpu).

2. **Stroke-edge AA.** Граница штрих↔фон антиалиасится иначе, чем у Edge; на
   тонких штрихах с большим периметром даёт ~1% (аналогично BUG-128
   font-parity). Практически неустранимо без репликации растеризатора Edge.

3. **Self-intersecting fill.** `tessellate_fill` использует ear-clipping, который
   не заполняет самопересекающиеся контуры (галстук-бабочка `X` в TEST-54
   остаётся незалитой, в Edge — залита nonzero). Нужен winding-/scanline-fill.

4. **Dash-on-curve.** Раскладка штрихов по flattened-кривой (круг/зигзаг в
   TEST-60) по фазе слегка расходится с Edge.

Все четыре — отдельные задачи. BUG-173 служит якорем для записей `KNOWN_DEBTORS`
'54'/'60'.
