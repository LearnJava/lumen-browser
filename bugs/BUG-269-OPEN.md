# BUG-269

**Статус:** OPEN
**Компонент:** layout (замещаемые элементы, aspect-ratio)
**Симптом:** `<img>` с атрибутом `width` (или CSS-шириной), но БЕЗ явной высоты рендерится с нулевой высотой — картинка невидима. Ожидание (CSS 2.1 §10.6.2 / CSS Images §5): высота считается из intrinsic aspect-ratio (`height = width × (ih / iw)`).

---

## Репро

```html
<img src="pic.png">              <!-- рисуется: intrinsic 120×80 -->
<img src="pic.png" width="240">  <!-- НЕ рисуется: высота схлопнулась -->
```

Headless: `lumen --screenshot out.png repro.html` — второй блок отсутствует (проверено пиксельным замером: одна полоса вместо двух). Воспроизводится и с PNG, и с растеризованным SVG (найдено при приёмке RP-5, формат источника нерелевантен).

## Первопричина (гипотеза, сверить с кодом)

`apply_intrinsic_size` (`crates/shell/src/main.rs`) добавляет intrinsic `width`/`height` презентационными атрибутами только когда у `<img>` НЕТ ни `width`, ни `height` («if neither»). При явном `width="240"` intrinsic-размеры не прокидываются вовсе → layout не знает ни intrinsic height, ни aspect-ratio → used height = 0.

## Как чинить

1. Прокидывать intrinsic размеры всегда (отдельным каналом, не презентационными атрибутами — чтобы не конфликтовать с авторскими), либо добавлять недостающий атрибут из aspect-ratio.
2. Правильнее по спеке: в layout для замещаемого элемента с `width: <fixed>` и `height: auto` использовать `height = width / aspect_ratio` (и симметрично). Это же закрывает `height`-only случай.

## Валидация

- Репро выше: оба блока видимы, второй 240×160.
- Регресс: `<img>` без атрибутов (intrinsic) и с обоими атрибутами не меняются.
