# BUG-204

**Статус:** FIXED 2026-06-24
**Компонент:** layout
**Тест:** TEST-85 (diff 1.98%)

## Описание

`anchor-name: --foo` базовое объявление — визуализация элемента отличается от Edge (стаб)

## Воспроизведение

`python graphic_tests/run.py --only 85` → FAIL 1.98%

## Как чинить

Реализовать базовую регистрацию anchor-name в layout — хотя бы статический positioned fallback.

## Решение

`anchor-name` оказался ни при чём. Истинная причина: в Lumen отсутствовало UA-правило
`body { margin: 8px }` (HTML Rendering §14.3.3). Тесты 85–89 — единственные, что НЕ сбрасывают
margin (`* { margin: 0 }`), поэтому Edge сдвигал `.__f`-рамку на 8px (тонкая рамка магенты сверху-слева,
клип справа-снизу), а Lumen рисовал её вплотную к краю viewport → диф ~2% по периметру рамки.
Контент (anchor-боксы) совпадал пиксель-в-пиксель в обоих движках.

Фикс: `apply_ua_body_margin` в `crates/engine/layout/src/style.rs` — выставляет `margin: 8px` на `<body>`
в UA-фазе (до author-каскада, поэтому `* { margin: 0 }` перекрывает). Все пять тестов (85–89) после фикса
дают <0.5% против Edge-эталона (CPU-снимок): 85→0.00%, 86→0.13%, 87→0.00%, 88→0.00%, 89→0.00%.

Этим же фиксом закрыты BUG-205…BUG-208 (та же первопричина).
