# BUG-242

**Статус:** FIXED 2026-06-23
**Компонент:** layout / paint / shell
**Файлы:** `crates/engine/layout/src/box_tree.rs`,
`crates/engine/paint/src/backend.rs`,
`crates/engine/paint/src/backends/femtovg_backend.rs`,
`crates/shell/src/main.rs`

## Описание

Фон канвы не покрывал вьюпорт целиком. Фон корневого элемента (или
`<body>`, распространённый на `<html>` функцией `propagate_canvas_background`)
красился только на rect html-бокса. У html с `height: auto` used-высота = высота
контента, поэтому в окне больше страницы фон заливал лишь полосу размером с бокс,
а остальная канва оставалась белой (UA-дефолт).

Наглядно на TEST-90 (AVIF-страница, фикс. `body{width:1024px;height:720px;
background:#ff00ff}`) при развёрнутом окне: мажента справа во всю ширину окна,
но только на высоту 720px; ниже контента — белая полоса. По CSS Backgrounds
§3.11.1 фон корня становится фоном канвы и покрывает весь вьюпорт.

Дефект не ловится `graphic_tests`: они рендерятся ровно 1024×720, где html-бокс
совпадает с канвой, и расхождения нет. Проявляется только когда окно ≠ размеру
страницы (например, maximize).

## Реальная причина

Painter красит фон любого бокса в его собственный `rect` (`background_clip_rect`).
Для корневого элемента это неверно: «область отрисовки фона корня = канва»
(весь вьюпорт + скроллируемая область), а не border-box элемента. Спец-случая
для корня в paint не было.

## Фикс

Чистый paint-special-case через clear-цвет кадра:

1. `lumen_layout::canvas_background_color(root)` — возвращает непрозрачный
   `background-color` бокса корневого элемента (тот, что `propagate_canvas_background`
   кладёт на `<html>`). Только полностью непрозрачные цвета (полупрозрачный фон
   корня должен композиться над UA-канвой — это уже делает собственный `FillRect`
   бокса в пределах его rect).
2. Новый метод трейта `RenderBackend::set_canvas_background(Option<Color>)`
   (дефолт — no-op; бэкенды без override продолжают чистить в белый).
3. Femtovg-бэкенд хранит `canvas_bg` и чистит **всю** поверхность этим цветом
   в `render()` вместо хардкоженого белого.
4. Shell вычисляет фон канвы из `self.layout_box` и вызывает
   `set_canvas_background` перед каждым `render`.

При размере окна = странице (graphic_tests, snapshot) clear-цвет полностью
перекрывается собственным `FillRect` html-бокса (≥ поверхности) → пиксели
идентичны прежним, регрессий тестов нет. Расхождение видно только когда окно
больше страницы — там фон страницы теперь заполняет вьюпорт.

Найдено при разборе TEST-90 (ветка `p3-bug209`), отдельно от BUG-241 (flex) и
BUG-209 (AVIF-декодер).
