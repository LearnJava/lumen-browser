# BUG-162

**Статус:** OPEN
**Компонент:** encoding
**Файл:** `crates/engine/encoding/src/detect.rs`

## Описание

Детектор кодировки для `example.com` (чистый ASCII-контент) выдал `ibm866`
вместо UTF-8/ASCII. На ASCII-странице это не портит вывод (ASCII ⊂ обеих
кодировок), но детектор ошибается на тривиальном входе — на пограничном
байтовом контенте даст неверный результат.

Обнаружено при рендеринге реальных сайтов.

## Старт расследования

В `crates/engine/encoding/src/detect.rs` проверить эвристику для входа без BOM,
без `<meta charset>` и без не-ASCII байтов: должна давать UTF-8 (или
windows-1252 per WHATWG fallback), но не ibm866. Вероятно, статистический
детектор кириллицы срабатывает на пустом/ASCII профиле байтов.
