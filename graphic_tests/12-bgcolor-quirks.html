LUMEN — QUIRKS MODE (нет DOCTYPE) + LEGACY BGCOLOR
body bgcolor="#1a2030" — HTML5 presentational hint (применяется в любом mode)
Body тёмно-синий — это от атрибута bgcolor на <body>. Author CSS перекрыл бы его, но здесь оставили чистый hint.
CSS hashless hex (только Quirks mode!) — color/background БЕЗ префикса #
color: f00 — без # — красный (Quirks)
color: 00aa00 — без # — зелёный (Quirks)
background: 4488ff — без # — синий фон (Quirks)
background: ffcc00 — без # — жёлтый фон (Quirks)
Эти же значения в Standards mode (с <!DOCTYPE html>) были бы invalid, и блоки остались бы без цвета / unstyled.
<table bgcolor> + Quirks UA-stylesheet reset font у table
Body font-size = 28px, color = yellow ↓ передаются на потомков, но <table> в Quirks их сбрасывает на initial:
td · redtd · greentd · blue
cellcellcell
В Quirks <table> игнорирует унаследованные font-size/color/text-align/white-space — это apply_quirks_table_reset (UA stylesheet, эквивалент Chromium/Firefox для legacy-страниц 90-х/2000-х).
legacy color parsing на bgcolor — мусор тоже принимается (HTML5 §2.4.6)
Парсер «rules for parsing a legacy color value» лояльный: принимает named, #rgb/#rrggbb, hashless hex, заменяет не-hex символы на «0».
