# STATUS-P5 — Code Health & Maintenance

**Developer:** Программист 5 (здоровье кода, аудит, чистка — НЕ фичи, НЕ баг-фиксы по геометрии)

P5 — периодическая ревизия всего проекта. Не пишет фичи (это P1/P2), не чинит
визуальные баги (это P3), не реализует CSS (это P4). Задача P5 — **находить
накопившийся мусор и расхождения, делать безопасную механическую чистку, а
крупные находки оформлять задачами для P1–P4**.

Принцип: **аудит + дешёвая безопасная правка + заведение задач.** Рефакторинг,
меняющий API или поведение, P5 НЕ делает в одиночку — оформляет как задачу и
отдаёт владельцу крейта (P1/P2). См. CLAUDE.md §«Code health: P5 only».

> ⚠️ **Запуск через оркестратор — всегда с `--max-tasks`.** Задачи P5
> рекуррентные (health-свип не уходит из «Next»), поэтому `has_tasks("P5")`
> всегда True и `python scripts/orchestrator.py P5` без лимита будет гонять
> ревизию бесконечно. Запускай `python scripts/orchestrator.py P5 --max-tasks 1`
> (один прогон) либо останавливай вручную `--stop P5`. Это отличает P5 от
> фич-сессий P1–P4, у которых «Next» со временем пустеет.

## Алиасы (триггеры → набор действий)

P5 понимает короткие команды. Каждая запускает фиксированный набор действий.
Запуск через скилл `/lumen-health-check <target>` или фразой-триггером.

| Алиас | Что делает | Команда скилла |
|---|---|---|
| `health` / `прогон здоровья` / `health check` | Полный свип: clippy + stubs + branches + docs + deps. Сводный отчёт. | `/lumen-health-check full` |
| `clippy` / `clippy-all` | `cargo clippy --workspace --all-targets -- -D warnings` целиком (P5-исключение из правила «-p»). Показать только error-блоки. | `/lumen-health-check clippy` |
| `stubs` / `заглушки` | Поиск `todo!()` / `unimplemented!()` / `unreachable!()` и висящих `// CSS:` хэндофов без хозяина + `grep OPEN BUGS.md`. | `/lumen-health-check stubs` |
| `branches` / `чистка веток` | Список влитых веток (`--merged main`), веток с `: gone]`, зомби-worktree. Предложить безопасное удаление. | `/lumen-health-check branches` |
| `docs` / `дрейф доков` | `python scripts/gen_symbols.py` → `git diff SYMBOLS.md` (рассинхрон символов); сверка статус-маркеров в lumen-plan.md с кодом. | `/lumen-health-check docs` |
| `deps` / `аудит зависимостей` | Новые/дублирующиеся зависимости: `cargo tree -d`, поиск незадокументированных `[dependencies]` без «Why this dependency». | `/lumen-health-check deps` |
| `dupes` / `дубликаты` | Поиск похожих/дублирующих полей в `ComputedStyle` и параллельно реализованных функций (риск параллельных сессий P1/P2/P4). | `/lumen-health-check dupes` |

Полный свип `health` запускает clippy → stubs → branches → docs → deps по очереди
и выдаёт единый отчёт. `dupes` в полный свип не входит (дорогой, по запросу).

## Workflow

1. **Запусти нужный свип** (алиас выше или `/lumen-health-check full`).
2. **Безопасную чистку делай сразу** (без отдельной задачи):
   - удаление веток, влитых в main (`git branch -d`, только `--merged`);
   - удаление зомби/осиротевших worktree (`git worktree prune` + `remove`);
   - регенерация `SYMBOLS.md` (`python scripts/gen_symbols.py`) — отдельным коммитом;
   - тривиальный clippy-фикс (неиспользованный импорт, `&` лишний) в своём крейте.
3. **Крупные находки НЕ чини сам** — заведи:
   - визуальный/логический баг → строка `OPEN` в `BUGS.md` (следующий BUG-NNN);
   - дублирование / нужный рефакторинг → пункт в `STATUS-P1.md`/`STATUS-P2.md` «Next»
     с описанием и `file:line`;
   - незавершённый CSS-хэндоф → пункт в `STATUS-P4.md` «Needs wiring».
4. **Ветка P5:** `p5-health-<дата>` или `p5-<тема>`, например `p5-health-2026-06-03`,
   `p5-cleanup-stale-branches`. Worktree обязателен.
5. **Запиши прогон в [`docs/HEALTH-LOG.md`](docs/HEALTH-LOG.md)** — одна запись на свип
   (дата · таргет · сводка · сделано · заведено), новые сверху. Делать **всегда**,
   даже если свип чистый — иначе факт прогона теряется (git фиксирует только свипы,
   давшие коммит).
6. **Коммит** — clippy-чистый, тесты зелёные на затронутых крейтах.

P5 правит код **только** в своём крейте при тривиальном clippy-фиксе. Любая правка
в чужом крейте, меняющая поведение, оформляется как задача владельцу.

## Notes

- **Не фичи, не баг-геометрия, не CSS** — это P1/P2/P3/P4. P5 = гигиена.
- **Безопасность важнее охвата.** Удалять только `--merged` ветки. Никаких
  force-delete веток, в которые что-то не влито.
- **Крупный рефакторинг — через владельца крейта.** P5 находит и описывает,
  делает P1/P2 обычным фич-флоу.
- **Workspace-clippy — осознанное исключение** из правила «всегда `-p`» (CLAUDE.md
  §Cargo output rules). Полный свип — единственная причина гонять `--workspace`.

См. CLAUDE.md §«Code health: P5 only» для полного описания границ роли.
