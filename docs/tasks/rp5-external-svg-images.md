# RP-5 — Внешние SVG-картинки (`<img src=*.svg>` / `background-image: url(*.svg)`)

**Developer:** P1 (+ P4 если понадобится проводка размеров из CSS) · **Branch:** `p1-rp5-external-svg` · **Size:** L · **Crates:** `lumen-image`, `lumen-paint`, `lumen-shell`
**Roadmap:** RP-5 (под RP «Рендер-паритет реального веба», P3)

---

## Статус

**OPEN.** Найдено аудитом 2026-07-02. Внешние SVG не отрисовываются на большинстве реальных сайтов (логотипы rust-lang.org, w3.org, иконки HN, соц-иконки). В логах: `Не декодируется /static/images/logo.svg: формат изображения не распознан по сигнатуре`.

## Что сейчас

- `crates/engine/image/src/lib.rs:38-40` — `supported_mime_types()` = `png, jpeg, gif, webp, avif`. **SVG нет.**
- `decode_raw()` (:69-96) сниффит сигнатуру и на неизвестном формате возвращает `Err(UnknownFormat)` (:95).
- **Инлайновый** `<svg>` в разметке рендерится (layout строит SVG-дерево → `DrawSvgPath`/`DrawSvgFill`), но это путь DOM→layout, а не image-decode. Внешний файл `.svg` идёт по image-пути и падает.

## Что сделать

Мост «SVG-байты → растровое `DecodedImage`» так, чтобы `<img>`/`background-image`/`content:url()` с `.svg` рисовались.

**Вариант A (переиспользовать свой SVG-движок) — предпочтителен, без новых зависимостей:**
1. Распарсить внешние SVG-байты в тот же внутренний SVG-IR, что строит инлайновый путь (найти, где layout превращает `<svg>`-поддерево в команды; вынести парс SVG-разметки в переиспользуемую функцию, не завязанную на DOM-арену).
2. По запрошенному размеру картинки (intrinsic из `width`/`height`/`viewBox` SVG, либо CSS-размер бокса) отрастрировать в `DecodedImage` (RGBA) — можно через существующий CPU-путь (`render_to_image_cpu` над display-list из SVG-IR).
3. Добавить сигнатуру в `decode_raw`: SVG определяется не магическими байтами, а по `content-type: image/svg+xml` ИЛИ по префиксу (`<?xml`/`<svg` после trim BOM/пробелов). Провести `content_type` в `decode_raw` (сейчас, вероятно, только байты) — mime знает шелл на fetch.
4. Кэшировать растр по (url, target-size) — SVG перерастрируется при смене размера.

**Вариант B (внешняя зависимость resvg/usvg):** быстрее написать, но `resvg` тянет много кода и свой шрифто-стек. По dependency-policy — provisional, нужен trait-anchor + критерий отказа. Использовать только если A окажется дорогим по покрытию SVG-фич.

Рекомендация: A, т.к. движок уже рисует SVG-контуры/градиенты/`<use>`; дублировать растеризатор через resvg — регресс по консистентности.

## Тонкости

- Внешний SVG **не исполняет скрипты** и обычно статичен — анимации/`<script>` внутри можно игнорировать (как большинство `<img>`-SVG).
- `currentColor` в SVG-иконках должен резолвиться в `color` бокса-хоста (частая причина «чёрных» иконок).
- Размер: если у `<img>` нет `width/height`, брать intrinsic из `viewBox`/`width`/`height` SVG (CSS Images §5.1).
