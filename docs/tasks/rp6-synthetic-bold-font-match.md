# RP-6 — Синтетический bold + системный подбор шрифта в CPU/headless-пути

**Developer:** P1 · **Branch:** `p1-rp6-synthetic-bold` · **Size:** M · **Crates:** `lumen-font`, `lumen-paint`
**Roadmap:** RP-6 (под RP, P3)

---

## Статус

**OPEN.** Найдено аудитом 2026-07-02. В headless/CPU-пути (`--screenshot`, cpu_vs_edge) доступен только бандловый **Inter Regular** — все заголовки/жирный текст рисуются обычным начертанием, каждая страница «выглядит не так». В живом окне femtovg подхватывает системные шрифты через `FontProvider`, но синтеза bold нет нигде: если подходящего bold-фейса нет — текст остаётся Regular.

## Что сейчас

- `crates/engine/paint/src/backends/femtovg_backend.rs:1189` — `const BUNDLED_FONT = include_bytes!(".../Inter-Regular.ttf")` (единственный фейс, вес 400).
- Системные шрифты грузятся только при заданном `FontProvider` (`set_font_provider`, :3405) — в CPU/headless он обычно не задан.
- `font-weight`/`font-style` доходят до рендера (`display_list.rs:385-388`), но синтеза нет: **не найдено** ни double-stroke, ни emboldening, ни fake-italic (skew).

## Что сделать

**Срез A — синтетический bold (быстро, максимальный визуальный эффект):**
1. Когда для запрошенного `font-weight >= 600` подобранный фейс имеет вес < 600 (нет настоящего bold), эмулировать жирность: обводка глифа тем же цветом с толщиной, пропорциональной кеглю (femtovg — `stroke` поверх `fill`; CPU tiny_skia — аналогично), либо небольшое расширение контура. Классический fake-bold.
2. Аналогично fake-italic: shear-трансформ (~12–14°) для `font-style: italic`, когда нет italic-фейса.
3. Только как fallback — если настоящий bold/italic-фейс есть, использовать его.

**Срез B — системный подбор в CPU/headless (опционально, но убирает Inter-монокультуру):**
1. Дать CPU-пути тот же `FontProvider`, что и окну, чтобы `render_source_to_png` подбирал системные семейства (по крайней мере bold Inter/дефолтные sans/serif). Осторожно: headless обязан оставаться **детерминированным** для cpu_vs_edge-гейта — системный подбор ломает воспроизводимость на разных машинах. Поэтому: под флагом/только для `--screenshot` пользователя, а тест-гейт держать на bundled-наборе. Возможно, добавить в бандл Inter-Bold/-Italic — детерминированно и закрывает 90% случаев.

Рекомендация: начать со **среза A** (эффект сразу, детерминизм не страдает) + добавить в бандл `Inter-Bold.ttf`/`Inter-Italic.ttf` (SIL OFL, как Regular).

## Тонкости

- `font-parity` (Inter vs шрифт Edge) — известный класс должников (BUG-128); RP-6 его не закрывает и не должен пытаться (порог 0.5% по gdigrab недостижим без тех же шрифтов). Цель RP-6 — чтобы **жирный был жирным**, а не пиксель-в-пиксель с Edge.
- Синтетический bold менять метрики строки не должен (ширина глифа ≈ та же) — иначе поедет layout. Обводка наружу минимальна.

## Валидация

- Графический тест с `font-weight: 700` / `font-style: italic` на бандловом шрифте: видимая разница начертаний.
- Регресс: cpu_vs_edge baseline без изменений (если срез B под флагом и бандл-набор не трогает существующие эталоны — проверить, не сместились ли тесты, где уже есть bold-текст).
