# F2-7 — Phase 2 closure: bump 0.2.0 → 0.5.0

**Developer:** P1
**Branch:** `p1-f2-7-version-bump`
**Size:** S
**Crates:** workspace-wide (version metadata only)

## Goal

Final step of Phase 2. Once F2-1…F2-6 are complete, raise the application version
from `0.2.0` to `0.5.0` — the Phase 2 target per the version policy
(`Phase 2 → v0.5 (target on phase completion)`) — and replace the
"in progress" phase wording with the completed-phase wording everywhere the
label is hard-written. The version itself lives in exactly one place
(`[workspace.package] version` in the root `Cargo.toml`); every
machine-readable version string (User-Agent, Sec-CH-UA, CDP
`Browser.getVersion`, BiDi capabilities, window title, startup banner)
derives from `CARGO_PKG_VERSION` and changes automatically — those sites must
NOT be edited. The only two manually-maintained literals are the
`navigator.userAgent` JS-shim string and the hard-coded "Phase 2" label in
the startup banner / docs.

## Prerequisites

- F2-1…F2-6 all complete: parity ≤ 0.5% on their graphic tests, or parked as
  `KNOWN_DEBTORS` in `graphic_tests/run.py` with an OPEN `BUG-NNN`.
- No other Phase 2 roadmap item is still "In progress" in `STATUS-P1.md`.
- **F2-7 is strictly last.** Do not bump the version while any Phase 2 goal is
  still open — the version number is a public claim that the phase is done.

## Steps

1. Edit `Cargo.toml:29`: `version = "0.2.0"` → `version = "0.5.0"`.
2. Run `cargo update -p lumen-shell` (or any `cargo check`) so `Cargo.lock`
   re-records the new workspace version for every `lumen-*` package; stage the
   resulting `Cargo.lock` (it is committed).
3. Edit `crates/js/src/dom.rs:5916` (section B): `Lumen/0.2.0` → `Lumen/0.5.0`.
4. Update every section-C phase-label / version mention listed above.
5. Edit `docs/roadmap.json:44` (section D), then run
   `python scripts/gen_roadmap.py` to regenerate the two HTML trees — do not
   edit the HTML by hand.
6. Move the F2-7 entry in `STATUS-P1.md` from "In progress" to "Recent".
7. Build + verify (next section).
