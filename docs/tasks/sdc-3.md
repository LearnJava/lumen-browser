## Контекст
SDC-1b даёт командный канал в живое окно (Navigate/Screenshot/… через `AutomationCommand`). SDC-3 —
первый потребитель: перевести `graphic_tests/run.py` с модели «процесс на тест» на «одно живое
окно на весь прогон». Это реализует уровни 2-3 стратегии `docs/plan/testing.md` (структурные
ассерты + in-process snapshot) против реального окна, как и задумано дизайн-доком (уровень 4
gdigrab по плану — ночной CI, не основной гейт).

Сейчас `capture_lumen` (run.py:712) на каждый тест: `Popen([LUMEN, page])` → ffmpeg-gdigrab →
kill. Перезапуск процесса/окна на каждый тест — главный расход и источник хрупкости (гонки
фокуса, «magenta marker not found»).

## Что сделать
1. Менеджер живого окна (по образцу IPC-клиента run.py:530): при старте прогона поднять оконный
   шелл с automation-каналом (SDC-1) один раз, дождаться готовности окна.
2. Калибровка один раз: первый тест TEST-00 → найти crop-offset по магента-маркеру → зафиксировать
   на весь прогон (не пересчитывать каждый тест).
3. Цикл по тестам: `Navigate(test_url)` → ждать ack «кадр готов» (таймаут + ретрай по образцу
   «magenta marker not found») → скриншот окна (через канал SDC-1 ИЛИ существующий gdigrab) →
   crop по сохранённому offset → diff с Edge.

5. Совместимость: `--ipc` (CPU-снимок) остаётся; gdigrab-per-launch заменяется одним живым окном.
   Старый `capture_lumen` оставить фолбэком или удалить, если новый путь стабилен.
