# 8A.7: Phase 4 — Shell Integration Plan

## Objective

Rewrite `lumen-shell` to use `BrowserSession` trait (implemented by `WinitSession`) as the primary browser interface instead of direct pipeline calls. This makes winit/wgpu one transport layer, not the center of architecture.

## Current State (After Phase 3)

- ✅ **WinitSession** fully implements `BrowserSession` trait for headless operations
- ✅ 40+ integration tests passing
- ❌ lumen-shell still uses direct pipeline calls: `source.load()`, `display_list`, `renderer`, `layout_box`
- ❌ WinitSession missing: GPU rendering, streaming, scroll state management, event dispatch
- ❌ No connection between lumen-shell and WinitSession

## Architecture Challenge

Lumen currently:
1. Loads HTML asynchronously in chunks (streaming pipeline)
2. Renders to GPU via `renderer` field
3. Manages complex state: animations, scroll, find, hints, etc.
4. Handles events directly: mouse, keyboard, touch

WinitSession currently:
1. Loads HTML synchronously (blocks on `navigate()`)
2. No GPU rendering (headless only)
3. No scroll state management
4. No event dispatch support

## Solution Approach: Incremental Refactoring

Instead of a complete rewrite, bridge the two architectures:

### Phase 4a: GPU Session Adapter (Week 1)

Create `GpuSession` trait in `lumen-driver` that extends `BrowserSession`:

```rust
pub trait GpuSession: BrowserSession {
    /// Render the current page to GPU and get display_list + rendered data
    fn render_to_gpu(&mut self) -> Result<RenderedPage>;
    
    /// Update scroll position and invalidate layout
    fn set_scroll(&mut self, target: &Target, delta: ScrollDelta) -> Result<()>;
    
    /// Get current viewport size
    fn viewport_size(&self) -> Size;
    
    /// Handle streaming load with callbacks for intermediate frames
    fn navigate_streaming(&mut self, url: &str, on_chunk: &dyn Fn(RenderedPage)) -> Result<()>;
}
```

Implement `GpuSession` for `WinitSession`:
- Add `Renderer` field
- Add `scroll_state` management
- Add streaming support via callback
- Implement GPU rendering via `Renderer::render_to_image()`

### Phase 4b: Migrate reload() (Week 1)

1. Change `Lumen::reload()` to use `GpuSession::render_to_gpu()` instead of `source.load()`
2. Preserve all state management (animations, scroll, UI)
3. Test that page rendering still works identically

### Phase 4c: Migrate navigation (Week 2)

1. Change `navigate_to()` to use `GpuSession::navigate()` instead of `source.load()` + reload
2. Add scroll restoration via `GpuSession::set_scroll()`
3. Preserve history management, bfcache, JS context

### Phase 4d: Event handling (Week 2)

1. Implement proper event dispatch in `GpuSession::click()`, `type_text()`, `scroll()`
2. Wire event loop through WinitSession
3. Test keyboard/mouse input

### Phase 4e: Testing & cleanup (Week 3)

1. Run full integration tests
2. Verify graphic_tests still pass
3. Clean up old pipeline code if possible
4. Update documentation

## Key Design Decisions

1. **Don't remove old pipeline yet** — keep `source.load()` as fallback during transition
2. **GpuSession is a bridge** — allows gradual migration without breaking changes
3. **Preserve Lumen's streaming** — essential for perceived performance
4. **Keep UI state separate** — animations, find, hints remain in Lumen, not in session

## Risks & Mitigation

| Risk | Mitigation |
|---|---|
| Lumen becomes incomprehensible during transition | Use git commits to mark clear checkpoints; revert any broken state |
| New trait leaks internals of WinitSession | Keep trait minimal; implementation details in private modules |
| Performance regression due to extra indirection | Measure with `cargo run -p lumen-bench --release` after each phase |
| Streaming becomes bottleneck | Cache rendered frames; only re-render on viewport change |

## Success Criteria

- ✅ `lumen-shell` compiles and runs without errors
- ✅ Page loading (file:// and http://), navigation, scroll work identically
- ✅ UI features (find, hints, address bar) still functional
- ✅ Animations smooth (60 FPS baseline)
- ✅ `cargo test -p lumen-driver` all pass
- ✅ `graphic_tests/run.py` passes (at least TEST-00 through TEST-15)
- ✅ Clippy clean: `cargo clippy -p lumen-shell --all-targets -- -D warnings`
- ✅ No performance regression in `lumen-bench` (within 5% baseline)

## Commit Strategy

Each phase = one commit:
1. **8A.7-p4a:** Create GpuSession trait + WinitSession impl
2. **8A.7-p4b:** Migrate reload() to GpuSession::render_to_gpu()
3. **8A.7-p4c:** Migrate navigation to GpuSession::navigate()
4. **8A.7-p4d:** Event handling + dispatch
5. **8A.7-p4e:** Tests + cleanup

If any phase breaks tests, roll back to previous commit and investigate.
