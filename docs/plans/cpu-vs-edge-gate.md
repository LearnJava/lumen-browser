Статус: УТВЕРЖДЁН пользователем 2026-06-11 (оба открытых вопроса решены), работы не начаты
Создан: 2026-06-11

## 1. Цель

Заменить медленный ffmpeg+gdigrab+Edge пайплайн (`graphic_tests/run.py`) быстрым
детерминированным Rust-тестом: **tiny-skia CPU-рендер каждой страницы
`graphic_tests/` сравнивается с эталонным скриншотом Edge, порог 0.5%**.

Ключевое свойство: tiny-skia бит-идентичен на Windows/Linux/macOS и не зависит
от GPU/драйвера. Результат теста — функция только кода движка. Страница, один
раз прошедшая 0.5%, проходит всегда и везде (в отличие от femtovg/wgpu-путей,
где обновление видеодрайвера сдвигает проценты).

Чем этот план НЕ является:
- НЕ замена `snapshot_cpu.rs` (побайтовый регрессионный гейт остаётся);
- НЕ доведение wgpu-fallback до паритета с femtovg (тупиковая двойная поддержка);
- НЕ femtovg headless (GPU-путь не даёт воспроизводимости между машинами).

## 7. Этап 4 — довести движок до 0.5% на каждой странице

**Роль:** P3 (обычный баг-цикл, ветки `p3-bug-<id>`). **Размер:** XL,
итеративно, главная по трудозатратам часть.

- [ ] 4.1 По таблице из 3.4: каждая страница >0.5% — строка `OPEN` в
      `BUGS.md` (многие уже заведены: BUG-076/079/080/082/084/085/088/090/
      092/094/098/099/100/101, BUG-125…130 и др. — сверить и не дублировать,
      в существующих обновить проценты на CPU-цифры).
- [ ] 4.2 P3 чинит в порядке убывания отклонения. Критерий закрытия бага —
      страница ≤0.5% в `cpu_vs_edge` (детерминированно: процент не плавает
      между прогонами и машинами).
- [ ] 4.3 После каждого фикса, меняющего картинку, — перегенерация эталона
      `snapshot_cpu` этой страницы в том же коммите.
- [ ] 4.4 **Сортировка багов по слою.** Если страница неправильная и в CPU,
      и в окне — дефект в layout/display list (чинится один раз для всех
      бэкендов). Если CPU ≤0.5%, а окно нет — дефект в
      `femtovg_backend.rs`; на этот гейт не влияет, заводится отдельным
      багом, ловится ночным `run.py`.
- [ ] 4.5 **Заведомо нереализуемое в Phase 0/1** (пример: 57-canvas —
      BUG-099, `getContext("2d")` — Phase 2). Пороги менять запрещено
      (правило 2026-06-11), страницы удалять запрещено. Механизм —
      **список известных должников с baseline-храповиком**:

      ```rust
      /// (страница, OPEN-баг, зафиксированное отклонение %).
      /// Выход из списка — только через измерение ≤0.5%.
      const KNOWN_DEBTORS: &[(&str, &str, f64)] = &[
          ("57-canvas", "BUG-099", 28.66),
      ];
      ```

      Семантика (возможна благодаря детерминизму: tiny-skia бит-стабилен,
      эталон Edge заморожен в репо → процент точен и воспроизводим):
      - факт > baseline → FAIL: на странице сломали уже работающее;
      - факт == baseline → OK, должник в ожидаемом состоянии (процент
        печатается красным в каждом отчёте);
      - факт < baseline → FAIL с сообщением «понизь baseline до X» —
        прогресс фиксируется, движение только вниз (храповик);
      - факт ≤ 0.5% → FAIL с сообщением «удали строку из списка» —
        список не может протухнуть; страница становится блокирующей.

      При заведении должника обязательно: diff-картинка подтверждает, что
      расхождение локализовано в области нереализованной фичи (для
      57-canvas — внутри прямоугольника `<canvas>`), это фиксируется в
      тексте BUG-NNN. Реализация фичи сама по себе НЕ выводит из списка —
      только измеренное ≤0.5%; промежуточные улучшения двигают baseline.
