# Состояние подсистем Lumen

Полное состояние каждого крейта на момент последнего merge: scope, что готово / отложено, число тестов, поведенческие инварианты. Этот файл обновляется при каждом коммите, реализующем пункт плана (см. `CLAUDE.md` → «Статус реализации — поддерживай актуальным»).

Чтобы узнать **сейчас** что есть в каком крейте — этот файл может отставать на несколько коммитов; ground truth — `git log --oneline` + `cargo doc -p <crate>` + чтение `src/lib.rs`.

---

### `lumen-core` ✅ (фундамент стабилен)

- Типы: `Error`, `Result<T>`, `Url`, `Event` (TabCreated/Closed, Navigation, PageLoaded, RequestStarted/Completed/Blocked), `TabId`, `Capability`, `CapabilityToken`, `Module` trait.
- Геометрия: `Rect`, `Point`, `Size`.
- `lumen-core::ext` — trait-точки расширения: `NetworkTransport`, `StorageBackend` (с origin-партиционированием + `list_keys`), `SearchProvider`, `FilterListSource` (загрузчик правил), **`RequestFilter`** (per-URL `should_block(&Url) -> Option<String>`; отделён от `FilterListSource`, потому что `HttpClient` зависит только от per-URL решения и не знает формата правил), `EncodingDetector`, **`EventSink`** (`fn emit(&self, event: &Event)`, `Send + Sync`, sink принимает `&Event` без `clone`; «молчаливого дефолта» нет — потребители держат `Option<Arc<dyn EventSink>>` и `None` означает «события никому не нужны»), **`DnsResolver`** (`fn resolve(&self, hostname: &str, port: u16) -> Result<Vec<SocketAddr>>`; реализации: `lumen-network::SystemDnsResolver` поверх `to_socket_addrs`, `lumen-storage::CachedDnsResolver` с TTL-кешем поверх `DnsCache`, `lumen-network::DohResolver` (RFC 8484, DNS-over-HTTPS), `lumen-network::DotResolver` (RFC 7858, DNS-over-TLS)), **`HstsEnforcement`** (`is_https_only(host, now_unix) -> bool`, `record_sts(host, max_age, include_subdomains, preload, now_unix)`; без `Result`, fail-open; реализация в `lumen-storage::hsts::HstsStore`, потребитель — `lumen-network::HttpClient::with_hsts(...)` для http→https upgrade и persist `Strict-Transport-Security` policy из ответов по RFC 6797), **`HttpCredentialProvider`** (`credentials(&HttpAuthChallenge) -> Option<HttpCredentials>`; HTTP Basic + Digest на 401, провайдер ищет creds по `(origin, realm, scheme)`; реализация — `lumen-network::StaticCredentialProvider` для тестов / curl-style `-u`, UI-popup и keyring — следующие задачи P4 / платформенного слоя).
- **Sprint 0 P3 trait-anchors (lumen-plan.md § «Sprint 0 — Контракты»):** 7 trait-точек под provisional accelerators из §5, каждая со stub-реализацией `Null*` («не поддерживается») — позволяет потребителям кодировать против trait-а до того, как настоящий provisional-crate подключён. `UnicodeProvider` (UAX #14 line-break / #29 grapheme+word segmentation / #9 bidi runs — под `icu4x.segmenter`+`icu4x.linebreak`; `NullUnicodeProvider` отдаёт пустые векторы). `IdnaProvider` (UTS #46 `to_ascii`/`to_unicode` — под `idna`-crate; `NullIdnaProvider::to_ascii` возвращает None, потребитель fall-back-ит на собственный Punycode из `lumen_core::idn`). `PublicSuffixList` (`public_suffix`/`registrable_domain`/`is_public_suffix` — под `publicsuffix`-crate, P3 п.2B; `NullPublicSuffixList` — все None/false, RFC 6265 §5.3 step 5 даёт strict-host fallback). `ContentDecoder` (HTTP `Content-Encoding`: `encoding()` + `decode(input) -> Result<Vec<u8>>`; per-кодек реализация — `gzip`/`deflate` свои через DEFLATE из `lumen-image`, `br`/`zstd` — `brotli-decompressor`/`ruzstd` provisional). **`UnsupportedContentDecoder { encoding: &'static str }`** — stub, `decode` всегда возвращает `Error::Other("content-encoding `X` not supported")` (а не identity passthrough — это не безопасный fallback). `FontFormat` (`format_name`/`can_decode`/`decode_to_sfnt` — под `woff2`-crate, P2 Phase 2 WebFonts; `NullFontFormat` отвергает всё). `SpellChecker` (`check`/`suggest`/`locale` — под `hunspell-rs`/`spellbook`, Phase 3; `NullSpellChecker::check` всегда true чтобы UI не подчёркивал всё). `HyphenationProvider` (`hyphenate(word, locale)`/`locales` — под `hyphenation` с TeX-словарями; `NullHyphenationProvider` отдаёт пустые векторы). Все Null-stub-ы dyn-safe + Send+Sync, покрыты unit-тестами на dyn-cast/return-values.
- **`JsRuntime` (lumen-core::ext)** — JS-runtime: `eval`/`set_global`/`get_global`/`call_function` через `JsValue` (JSON-совместимые типы: Null/Undefined/Bool/Number/String/Array/Object с sorted-by-key entries). `JsError { Parse(String), Runtime(String), NotImplemented }` + `JsResult<T>`. `NullJsRuntime` — все методы возвращают `JsError::NotImplemented`; первый реальный runtime — QuickJS через `rquickjs` (Phase 2-3, P3 п.1B). Trait помечен dyn-safe + Send+Sync.
- В комментариях задокументированы оставшиеся будущие trait-точки: `WindowingBackend`, `RenderBackend`, `TlsBackend`, `Hasher`. Тело trait-а добавим при первой реализации. (`FontProvider` реализован — см. `lumen-font::SystemFontIndex`.)
- **`lumen_core::punycode::encode`** — RFC 3492 Punycode encode (bootstring base=36, tmin=1, tmax=26, skew=38, damp=700, initial_bias=72, initial_n=128). 8 unit-тестов на известных IDN-метках (пример → e1afmkfd, рф → p1ai, президент → d1abbgf6aiiy, тест → e1aybc, рус → p1acf, CJK 你好 → 6qq79v).
- **`lumen_core::idn::domain_to_ascii`** — IDNA `ToASCII` (упрощённое подмножество): lowercase → split по `.` → label-by-label (ASCII passthrough, иначе `xn--<punycode>`). Не делает NFC normalization и UTS #46 mapping (для русских доменов уже в NFC — практически достаточно). 10 unit-тестов (включая идемпотентность для уже `xn--…`, mixed ASCII+IDN субдоменов, trailing dot для FQDN, case-нормализация).
- **`Url` — структурированный тип** (`scheme`, `host`, `port`, `path`, `query`, `fragment`, `serialized` cache). Парсер по упрощённой RFC 3986 §3 в рамках scope: http/https/file (требуют `//` authority), data: и неизвестные схемы (без authority). API: `scheme()`, `host()` (Unicode для адресной строки), `port()` (None если не указан), `effective_port()` (с дефолтами http=80/https=443), `path()`, `query()`, `fragment()`, `as_str()` (serialized round-trip), `host_ascii()` (Punycode для DNS/TLS/Host header — RFC 7230 §5.4, RFC 6066 §3; пустой host у data: даёт пустую строку без ошибки), `path_and_query()` (для HTTP request line), `resolve(reference)` (упрощённый RFC 3986 §5.3: абсолютные / `//` protocol-relative / `/` absolute path / `#` fragment-only / `?` query-only / относительные — без нормализации `.`/`..`). userinfo (`user:pass@`) отбрасывается на парсинге; ASCII case-insensitive scheme; невалидный port → InvalidUrl. Этим типом теперь пользуются `lumen-network` (через `host_ascii`/`effective_port`/`path_and_query`/`resolve` для редиректов) и `lumen-shell` (через `Url::parse(base).resolve(href)` для относительных `<link href>`). Никакого ad-hoc парсинга URL в потребителях больше нет.
- **`lumen_core::hash`** — общий модуль протокольных хэшей. `sha256(&[u8]) -> [u8; 32]` по FIPS 180-4, `hex_lower(&[u8]) -> String` (lowercase, без префиксов/разделителей), `sha256_hex(&[u8]) -> String` (sha256 + hex). Не security-критичный crypto (для KDF / TLS / X.509 — rustls, exception #3); SHA-256 как протокольный hash: HTTP Digest auth response (RFC 7616 §3.4.3), Safe Browsing canonical-expression lookup, SRI digest. MD5 (RFC 1321) живёт локально в `lumen-network::auth` — нужен только для legacy Digest, выносить наружу смысла нет. 6 unit-тестов (FIPS 180-4 vectors: empty / abc / 56-byte two-block + hex_lower padding/empty).
- 127 тестов (24 url-parse/resolve/host_ascii + 8 Punycode + 10 IDN + 6 hash + ~12 JsRuntime/HttpCredentialProvider/JsValue + **13 Sprint 0 P3 stub-ов** (null_unicode/idna/public_suffix/font_format/spell_checker/hyphenation + unsupported_content_decoder + sprint0_send_sync) + остальные базовые).

### `lumen-dom` ✅ (полный API на текущий scope)

- Arena-based: `Vec<Node>` + `NodeId(u32)`. Нет `Rc/RefCell`, нет циклов.
- Типы: `Document`, `Node` (parent + children + data), `NodeData` (Document / Doctype / Element / Text / Comment), `QualName`, `Namespace` (HTML/SVG/MathML/Xml/XmlNs/XLink), `Attribute`.
- API: `create_element / create_text / create_comment / create_doctype`, `append_child`, `detach`, `get / get_mut`, `root`, `len`, **`base_href()`** (HTML5 §4.2.3 — извлечение `<base href>` для resolve относительных URL), **`find_first_element(predicate)`** (pre-order поиск первого элемента, удовлетворяющего predicate).
- `Display` impl печатает дерево с отступами — для отладки.
- **InputType (HTML5 §4.10.5):** enum для парсинга `<input type="...">` атрибута. 22 стандартных типа: `Text` (default), `Password`, `Email`, `Tel`, `Url`, `Number`, `Search`, `Date`, `DateTimeLocal`, `Time`, `Month`, `Week`, `Color`, `Range`, `Checkbox`, `Radio`, `File`, `Submit`, `Reset`, `Button`, `Image`, `Hidden` + `Other(String)` для forward-compat. `InputType::parse(s)` case-insensitive, trim whitespace. Метод `Node::input_type() -> Option<InputType>` возвращает `None` для не-`input` элементов, `Some(Text)` для `<input>` без `type` (HTML5 default). Helpers: `is_textual()` (text/password/email/tel/url/number/search — поля с буквенным контентом), `is_button_like()` (submit/reset/button/image). `as_str()` + `parse()` — round-trip.
- **DocumentMode (HTML5 §13.2.6.2):** enum `NoQuirks` (default) / `Quirks` / `LimitedQuirks` + поле `Document.mode` с `mode()`/`set_mode()` геттером/сеттером. Tree builder выставляет режим при обработке первого DOCTYPE-токена (см. `lumen-html-parser::quirks_mode::detect_document_mode`); при полном отсутствии DOCTYPE-токена — `Quirks` по §13.2.6.4.1. `Document::new()` даёт `NoQuirks` для удобства программного использования и unit-тестов. Layout/cascade сейчас режим **не читают** — это задел под десятки legacy CSS-переключений (table sizing, body-background propagation, font-size в `<table>`, и т.д.).
- 27 тестов: 7 базовых + cyrillic-инварианты + 6 для base_href + 12 для InputType + **2 для DocumentMode (default NoQuirks, set/get round-trip)**.

### `lumen-html-parser` 🟡 (минимум)

- **Готово:** iterator-based FSM (Tokenizer); состояния Data, TagOpen, TagName, EndTag, BeforeAttributeName, AttributeName, AfterAttributeName, BeforeAttributeValue, AttributeValue (quoted/unquoted), SelfClosingStartTag, MarkupDeclarationOpen, Comment, CommentEnd, **RAWTEXT** (для `<script>` и `<style>`), **RCDATA** (для `<title>` и `<textarea>`), **DOCTYPE**. Character references: **расширенный набор ~250 named entities** через модуль `entities` (бинарный поиск по сортированной таблице — Latin-1/diacritics, Greek alphabet, кавычки и тире `&mdash;`/`&ldquo;`/`&rdquo;`, символы `&copy;`/`&reg;`/`&trade;`/`&euro;`, математика `&sum;`/`&prod;`/`&ne;`/`&le;`/`&ge;`/`&asymp;`/`&infin;`, стрелки `&larr;`/`&rarr;`/`&hArr;`, whitespace `&ensp;`/`&emsp;`/`&thinsp;`/`&zwnj;`/`&zwj;`/`&lrm;`/`&rlm;`); все имена case-sensitive с обязательным trailing `;`; legacy без `;` (~100 ссылок) не поддерживаются. Numeric `&#NNN;` / `&#xHHHH;` через `char::from_u32`. Lenient tree builder с void-элементами и self-closing.
- **RAWTEXT / RCDATA детали:** объединённое поле `text_only: Option<(String, bool)>` — `bool` = `decode_entities`. После `<script>`/`<style>` (не self-closing) выставляется `(name, false)` — RAWTEXT, character references литеральны. После `<title>`/`<textarea>` — `(name, true)` — RCDATA, entities декодируются (нужно для `<title>Foo &amp; Bar</title>` → `Foo & Bar`). В обоих режимах `<` без `/tag` — текст; завершение только `</tag` + терминатор (whitespace / `/` / `>` / EOF), case-insensitive. `</scripto>` НЕ матчит (`o` не терминатор). Self-closing `<script/>` / `<title/>` режим **не** включают (симметрично tree_builder, который для self-closing не пушит элемент в стек). `is_raw_text_element(name)` и `is_rcdata_element(name)` — две отдельных функции; расширяемо для будущих специальных контентов (`iframe`, `noembed`, `noframes`, `noscript`, `plaintext`).
- **DOCTYPE детали:** `Token::Doctype { name, public_id: Option<String>, system_id: Option<String> }` (HTML5 §13.2.5.53–72). После `<!DOCTYPE` keyword (case-insensitive `doctype`/`DOCTYPE`) парсятся: name (lower-case, до whitespace или `>`), опционально `PUBLIC "id" "id"` или `SYSTEM "id"` с поддержкой одинарных и двойных кавычек. Lenient: `<!DOCTYPE>` без имени даёт пустой name, неполные DOCTYPE-ы не валятся. **`public_id`/`system_id` — `Option<String>`**: `None` если keyword отсутствует, `Some("")` если присутствует с пустой строкой. Различие важно для quirks-detection (limited-quirks правила для HTML 4.01 Frameset/Transitional зависят от наличия system_id, не его содержимого). Tree builder создаёт `NodeData::Doctype` узел (с `unwrap_or_default()` для DOM — арена не различает missing/empty). Прочие markup declarations типа `<![CDATA[...]]>` или `<!ENTITY ...>` по-прежнему молча skip-аются до `>`.
- **Quirks mode detection (модуль `quirks_mode`, HTML5 §13.2.5.1 «The initial insertion mode»):** функция `detect_document_mode(name, public_id, system_id) -> DocumentMode`. Алгоритм: (1) `name != "html"` → `Quirks`; (2) exact-match public_id в [`HTML`, `-/W3C/DTD HTML 4.0 Transitional/EN`, `-//W3O//DTD W3 HTML Strict 3.0//EN//`] → `Quirks`; (3) exact-match system_id `http://www.ibm.com/data/dtd/v11/ibmxhtml1-transitional.dtd` → `Quirks`; (4) public_id начинается (case-insensitive) с одного из ~55 префиксов исторических DTD (IETF HTML 2.0/3.x, Netscape, Microsoft IE 2.0/3.0, Sun HotJava, HTML 4.0 не-4.01 Frameset/Transitional) → `Quirks`; (5) system_id отсутствует AND public_id начинается с `-//W3C//DTD HTML 4.01 Frameset//`/`Transitional//` → `Quirks`; (6) public_id начинается с `-//W3C//DTD XHTML 1.0 Frameset//`/`Transitional//` → `LimitedQuirks`; (7) system_id присутствует AND public_id с HTML 4.01 Frameset/Transitional → `LimitedQuirks`; иначе → `NoQuirks`. Все matches ASCII case-insensitive (spec). Tree builder зовёт detection при первом DOCTYPE-токене (последующие игнорируются как синтаксическая ошибка); при полном отсутствии DOCTYPE — выставляет `Quirks` (§13.2.6.4.1). **Detection-only** — layout/cascade пока не читают `Document.mode`; реальное переключение CSS-поведений (table sizing, body-background propagation, font-size в `<table>`, и т.д.) — отдельные задачи, по мере необходимости.
- **srcset parser + picker (HTML5 §4.8.4.3.5 + §4.8.4.3.7):** модуль `srcset` экспортирует `SrcsetCandidate { url, descriptor }`, `SrcsetDescriptor::{Density(f32), Width(u32)}` + `SrcsetDescriptor::DEFAULT` (= `Density(1.0)`), `parse_srcset(&str) -> Vec<SrcsetCandidate>` и `pick_best_for_density(&[..], dpr)`. Грамматика парсера — упрощённая lenient: candidate-list разделён запятыми (top-level), внутри candidate — `url descriptor?`; descriptor — `Nx` (float density) или `Nw` (integer width), case-insensitive суффикс. Trailing commas в URL (HTML5 §4.8.4.3.5 step «remove all trailing U+002C COMMA characters») трактуются как «descriptor отсутствует». Невалидный descriptor (не x/w, отрицательный, ноль) — кандидат silently отбрасывается. **Density-picker** реализует §4.8.4.3.7 для density-only кандидатов: (1) smallest sufficient density ≥ dpr; (2) fallback на наибольший если все меньше; (3) при равенстве — первый по source-order. Невалидный dpr (≤0 / NaN) трактуется как 1.0.
- **sizes attribute + width-picker (HTML5 §4.8.4.4 + §4.8.4.3.7):** `SizeLength` (Px/Em/Rem/Vh/Vw/Vmin/Vmax/Percent) с `resolve(viewport)` в CSS-пиксели — `em` ≡ `rem` (preload-сканер работает до построения DOM, element-context font-size недоступен); `Percent` резолвится от viewport width (типичная браузерная практика). `SizesViewport { width_px, height_px, root_font_size_px, prefers_dark }` + `SizesViewport::DEFAULT` (1024×768, 16px, light). `Orientation::{Portrait, Landscape}`. `ColorScheme::{Light, Dark}`. `MediaClause` — единичная feature (min-width / max-width / min-height / max-height / orientation / **prefers-color-scheme**). `MediaCondition::{All { negated, clauses }, Unsupported}` — AND-список с опциональным ведущим `not` (Media Queries L4 §3.2 — инвертирует результат _всей_ clause; `Unsupported` под `not` остаётся unknown = false, spec: «If the result is unknown, then the negation also evaluates to unknown»). `SourceSize { condition: Option<MediaCondition>, value: SizeLength }`. **`parse_sizes(input)`** делит по top-level запятым, каждый элемент — либо одиночный length (default-fallback), либо `<media-condition> <length>` (разделение по последнему whitespace — length всегда последний токен без скобок; внутри `()` `" and "` не встречается, поэтому case-insensitive split по `" and "` после lowercase-нормализации даёт правильные клаузы). Невалидный length / неизвестная feature — silent drop элемента. **`evaluate_sizes(sizes, viewport)`** возвращает CSS-пиксели первой матчащей `SourceSize`; fallback на `viewport.width_px` (100vw) если ни одна не sматчила. **`pick_best_for_width(candidates, source_size_px, dpr)`** — effective density `Nw / source_size_px`, дальше тот же алгоритм «smallest sufficient ≥ dpr → fallback на наибольший → tie-break первый-в-списке». Density-кандидаты (`Nx`) в w-picker-е игнорируются: смешивание `Nw`+`Nx` нарушает spec, мы lenient. Невалидный `source_size_px` (≤ 0 / NaN / ∞) → None. **`parse_media_condition`** экспортирован публично — picture-picker использует тот же AST для атрибута `<source media>`; распознаёт ведущий `not` через `strip_leading_not` (требует whitespace или `(`-границу, чтобы `notebook`-как-ident не превращался в keyword). `only` намеренно не поддерживается в `<source media>` / `sizes` (L3-backcompat-модификатор, имеет смысл только в stylesheet-уровневом `@media`).
- **`<picture>` / `<source>` selection (HTML5 §4.8.4.4):** модуль `picture`. `PickedSource { url: String, intrinsic_width: Option<u32>, intrinsic_height: Option<u32> }` — финальный URL выбранного источника плюс author-объявленные intrinsic-размеры в CSS-пикселях (HTML5 §10 «Mapped attributes» — `<source width|height>` / `<img width|height>` integer-атрибуты). Поля для CLS-protection (Cumulative Layout Shift) — layout/shell могут зарезервировать место под коробку ДО загрузки картинки. `parse_dim_attr` парсит HTML5 «rules for parsing non-negative integers»: trim, optional `+`, digit-prefix; отрицательные/percentage/нечисловое → None. MIME-тип и реальные пиксельные размеры (после decode) остаются работой image-pipeline-а. `PictureParams { viewport: SizesViewport, dpr: f32, supported_types: Option<&[&str]> }` — `supported_types=None` отключает type-фильтр (preload-сканер до того, как codec-registry готов; default тоже None). **`pick_picture_source(doc, picture_node, &params)`** — walks дочерних узлов в source-order. Для каждого `<source>`: пропуск без srcset; type-фильтр (case-insensitive lookup в supported_types; пустой `type=""` после trim трактуется как «отсутствует» = match-everything); media-фильтр (через `parse_media_condition` — те же грамматика и AST, что у sizes-условий); если оба прошли — pick через srcset+sizes (width-picker когда есть хоть один `Nw`-кандидат, иначе density-picker), затем `fill_intrinsic_dims` забирает width/height у того же `<source>`. Если ни один source не дал URL — fallback на первый встретившийся `<img>` ребёнок через `pick_img_source` (intrinsic — у `<img>`, не у не-выбранных `<source>`-ов). Misuse (`pick_picture_source` на не-`<picture>`, `pick_img_source` на не-`<img>`) → None. Не резолвит относительные URL — это работа shell (`Url::resolve`).
- **Push-tokenizer + incremental tree builder (P1 п.3B):** `PushTokenizer::feed(&str)` / `end() -> Vec<Token>` — обёртка над тем же pull-Tokenizer-ом, что и iterator-API. Накапливает chunk-и в owned `String`-буфере, на каждом `feed` находит «безопасную точку среза» (`find_safe_split`) и токенизирует до неё, оставляя хвост на следующий вызов. Safe-split учитывает: незакрытый `<tag…`/`</tag…` (терминатор `>`), незакрытый `<!--…-->` (терминатор `-->`, `>` внутри не считается), незакрытый `<!DOCTYPE…>`, незавершённый `&entity;` в data state и в RCDATA (`<title>`/`<textarea>`); в RAWTEXT (`<script>`/`<style>`) entity не декодируется и буферизации не требует. `Tokenizer::with_state(input, text_only)` + `pos()` + `text_only_state()` — пара публичных методов для возобновления state между chunk-ами. Pull-Tokenizer изменён так, чтобы при EOF в text-only loop восстанавливать `text_only`-поле (раньше `.take()` терял state — push-режим переключался в data state посреди RAWTEXT). `IncrementalTreeBuilder { feed(&str), finish() -> Document }` — push-вариант `parse()`-функции: держит `Document` + DOM-стек + `seen_doctype` между вызовами `feed`, прогоняет push-токены через общий с `parse()` `apply_token`-helper. **Инвариант: при одинаковом входе push и pull дают побайтово равный `Document`** — обеспечивается **text-node coalescing**: если последний ребёнок parent-а — text-node, новый `Token::Text` дописывается к нему вместо создания второго text-node (push может разбить непрерывный текст на много токенов по chunk boundary; pull всегда отдаёт цельным куском, для него coalescing — no-op). UTF-8: `feed(&str)` принимает уже валидный UTF-8, caller отвечает за границы code point-ов; `feed_bytes(&[u8])` с буферизацией partial UTF-8 — отдельная задача. Разблокирует P3 п.4B (streaming pipeline shell-side).
- **Preload scanner (HTML LS §13.2.6.4.7 «Speculative HTML parsing»):** модуль `preload_scanner`. `scan_preload_hints(input) -> Vec<PreloadHint>` бежит поверх обычного Tokenizer-а без построения DOM и эмитит spec-овский набор subresource-hints в source-order. `PreloadHint` enum: `Stylesheet { url }` для `<link rel=stylesheet>`, `Script { url }` для `<script src>`, `Image { url, srcset, sizes }` для `<img>`, `SourceSet { srcset, sizes }` для `<source srcset>` (внутри picture/video/audio — `media`-атрибут НЕ учитываем: preload speculative, фетчим все candidate URL pessimistically), `Preload { url, as_kind }` для `<link rel=preload>`, `Preconnect { url, dns_only }` для `<link rel=preconnect>`/`rel=dns-prefetch`. `rel` — multi-token (`rel="preload stylesheet"`), эмитим отдельный hint на каждый известный token; неизвестные `rel`-keywords (icon / canonical / manifest) — silent skip. URL trim-аем (whitespace-only `href` → drop), не валидируем и не резолвим — это работа caller-а (shell через `Url::resolve(base)`). Содержимое `<script>`/`<style>` (RAWTEXT) не парсится как теги (`<img>` внутри литерален — корректно по spec). Caller сканера ещё нет: shell-pipeline сейчас собирает stylesheet-href через DOM-walk; интеграция в streaming pipeline — задача P4.
- **Отложено:** CDATA, полный набор named character references (~2125 имён по HTML5 spec — у нас 250+ самых частых), legacy entities без `;` (HTML4 compat, ~100 имён), insertion modes (in_table, in_select, in_caption, и т.д.), `<iframe>` / `<noembed>` / `<noframes>` / `<noscript>` / `<plaintext>` text modes (нужны при первой реальной странице), application Quirks-mode переключений в layout/cascade (сейчас detection only), L4 nested `(not (...))`-conditions в media-attributes, `calc()`/`min()`/`max()` в length-значениях sizes, `loading="lazy"` через IntersectionObserver (preload + scheduling — задача shell + runtime).

### `lumen-css-parser` 🟡 (полный набор CSS3-селекторов)

- **Готово:** `selector_list { decl_list }` парсинг. Selectors Level 3: simple selectors (`Type`, `Class`, `Id`, `Universal`, `Attribute`, `PseudoClass`, `PseudoElement`), compound (`p.foo#bar:first-child`), complex с combinator-ами (descendant ` `, child `>`, next-sibling `+`, later-sibling `~`). Attribute-операторы: `=`, `~=`, `|=` (для `lang`), `^=`, `$=`, `*=`. **Case-insensitive флаг `[attr=val i]`** (CSS Selectors L4 §6.3.6) — после value распознаётся `i` / `I` (ASCII case-insensitive) или `s` / `S` (явно case-sensitive, default); хранится в `AttrSelector.case_insensitive: bool`; применим ко всем шести операторам. **Structural pseudo:** `:first-child`, `:last-child`, `:only-child`, `:empty`, `:root`, `:first-of-type`, `:last-of-type`, `:only-of-type`, **`:placeholder-shown`** (CSS Selectors L4 §15.1 — типизированный variant, matcher в layout). **Функциональные pseudo:** `:nth-child(an+b)`, `:nth-last-child(an+b)`, `:nth-of-type(an+b)`, `:nth-last-of-type(an+b)` с ключевыми словами `odd` / `even` (хранятся как `NthSpec { a, b }`, `.matches(index)` решает уравнение `i = a*n + b` при `n ≥ 0`). `:not(compound)` — отрицание; запрещены combinator-ы внутри и nested `:not(:not(...))`, такие формы дают `Unsupported`. **CSS4 `:is(selector-list)` и `:where(selector-list)`** — матчат, если матчит хоть один из селекторов; внутри разрешены любые complex-селекторы (combinator-ы, attribute, structural pseudo). Пустой список `:is()` / `:where()` → `Unsupported(name)`. Interactive (`:hover`, `:focus`, …) сохраняются как `Unsupported(name)` и при матчинге всегда возвращают false. Pseudo-elements `::name` парсятся отдельным узлом, никогда не матчат. **Specificity** по CSS3 §16 / CSS4 §17: `:not` сам не считается, contributes specificity внутреннего compound. `:is(...)` сам не считается, contributes максимум specificity по списку. `:where(...)` всегда 0. Прочие pseudo-classes считаются как class. Декларации — как пары строк. Lenient recovery, комментарии `/* */`, пропуск `@`-правил. Парсер `parse_complex_selector` прерывается также на `)`, чтобы корректно работать внутри функциональных pseudo.
- **`!important` флаг (CSS Cascade L4 §8.1):** парсер `extract_important` отделяет `!important` от value (с опциональным whitespace между `!` и словом, ASCII case-insensitive). Не трогает `!important` внутри строковых литералов: `content: "!important"` остаётся value=`"!important"`, important=false. Хранится в `Declaration.important: bool`.
- **CSS-wide keywords (CSS Cascade L4 §7):** `inherit` / `initial` / `unset` / `revert` парсятся как обычные value-строки в css-parser-е (без специальной грамматики); распознаются на этапе `apply_declaration` через `parse_css_wide_keyword(value) -> Option<CssWideKeyword>` (ASCII case-insensitive, trim whitespace). Семантика: `Inherit` → computed value родителя; `Initial` → initial value из спецификации (берётся из `ComputedStyle::root()`); `Unset` → inherit для inherited-свойств, initial для non-inherited; `Revert` в Phase 0 ≡ `Unset` (UA / User origin не отделены чётко). Применение в `apply_css_wide_keyword(style, prop, kw, inherited)` — отдельная функция с per-property branches для всех ~40 поддерживаемых свойств (включая shorthand-ы: margin / padding / border / border-* / border-radius / outline / overflow + counter-reset/-increment). Неизвестные имена — silent no-op (как и для обычных declarations). `inherited: &ComputedStyle` параметр прокинут в `apply_declaration` из `compute_style`. `font-size: inherit/initial/unset/revert` обрабатывается отдельно в `apply_font_size` pre-pass — раньше main-pass, потому что em-basis от font-size зависит.

- **@media queries (Media Queries L4):** `Stylesheet.media_rules: Vec<MediaRule>` хранит `@media`-блоки наряду с обычными `rules`. `MediaRule { query: MediaQuery, rules: Vec<Rule> }`. `MediaQuery.clauses: Vec<MediaQueryClause>` — OR-список (comma-separated). `MediaQueryClause { negated: bool, conditions: Vec<MediaCondition> }` — AND-list внутри одной clause с опциональным ведущим `not`/`only` модификатором; `negated` отвечает за «not» (`only` хранится как `negated=false`, потому что L3-backcompat-модификатор не имеет семантического эффекта в новых браузерах). `MediaCondition` enum: `MediaType("screen"/"print"/"all"/...)`, `Feature(MediaFeature)`, `Unsupported` (forward-compat для неизвестных feature-имён). `MediaFeature` Phase 0: `MinWidth(f32)`, `MaxWidth(f32)`, `MinHeight(f32)`, `MaxHeight(f32)`, `Orientation(Portrait/Landscape)`, `PrefersColorScheme(Light/Dark)`. В layout cascade: после обычных `sheet.rules` итерируется `sheet.media_rules`; для каждого `media.query.matches(ctx)` рулесы добавляются с `rule_idx` продолжающим нумерацию от обычных — это даёт корректное правило «source-position later wins» для @media-rules, но **с упрощением**: все @media трактуются как идущие после обычных, не строго по source-position (Phase 0 ограничение). `MediaContext { media_type, width, height, prefers_dark }` создаётся через `media_context_from_viewport(viewport)` со значениями screen/false по умолчанию — shell может позже расширить. 23 теста (+11): min-width match/no-match, max-width, orientation landscape/portrait, screen type matches, print type doesn't match, AND combination, OR via comma, @media-rule overrides regular, unknown feature doesn't match, prefers-color-scheme: light default-true; **only-screen парсится как no-op, only без media-type → Unsupported, not screen инвертирует match, not all and (min-width: 1000px) инвертит inner, not с Unsupported остаётся unknown, not not screen → Unsupported, OR `not screen, print` — не матчит screen и матчит print, `notepad` НЕ keyword, prefers-color-scheme: light/dark с прокинутым prefers_dark, not all and (prefers-color-scheme: dark) на светлой теме**.

Value читается `parse_value_until_terminator` до `;`/`}` с уважением к строковым литералам, что естественно покрывает `var(--c, fallback)` с запятыми внутри и `rgba(0, 0, 0, 0.5)` в fallback. Substitution `var()` делается уже в layout (см. `lumen-layout`). `!important` для custom properties работает через тот же `extract_important`.
- **CSS `@layer` (CSS Cascade L5 §6.4):** `Stylesheet.layer_order: Vec<String>` хранит cascade-приоритет layer-имён (first = lowest priority), `Stylesheet.layers: Vec<LayerRule { name, rules }>` — rules в layer-блоках. Парсер поддерживает обе формы: statement-form `@layer base, components, utilities;` регистрирует имена без rules; block-form `@layer name { rules }` или анонимный `@layer { rules }` (auto-name `__anon_<n>__`). Если имя встречается в обеих формах или повторно — `layer_order` сохраняет только первое упоминание (idempotent), но `layers` создаёт отдельную запись для каждого блока. `is_layer_name` валидирует CSS-ident (с поддержкой dotted sub-names типа `base.text`). Невалидные имена в statement-form пропускаются; для block-form правило становится анонимным. **Phase 0: только parse + storage**; cascade-интеграция (sort-key с layer-rank, unlayered wins) — отдельная задача (требует расширения compute_style cascade tuple и source-position tracking).
- **CSS `@font-face` (CSS Fonts L4 §4):** `Stylesheet.font_faces: Vec<FontFaceRule>`. `FontFaceRule { family, sources: Vec<FontFaceSource>, weight, style, display, unicode_range }`. `FontFaceSource { kind: FontFaceSourceKind (Url/Local), value, format: Option<String> }`. Парсер `parse_font_face_body` извлекает descriptors: `font-family` (без кавычек), `src` (comma-separated через `parse_font_face_src` — поддерживает `url("...") format("...")` и `local("...")`), `font-weight/-style/-display/unicode-range` (сырые строки — font-matcher разбирает по своему). Без `font-family` правило skip-ается. `split_top_level_commas` — общий helper для split-а с уважением к `(...)` nesting и string-литералам.
- **CSS `@import url(...)` (CSS Cascade L4 §6.5):** `Stylesheet.imports: Vec<ImportRule>`. `ImportRule { url: String, media: MediaQuery }`. Парсер `parse_import_body` распознаёт три формы URL: `url("path")` / `url('path')` / `url(path)` без кавычек, а также bare `"path"` / `'path'`. Опциональный media-query идёт после URL до `;`. Парсер только извлекает URL — fetch и инкорпорация в каскад — задача потребителя (shell), потому что требует сетевой/файловой загрузки относительно base-href. ImportRule экспортирован из `lumen-css-parser`.
- **CSS Properties and Values L1 §1.1 `@property`:** `Stylesheet { rules, properties }` — рядом с обычными правилами хранится `Vec<PropertyRule { name, syntax, inherits, initial_value }>`. `parse_stylesheet` распознаёт `@property` через `parse_at_rule` → `parse_property_body`: ожидает `--name`, читает блок declarations через существующий `parse_declaration_block`, извлекает три обязательных дескриптора (`syntax` — CSS-string в кавычках, через `strip_css_string`; `inherits` — case-insensitive `true`/`false`; `initial-value` — value-строка как есть). Имя без ведущих `--`, отсутствующий `syntax`/`inherits`, или `syntax != "*"` без `initial-value` — @property невалидно, не регистрируется. Повтор имени — обе записи в `Vec`; last-wins реализуется на потребительской стороне. Прочие @-правила (`@media`, `@import`, …) синтаксически пропускаются как раньше.
- **Отложено:** `:not(complex)` со списком селекторов или combinator-ами, namespace prefix в селекторах, типизированные значения деклараций (length / color / calc — типы хранятся в layout, не в parser).

### `lumen-layout` 🟡 (block + inline-flow + replaced + word-wrap + cascade)

- **Готово:** `LayoutBox` дерево, **specificity-based style cascade с !important**: для каждого правила берётся максимальная specificity среди его complex-селекторов, все matched declarations сортируются по `(important, specificity, rule_order, decl_index)` и применяются по возрастанию. `important` идёт первым в ключе — `true > false` ставит !important-декларации в конец, и они выигрывают у normal даже при меньшей specificity (CSS Cascade L4 §8.1). При равенстве — позже объявленная. **Matching complex selector-а — справа налево с back-tracking** (`matches_chain`): для descendant и later-sibling combinator-ов перебираются ВСЕ кандидаты (предки / earlier-siblings) и рекурсивно проверяется суффикс — это корректно решает патологические случаи вроде `.x + a ~ span` с несколькими `a`-siblings без класса `.x`. Для child / next-sibling — один кандидат, без back-tracking. Combinator-ы: descendant / child / next-sibling / later-sibling. **Pseudo-classes:** все CSS3 structural и functional — `:first-child`, `:last-child`, `:only-child`, `:empty`, `:root`, `:first-of-type`, `:last-of-type`, `:only-of-type`, `:nth-child(an+b)`, `:nth-last-child`, `:nth-of-type`, `:nth-last-of-type`, `:not(compound)`. **CSS Selectors L4 `:placeholder-shown`** (§15.1) — matcher `matches_placeholder_shown(doc, node)`: ограничен `<input>`/`<textarea>`, требует непустой (после trim) `placeholder`-атрибут И отсутствие value (`value=""`/нет атрибута; для `<textarea>` — нет non-whitespace текстовых детей через `has_non_whitespace_text`). В Phase 0 без runtime form-state «нет value» = static DOM-проверка, что даёт корректный matching для author-styled статической формы. **CSS4 `:has(rs-list)`** через `matches_relative` — для implicit descendant / `>` / `+` / `~` ищется кандидат в подходящем месте дерева; сам элемент `E` не проверяется (descendants only). Helpers `element_index` (1-based, среди element-sibling-ов) и `element_index_of_type` (среди sibling-ов с тем же тегом) с `from_end` опцией. Attribute selectors: все операторы, с поддержкой ASCII case-insensitive флага (`[attr=val i]`) — сравнение через `eq_ignore_ascii_case` на байтах, чтобы не упираться в char-boundary в UTF-8 строках; не-ASCII (cyrillic) сравнивается побайтово. Наследуемые свойства: `color`, `font-size`, `line-height`, `font-style`, `font-weight`, `font-family`, `text-transform`, `text-align`, `text-decoration-line`. **`FontStyle` (Normal / Italic / Oblique)** через `font-style` property + UA stylesheet для семантических тегов (`<em>`, `<i>`, `<cite>`, `<dfn>`, `<address>`, `<var>` → italic); **`FontWeight` (1..1000, normal=400 / bold=700)** через `font-weight` property с поддержкой keyword-ов (`normal`, `bold`, относительных `lighter`/`bolder` по таблице CSS Fonts L4 §2.4.3) и числовой шкалы; UA stylesheet для `<b>`, `<strong>`, `<th>`, `<h1>`–`<h6>` → bold. **`FontStretch` (CSS Fonts L4 §2.5)** — хранится в десятых долях процента (`FontStretch(1000)` = 100%); 9 keyword-ов (`ultra-condensed` … `ultra-expanded` с дробными `semi-condensed` = 87.5%, `semi-expanded` = 112.5%); численные `%` парсятся и клампятся в [50%, 200%]; inherited; real stretch-варианты требуют variable-font wdth-axis или отдельные fontfiles, в Phase 0 рендерер всегда Inter Regular. `text_rendering_eq` сравнивает font_style, font_weight, font_variant и font_stretch, чтобы italic/bold/small-caps/stretched-фрагменты не сливались с обычными. Реальная отрисовка italic / bold вариантов в paint пока не реализована (нужны Italic / Bold fontfiles или affine skew transform / faux-bold), но layout уже различает. **`font-family`** — `Vec<String>` приоритизированного списка, парсер поддерживает quoted (`"Times New Roman"` / `'Open Sans'`) и unquoted multiword имена со схлопыванием whitespace; generic-family (`serif`, `sans-serif`, `monospace`, …) хранятся как обычные строки. Inherited. Phase 0 рендерер всегда Inter — задел под будущий font matcher. **`TextTransform` (None / Uppercase / Lowercase / Capitalize)** — `text-transform: …` применяется к `InlineSegment.text` при сборке (до wrapping и measurer), используя `char::to_uppercase`/`to_lowercase` стандартной библиотеки — корректно работает для кириллицы. `capitalize` — упрощённо первая буква каждого whitespace-разделённого токена. **`text-indent` (resolved px)** — отступ перед первой строкой inline-content (CSS Text L3 §7.1), inherited; `wrap_inline_run` стартует `current_x = text_indent` для первой строки, последующие — с 0. Поддерживает px/em/rem/vh/vw; `%` пока игнорируется (нужен containing-block-width). **`white-space` (Normal / Nowrap)** — `Nowrap` отключает word-wrap (передаём `f32::INFINITY` как `wrap_width`); `Normal` — обычный greedy wrap; pre/pre-wrap/pre-line отложены (нужен preserved whitespace в input). Inherited. **`opacity` (0..1)** — CSS Color L3 §3.2, не наследуется; парсер принимает число и проценты, clamp вне диапазона; в layout только хранится — реальный alpha-blending paint-уровня — отдельная задача. **`outline` (`outline-width` / `-style` / `-color` + shorthand + `outline-offset`)** — CSS UI L4 §3, не наследуется; **не занимает места в коробке** (в отличие от border) — `rect.width`/`height` неизменны, отрисовка позже как «слой» поверх / снаружи; цвет `None` = currentColor; offset поддерживает отрицательные значения (рисует внутрь). **`visibility` (Visible / Hidden / Collapse)** — CSS Display L3 §4, **inherited** (отличается от display); `Hidden` оставляет коробку в layout (высота сохраняется), но не рисуется — потомок может явно вернуть себя через `visibility: visible`. **`overflow` / `overflow-x` / `overflow-y`** (Visible / Hidden / Clip / Scroll / Auto) — CSS Overflow L3, не наследуется; shorthand принимает 1 или 2 значения; реальный clipping / scrollbars в paint pipeline пока нет. **`text-overflow`** (Clip / Ellipsis) — CSS UI L4 §10.1, не наследуется; работает только в комбинации с overflow != Visible + nowrap; real truncation в paint отложен. **`cursor`** — CSS UI L4 §8.1, inherited; полный набор из 36 standard keyword-ов (auto/default/pointer/text/wait/move/grab/grabbing/all 8 resize-направлений/zoom-in/zoom-out/…); URL-fallback парсер игнорирует и использует последний keyword из comma-list. Использование при mouse-handling — позже. **`box-shadow`** — `Vec<BoxShadow>` (offset_x/y, blur, spread, color, inset), CSS Backgrounds L3 §4.6, не наследуется; парсер `parse_box_shadow_one` собирает токены, балансируя `()` чтобы `rgba(0,0,0,0.5)` не порвался; список через `split_top_level_commas`. Реальная отрисовка теней (Gaussian blur, spread, inset clipping) в paint pipeline — отдельная большая задача. **`text-shadow`** — `Vec<TextShadow>` (offset_x/y, blur, color), CSS Text Decoration L3 §4, **inherited** (отличается от box-shadow); тот же парсер с балансировкой `()`; `none` нужен явно, чтобы откатить родительское inherited значение. **`border-radius`** — 4 поля `border_{tl,tr,br,bl}_radius: f32` (resolved px), CSS Backgrounds L3 §5; shorthand с правилом 1-4 токенов через `expand_border_4` (TL TR BR BL); индивидуальные `border-{corner}-radius`; elliptical-форма `Npx / Npx` берёт только горизонтальный радиус (Phase 0 без vertical-radius); отрицательные значения clamp до 0; не наследуется. Реальный rounded-rect clipping в paint pipeline — отдельная задача. **`letter-spacing` (resolved px)** — дополнительное расстояние между парами символов (CSS Text L3 §11.2), inherited; добавляется в word-width (`(n−1)·ls`) и в word-boundary (`space_w + ls`); может быть отрицательным; `text_rendering_eq` учитывает letter_spacing, чтобы фрагменты с разным spacing не сливались. **`word-spacing` (resolved px)** — дополнительное расстояние **только** между словами (CSS Text L3 §11.3), inherited; добавляется в `gap_with_ls = space_w + ls + ws`; может быть отрицательным; ширина одиночного слова неизменна. Свойства с парсингом: `display` (block/inline/none), `color` (полный CSS3 набор named colors — 147 цветов + `rebeccapurple` из CSS4 §6.1 + `transparent`; `gray`/`grey` варианты эквивалентны; матчинг case-insensitive через бинарный поиск по сортированной таблице `NAMED_COLORS`; hex `#RGB`/`#RRGGBB`/`#RGBA`/`#RRGGBBAA` + `rgb()` / `rgba()` / `hsl()` / `hsla()` с запятыми или whitespace, slash-alpha modern syntax `rgb(r g b / a)`, проценты для каналов, hue в `deg`/`turn`/`rad`/`grad` (CSS Color L4 §9), clamp вне диапазона), `background-color`, `font-size`, `line-height`, `margin` (+ 4 стороны), `padding` (+ 4 стороны), `text-align` (left/center/right), **`text-decoration` / `text-decoration-line`** (комбинируемые keyword-ы `underline` / `overline` / `line-through` / `none`, прочие токены `solid` / `wavy` / `dashed` / `blink` и цвет — игнорируются), **`width` / `height` (px/em/rem; `auto` = не задано)**. `TextDecorationLine` хранится как struct из трёх булевых полей; наследуется на детей через каскад. Whitespace-only текстовые узлы и комментарии пропускаются. **Line wrapping:** `TextMeasurer` trait + `layout_measured()` разбивают текст по словам на строки с реальными шрифтовыми метриками. **Inline-flow:** `BoxKind::InlineRun { segments, lines }` — текстовые узлы и inline-элементы (`<a>`, `<span>`, `<em>`, `<strong>`, и т.д.) группируются в один поток; каждый сегмент хранит свой стиль (цвет ссылки, decoration); слова с одинаковым rendering-стилем сливаются в один `InlineFrag` на строке; `InlineFrag.width` хранит измеренную ширину текста (для align_lines и подрисовки text-decoration в paint). `ComputedStyle::text_rendering_eq` сравнивает color/font_size/line_height/text_decoration_line. `align_lines()` сдвигает `frag.x` после wrap для center/right выравнивания.
- **Готово (тестовая инфра):** `serialize_layout_tree(&LayoutBox) → String` — детерминированный текстовый формат всего layout-дерева (kind / rect / non-default style включая text-align, w=, h=, box-sizing, decoration / segments / lines), плюс 16 golden-тестов в `tests/snapshot_tests.rs` (empty / paragraph / styles / nested / inline + link / line wrap / cyrillic / stacked / display:none / nth-child(odd) / :not(.x) / descendant / underline-on-link / border-solid / border-top / box-sizing border-box). Механизм `UPDATE_SNAPSHOTS=1` для регенерации (как в lumen-paint).
- **Готово (relative units):** `Length { Px, Em, Rem, Percent, Vh, Vw, Vmin, Vmax }` + `parse_length`. Cascade — два прохода: pre-pass для `font-size` (em/% относительно parent fs, rem от `ROOT_FONT_SIZE` = 16, vh/vw/vmin/vmax — от `viewport`), затем main-pass для остального (em/% относительно computed fs текущего элемента). `line-height: 150%` / `1.5em` корректно превращается в коэффициент 1.5; `5vh` / `2vw` тоже поддерживаются. **Viewport units** (CSS Values L3 §6.1.2): `1vh = 1% от viewport.height`, `1vw = 1% от viewport.width`, `vmin = 1% от min(w,h)`, `vmax = 1% от max(w,h)`. Viewport прокинут через `compute_style → apply_declaration → resolve_box_length → Length::resolve` — все четыре функции принимают `viewport: Size`. `%` в margin/padding требует containing-block-width и Phase 0 игнорируется (молча).

Парсинг: `border` shorthand, `border-{side}` per-side shorthand, `border-{width,style,color}` multi-value (1–4 токена по CSS-правилу), `border-{side}-{prop}` individual. `border_*_color: Option<Color>` (None = currentColor). Контент-область корректно уменьшается на ширины border: `content_x/y` учитывают border, `content_width` убирает border_left+border_right. Высота и ширина бокса включают border-widths. `snapshot.rs` выводит `bw=(...)` и `bs=(...)` для ненулевых border.
- **Готово (box-sizing):** `BoxSizing` enum (ContentBox/BorderBox) + поле `box_sizing` в `ComputedStyle`. Парсер `box-sizing: content-box | border-box` (case-insensitive). Не наследуется (CSS Basic UI 3 §4.1) — сбрасывается на ContentBox в каждом `compute_style`. В `lay_out`: при `width`/`height`, заданных в content-box — добавляем padding+border сверху (старая модель), в border-box — `rect.width = w`, `rect.height = h`, контент-область сжимается. Анонимный `InlineRun` тоже сбрасывает box-sizing для чистоты snapshot-а. `snapshot.rs` печатает `box-sizing=border-box` только когда отличается от default.
- **Готово (min/max dimensions):** 4 поля `min_width` / `max_width` / `min_height` / `max_height: Option<f32>` (CSS 2.1 §10.4), не наследуются. None = «нет ограничения» (для min эквивалентно 0, для max — `none`). Парсер: `min-width: auto` и `max-width: none` оставляют None; отрицательные значения отбрасываются спецификацией; px/em/rem/vh/vw поддерживаются (Phase 0 без `%`). В `lay_out` clamp применяется к `rect.width` после `width`-расчёта и к `rect.height` после `height`/content-расчёта; порядок `max сначала, потом min` автоматически даёт правило «при min > max побеждает min». min-/max- интерпретируются в той же box-sizing модели (content-box добавляет padding+border, border-box — нет). Анонимный `InlineRun` сбрасывает все 4 поля.
- **Готово (writing modes):** `Direction` enum (Ltr / Rtl) + поле `direction: Direction` в `ComputedStyle`, **inherited** (CSS Writing Modes L3 §2.1). Парсер `direction: ltr | rtl` (ASCII case-insensitive по Values L4 §2.4); невалидное значение оставляет inherited. В Phase 0 layout только хранит и распространяет через каскад — реальный RTL line-flow / bidi reordering (UAX #9 Unicode Bidi Algorithm) требуют отдельного движка и переписанного `wrap_inline_run`. Точка хранения зафиксирована заранее, чтобы при добавлении `dir`-атрибута / `<bdo>` / bidi не нужно было ретрофитить структуру.
- **Готово (CSS math-функции — CSS Values L4 §10, §10.6, §10.7-10.9):** `Length::Calc(Box<CalcNode>)` хранит AST выражения; `CalcNode::resolve(em, pb, vp) -> Option<f32>` рекурсивно вычисляет в `f32`-пиксели. Поддерживаются: `calc(expr)`, `min(...)`, `max(...)`, `clamp(min, val, max)`, **тригонометрия** (`sin/cos/tan/asin/acos/atan/atan2`), **экспоненциальные** (`pow/sqrt/exp/log/hypot`), **signs/stepping** (`abs/sign/mod/rem/round`). Все 15+ функций обрабатываются унифицированно через `CalcNode::Func(MathFn, Vec<CalcNode>)` + `resolve_math_func` (`f64`-точность для trig/log, результат сужается до f32 с проверкой `is_finite()`; NaN/∞ → None). Angle-units (`deg`/`rad`/`turn`/`grad`) конвертируются в радианы при лексинге и хранятся как `Number(rad)`. CSS spec: `clamp(a, b, c) ≡ max(a, min(b, c))` — при `min > max` побеждает `min`; `mod(a, b)` имеет знак делителя `b`, `rem(a, b)` — знак делимого; `sign(0) = 0` (а не `+1` как std::signum); `log(x)` без второго аргумента = `ln(x)`, `log(x, base)` = логарифм по основанию; `hypot(a, b, ...) = sqrt(a² + b² + ...)`. `round([<strategy>,] val[, step])` — round-to-step по CSS Values L4 §10.5.1; strategy keyword (`nearest` / `up` / `down` / `to-zero`) опционален, ASCII case-insensitive (имена нормализуются в lower-case лексером); отсутствие keyword ≡ `nearest`. Без `step` — step=1. Зашит в `MathFn::Round(RoundStrategy)` — strategy снимается до общего `parse_arg_list`-а в `parse_function_call` (после keyword обязательна запятая). `resolve_math_func` для каждой стратегии делит `val / step` и применяет `round`/`ceil`/`floor`/`trunc` соответственно, умножая обратно на step. Знак step сохраняется (spec не делает abs); деление на 0 → declaration invalid; неизвестный strategy keyword → declaration invalid (ident-без-`(` падает в `parse_calc_factor`). Лексер ident-токенов теперь допускает цифры/дефис после первой буквы (`atan2`, `border-box`-like имена не рвутся). Все 4 базовые функции — top-level value или вложенные внутри `calc()`/`min()`/`max()`/`clamp()` плюс любой math-функции из §10.7-10.9 (в любых комбинациях). Парсер — recursive-descent поверх `tokenize_calc`: top-level value, начинающееся с ASCII-буквы и содержащее `(`, обрабатывается через `parse_math_function_value`. Лексер выдаёт токены `Num(f32, unit)` / `Ident(name)` (case-insensitive, допускает цифры/дефис после первой буквы — для `atan2` и подобных) / `Plus` / `Minus` / `Star` / `Slash` / `LParen` / `RParen` / `Comma`. Грамматика: `expr := term (('+'|'-') term)*`, `term := factor (('*'|'/') factor)*`, `factor := ('-'|'+') factor | Ident '(' arg-list ')' | Num | '(' expr ')'`, `arg-list := expr (',' expr)*`. `parse_function_call` обрабатывает: `calc` (1 expr), `min`/`max` (1+), `clamp` (3), `sin/cos/tan/asin/acos/atan/sqrt/exp/abs/sign` (1), `atan2/pow/mod/rem` (2), `log` (1 или 2), `hypot` (1+), `round` (1 или 2). Неизвестное имя → declaration invalid (None). Унарный `-` — через `0 - factor` рекурсивно (корректно для `calc(-10px + 5px)` и `calc(20px + (-5px))`); унарный `+` — no-op. Поддерживаемые единицы внутри math-функций: те же что у `parse_length` (px/em/rem/vh/vw/vmin/vmax/%) + unitless `Number`. Неизвестная единица (`pt`/`mm`) → declaration invalid. Деление на 0 → `None`. Пустой `min()` / `max()` → None. Интеграция с var(): `var()` разворачивается раньше — `width: min(var(--w), 50px)` работает. Phase 0 ограничения: `line-height: calc(...)` использует общий путь «резолв в px → делим на font-size», что для unitless-чистого calc даст неверный результат (используйте bare `line-height: 1.5`); `inherit`/`initial`/`unset`/`revert` (CSS Cascade) ещё не реализованы — компонент `calc()` ничего не наследует.

HTML5 presentational hints — атрибуты `width`/`height` парсятся как unitless целые пиксели (`parse_html_dimension`) и записываются в `style.width`/`style.height` ДО CSS-каскада, поэтому любое author-CSS правило перекрывает атрибут (HTML5 §10 «Mapped attributes»). - **Готово (CSS Variables L1 — `--name` + `var()`):** `ComputedStyle.custom_props: HashMap<String, String>` хранит resolved-cascade всех custom property declarations. Все custom properties inherited по спеке (`compute_style` копирует `inherited.custom_props.clone()` в init). Каскад — три прохода поверх отсортированного `matched`: font-size pre-pass → **custom-properties pass** (для каждой declaration с `property.starts_with("--")` делает `custom_props.insert(name, value.clone())`) → main-pass. Custom-pass отдельный, чтобы любая обычная декларация в main-pass видела финальное значение custom property независимо от source order (например, `color: var(--c); --c: red` корректно даёт красный). Substitution `var(--name [, fallback])` происходит в `apply_declaration` в самом начале: если `decl.value.contains("var(")` — рекурсивно вызывается `expand_vars(&decl.value, &style.custom_props, 0)`, результат заменяет `val: &str` для оставшегося match-кода. При неудаче (имя не найдено и нет fallback / превышена глубина рекурсии 32 / синтаксис сломан) функция возвращает `None` и `apply_declaration` ранний return — declaration treated as if not present (CSS Variables L1 §3.3 «invalid at computed value time»). `find_var_open` ищет `var(` byte-wise с пропуском строковых литералов (одинарные и двойные кавычки), `parse_balanced_to_close` учитывает вложенные `(`/`)` и строки, `split_var_args` режет аргументы по первой top-level запятой (чтобы `rgba(0,0,0,0.5)` внутри fallback не порвался). Если ни один ancestor не вкладывал — parent = root. Анонимные `BoxKind::InlineRun`/`Skip` пропускаются (то же правило, что в `stacking::box_can_own_stacking_context`) — иначе inline-wrapper под `opacity:0.5` создавал бы фантомный effect-узел. **Mat4 расширен** 2D-builders: `translation_2d(tx, ty)`, `scale_2d(sx, sy)`, `rotate_2d(theta)` (radians), `skew_x(angle)`, `skew_y(angle)`, `from_2d_affine(a, b, c, d, e, f)` (CSS Transforms L1 §13.10), и column-major `multiply` (OpenGL-конвенция: `(lhs · rhs)·p` = `lhs · (rhs · p)`). 22 unit-теста: Mat4 builders + composition + transform-property triggers + opacity/filter/blend/isolation/overflow/clip-path/nested chains + независимость деревьев + skip анонимных run-ов + множественные siblings.

### `lumen-image` 🟡 (PNG + JPEG baseline/progressive + sniff-dispatch)

- **Готово (sniff-dispatch — `decode(bytes)`):** публичный `decode(bytes: &[u8]) -> Result<Image, ImageError>` различает PNG (`89 50 4E 47 0D 0A 1A 0A`) и JPEG (`FF D8 FF`) по первым байтам и вызывает соответствующий декодер. `JPEG_SIGNATURE_PREFIX = [0xFF, 0xD8, 0xFF]` — третий байт `FF` (начало следующего marker по ISO/IEC 10918-1 §B.2.4) делает sniff надёжным; одиночные `FF D8` встречаются в случайных бинарниках. `ImageError` enum: `UnknownFormat` (сигнатура не распознана или вход короче нужного для распознавания) / `Png(DecodeError)` / `Jpeg(JpegError)` + `From`-impls + `Display`. Это отличается от `decode_png(non-png)`, который сразу даёт `DecodeError::InvalidSignature`: общий dispatch более снисходителен и оставляет решение «как реагировать на чужой формат» за caller-ом. Старые `decode_png` / `decode_jpeg` остались — backward compat.
- **Готово (PNG-pipeline):** свой CRC32 (IEEE 802.3 reflected, таблица предкомпилирована `const fn` — без runtime-инициализации, без `LazyLock`); chunk reader по PNG §11.2.2 (4 BE length + 4 type + data + 4 CRC, ограничение length < 2^31, ошибки `BadCrc { kind, expected, actual }`, `ChunkTooLong`); парсер IHDR (13 байтов: width/height u32 BE, bit_depth + color_type + compression/filter/interlace методы, валидация таблицы 11.1 — разрешённые комбинации bit_depth × color_type). `Image { width, height, format, data }` хранится плотно row-major без padding-а.
- **Готово (DEFLATE/zlib):** свой inflate по RFC 1951 + zlib-обёртка по RFC 1950. Три типа DEFLATE-блоков: stored (00, копия после byte alignment), fixed Huffman (01, таблицы 7/8/9 бит из §3.2.6), dynamic Huffman (10, code-length-codes из §3.2.7 + re-run коды 16/17/18). LZ77 со sliding window до 32 КБ через побайтовое копирование (корректно работает на overlapping back-references вроде `distance=1`). zlib header: CMF.CM=8, проверка `(CMF<<8|FLG) % 31 == 0`, FDICT=0. Adler-32 в трейлере. `BitReader` LSB-first; канонический Huffman через ranges-of-codes (`first_code[L]`, `count[L]`, `offset[L]` + `sorted_symbols`) с Kraft-McMillan валидацией при build-е. Bounded allocations: 3 маленьких Huffman-декодера + output Vec.
- **Готово (filter undo):** все 5 PNG-фильтров скан-линий (PNG §9.2): None / Sub / Up / Average / Paeth. Wraparound u8 арифметика; Paeth-предиктор в i16 для корректного сравнения abs. Для первой строки `b = c = 0`, для первых bpp байтов `a = c = 0`.
- **Готово (palette, PNG §11.2.3 + §11.3.2):** `PLTE` парсер — 1..=256 RGB-triples, проверка делимости длины на 3; `tRNS` в палитровом контексте — alpha-таблица до `len(PLTE)` байтов, отсутствующие entries дополняются 255 (opaque); расширение палитровых индексов в `Rgb8` (без `tRNS`) или `Rgba8` (с `tRNS`). Orchestrator валидирует ordering (PLTE до IDAT, tRNS после PLTE), запрещает `PLTE` на color_type 0 / 4 (grayscale), обнаруживает duplicate-чанки и индексы за пределами палитры. `PaletteError` с детализированными вариантами (`MissingForIndexed`, `UnexpectedForGrayscale`, `BadPlteLength`, `PlteOutOfRange`, `TrnsBeforePlte`, `TrnsTooLong`, `DuplicateChunk`, `IndexOutOfRange`). Ограничение Phase 0 — только `bit_depth = 8`; 1/2/4-битная palette требует bit-unpacking и идёт отдельной задачей.
- **Готово (sub-byte depth, PNG §7.2 + §13.12):** новый модуль `png::sub_byte`. `unpack_bits(packed, w, h, bit_depth)` распаковывает скан-линии глубиной 1/2/4 бита в один байт-сэмпл per pixel, **MSB-first** (старший бит — первый сэмпл); число сэмплов в байте = 8/bit_depth; bytes_per_scanline = ceil(W × bit_depth / 8) с trailing-битами в padding'е. Для grayscale `scale_grayscale_to_8bit(samples, bit_depth)` масштабирует: 1-bit × 255 (→ 0/255), 2-bit × 85 (→ 0/85/170/255), 4-bit × 17 (→ n × 0x11). Для palette индексы остаются «as is» — `expand_to_rgb/rgba` принимает диапазон 0..(2^depth-1) при условии, что PLTE имеет достаточно entries. Допустимо только для color_type 0 (grayscale) и 3 (palette) per IHDR §11.2.2 table 11.1 (уже валидируется в `Ihdr::parse`). Для filter::unfilter передаётся `filter_bpp = max(8, channels × bit_depth) / 8`, что для sub-byte даёт 1 байт — соответствует PNG §9.2 «filter unit для sub-byte — 1 byte».
- **Готово (orchestrator `decode_png`):** проверка 8-байтовой сигнатуры `89 50 4E 47 0D 0A 1A 0A`; первый чанк обязательно IHDR; auxiliary-чанки (sRGB / gAMA / pHYs / tEXt / iCCP / cHRM) игнорируются (PNG §11.3 — ancillary safe-to-ignore); `PLTE` / `tRNS` парсятся и валидируются по правилам ordering; IDAT-ы конкатенируются в один zlib-поток; IEND маркирует конец. Поддержаны `PixelFormat::{Gray8, GrayAlpha8, Rgb8, Rgba8}` при `bit_depth ∈ {8, 16}` без interlacing, **palette при `bit_depth = 1/2/4/8`** (формат выхода — `Rgb8` без `tRNS`, `Rgba8` с `tRNS`), **grayscale при `bit_depth = 1/2/4/8`** (выход — `Gray8` после scaling). Interlaced проверяется раньше любой path-specific логики и возвращает `Unsupported(Interlaced)`. `Ihdr::pixel_format()` остался для не-палитровых и не-sub-byte-grayscale случаев — палитра и sub-byte grayscale обрабатываются в orchestrator-е отдельно.
- **Готово (16-bit downsample, PNG §11.2.2 table 11.1):** при `bit_depth = 16` для color_type 0/2/4/6 — `filter_bpp = channels × 2`, `scanline_bytes = width × channels × 2`, фильтры (§9.2) работают над 16-битными байтовыми единицами как обычно. После `filter::unfilter` сэмплы остаются в big-endian u16; функция `downsample_16bit_to_8bit` оставляет только high-byte каждой пары (chunks_exact(2)). Это эквивалент `PNG_TRANSFORM_STRIP_16` в libpng — точность 0xFFFE → 0xFF, не 0xFE; для веб-отображения этого достаточно. Точное округление `(s+128)/257` отвергнуто как усложнение без визуального выигрыша. `Ihdr::pixel_format()` для 16-bit возвращает 8-bit format (Gray8/GrayAlpha8/Rgb8/Rgba8 — одинаково для bit_depth 8 и 16). 16-bit для palette запрещён IHDR §11.2.2 table 11.1 — отбрасывается в `Ihdr::parse()` как `BadBitDepthForColorType`. `UnsupportedReason::SixteenBitDepth` удалён из enum.
- **Готово (tRNS для non-palette, PNG §11.3.2.1):** новый модуль `png::trns_nonpalette`. Для color_type 0 (Grayscale) — `parse_trns_grayscale` читает 2 байта в u16 (gray-sample, считающийся прозрачным); для color_type 2 (RGB) — `parse_trns_rgb` читает 6 байт в (r,g,b) тройку u16. Для color_type 4/6 — спека запрещает tRNS, орхестратор отдаёт `PaletteError::UnexpectedForAlphaType`. tRNS-значения хранятся как 16-битные независимо от bit_depth (спека: range `0..2^bitdepth-1`); функция `normalize_trns_value_to_8bit(raw, bit_depth)` приводит их к 8-битному эквиваленту тем же путём, что используется для сэмплов: 16 → high byte (`>> 8`), 8 → low byte, 1/2/4 → grayscale scale (×255/85/17). Сравнение делается **после** всех трансформаций — это даёт чистый match-on-8-bit, согласованный с потерями downsample. `expand_gray_with_trns(samples, transparent)` производит GrayAlpha8 (alpha=0 при сэмпл==transparent, иначе 255); `expand_rgb_with_trns(samples, (r,g,b))` производит Rgba8 — оба без полутонов (бинарный match). Формат выхода: Gray8→GrayAlpha8 при наличии tRNS, Rgb8→Rgba8 при наличии tRNS. Известное ограничение Phase 0: при 16-bit downsample-е два различных u16 с одним high byte оба считаются match. Точное 16-bit сравнение потребует рефактора с pre-downsample compare — отдельной задачей при первом реальном кейсе.
- **Готово (Adam7-interlacing, PNG §8.2 + §13.10):** `decode_adam7` обрабатывает interlace_method=1 — 7-pass раскладку (`ADAM7_PASSES` const). Алгоритм: per-pass — `adam7_dim(image_dim, start, stride)` вычисляет `pw`/`ph`; пустые passes (`pw==0 || ph==0`) skip-аются (`stream_pos` не двигается); для каждого непустого — `pass_scanline_bytes = ceil(pw * channels * bit_depth / 8)`, `pass_filter_width = pass_scanline_bytes / filter_bpp`, потребляется `(1 + pass_scanline_bytes) * ph` байт inflate-потока; вызывается тот же `filter::unfilter`; результат прогоняется через `SampleTransform::apply` (sub-byte unpack+scale или 16-bit downsample) — общая логика с non-interlaced. Финальная раскладка: пиксель `(px,py)` sub-image → `(start_col + px*col_stride, start_row + py*row_stride)` в `out[]`. После последнего pass-а — strict tail check: лишние байты в IDAT → `BadImageDataSize`. `SampleTransform` (struct с `is_sub_byte`/`is_16bit`/`is_grayscale`/`bit_depth`) выделен как захват общих параметров пост-unfilter transformations. Pipeline: при `header.interlaced` идёт через `decode_adam7`, иначе — однократный `unfilter + xform.apply`. Palette/tRNS expand работает поверх собранного samples-буфера одинаково для обоих путей. `UnsupportedReason::Interlaced` удалён.
- **Готово (JPEG baseline + progressive, ISO/IEC 10918-1):** модуль `jpeg/` с собственным декодером baseline DCT (SOF0) **и progressive DCT (SOF2)**. API: `decode_jpeg(bytes) -> Result<Image, JpegError>` — auto-dispatch по SOF marker. Формат выхода: `Gray8` для 1-компонентных файлов, `Rgb8` для 3-компонентных YCbCr. Никаких сторонних crate-ов (`jpeg-decoder` / `image` отвергнуты по §5). Подмодули:
  - **`marker.rs`:** reader marker-segments (`0xFF NN` + optional 2-байтная BE длина включая саму длину). Парсит SOI/EOI/SOF0/**SOF2**/DHT/DQT/SOS/DRI/APPn/COM/RSTn; пропускает fill-байты `0xFF` между маркерами (§B.1.1.2). SOF1/3/5-7/9-11/13-15 (extended/lossless/hierarchical/arithmetic) → `UnsupportedSof`; DAC (arithmetic coding) → `UnsupportedFeature`; DNL → `UnsupportedFeature` (редкость). `Frame { width, height, components, h_max, v_max, is_progressive }` — флаг устанавливается по SOF marker-у (SOF0=false, SOF2=true). **`ScanInfo { components, ss, se, ah, al }`** — параметры одного scan-а после SOS. Baseline: `Ss=0, Se=63, Ah=Al=0`, `Ns=Nf` (interleaved). Progressive: DC scan `Ss=Se=0` может быть interleaved или non-interleaved (Ns=1); AC scan `1≤Ss≤Se≤63` всегда Ns=1; `Al` — successive approximation bit-shift; `Ah=0` initial, `Ah=Al+1` refinement. **`read_next_segment_after_scan(ctx)`** — entry для progressive multi-scan loop: между scan-ами обрабатывает промежуточные DHT/DQT/DRI и возвращает `NextSegment::Scan(scan_info)` или `Eoi`. **ZIGZAG[64]** const, **`parse_dqt`** для 8/16-bit таблиц (values хранятся уже в natural order — de-zigzag на парсинге), **`parse_dht`** для до 4 DC + 4 AC Huffman tables. Sanity sum(Hi×Vi) ≤ 10. 19 unit-тестов.
  - **`huffman.rs`:** canonical Huffman build (Annex C). `bits[L-1]` — число кодов длины L. Сначала длина L1: коды 0, 1, …, bits[L1-1]-1; переход к L+1: `code = (last + 1) << 1`. Хранение через `min_code[L]`, `max_code[L]`, `value_offset[L]`. Декод бит-по-биту: `code = (code << 1) | next_bit; if code <= max_code[L]: return values[value_offset[L] + (code - min_code[L])]`. Kraft-McMillan валидация при build-е (`max_code >= 2^L+1` → `BadHuffmanCodes`). 4 unit-теста.
  - **`bit_reader.rs`:** MSB-first битовый ридер с byte-stuffing. На `0xFF` — peek следующего байта: `0x00` → возвращает `0xFF` как data; иначе marker (RSTm/EOI/DNL/etc), сохраняется в `self.marker` и refill прекращается. `read_bit()` / `read_bits(n)` (n ≤ 16). `read_restart_marker()`: на границе restart-интервала сбрасывает буфер и возвращает marker (либо из `self.marker` если refill уже его видел, либо читая `FF Dn` напрямую — нужно, потому что данных в buffer могло хватить и refill не запускался). 8 unit-тестов.
  - **`idct.rs`:** прямой 2D IDCT 8×8 с фиксированной точкой ×1024. Сначала row-pass, потом column-pass. Cos-таблица предкомпилирована (`C[0..8]` — округлённые `cos(kπ/16) × 1024`). Финальный сдвиг: `(acc + 0x400) >> 11` (учитывает `1/2` из формулы IDCT и `1/1024` денормирование). Level shift +128 и clamp в `0..=255`. 4 unit-теста (DC-only → uniform block, zero block → 128, large DC clamps).
  - **`color.rs`:** YCbCr → RGB по ITU-R BT.601 (JFIF §7). Коэффициенты × 65536 для целочисленной арифметики: `91881`, `22554`, `46802`, `116130`. Clamp в `0..=255`. 4 unit-теста.
  - **`scan.rs`:** baseline entropy-coded scan loop. Per-component grids ширины `mcus_per_row × Hi × 8` пикселей. DC predictors per-component. Для каждого MCU декодируем `Hi × Vi` блоков per component: `decode_block` через Huffman DC (size → bits → EXTEND), потом AC (RS-byte = run<<4|size; EOB=0x00, ZRL=0xF0, иначе skip `run` + value), de-zigzag, dequant (умножение на de-zigzagged QT), IDCT + level-shift + clamp. **Chroma upsampling** — nearest neighbour по формуле `(x × h_i / h_max, y × v_i / v_max)`. **Restart intervals** (DRI): каждые `restart_interval` MCU читаем RSTm, проверяем циклический счётчик 0..=7, сбрасываем DC predictors. Финальная сборка обрезает grid до `width × height` (если не кратно MCU). 1 unit-тест.
  - **`progressive.rs`:** multi-scan progressive decoder (§G). Аллоцирует coefficient buffers per-component (`Vec<i32>` размера blocks_h × blocks_v × 64) — все scan-ы пишут в них. Главный loop: `decode_one_scan` → `SegmentReader::read_next_segment_after_scan` → repeat → EOI → `finalize_pixels` (dequantize + IDCT + YCbCr→RGB upsample). Поддержаны 4 типа scan: **DC initial** (Huffman size → bits → EXTEND → `dc_pred += delta` → `block[0] = dc_pred << Al`), **DC refinement** (`block[0] |= bit << Al`), **AC initial** (обычный RLE+EOBn, значения `<< Al`; EOBn-extension: `eob_run = (1<<r) + read_bits(r) - 1` блоков skip), **AC refinement** (§G.1.2.3 — самый тонкий: refine 1 битом существующие non-zero коэффициенты, новые non-zero вставляются между ними после пропуска `run` zero-positions; ZRL = пропустить 16 zeros; EOBn = войти в EOB-mode для текущего блока и последующих). Non-interleaved AC scan: MCU = 1 блок (проход по всем блокам компонента). Interleaved DC: MCU как в baseline, внутри MCU — Hi×Vi блоков на компонент. Restart intervals работают так же. 2 unit-теста (extend, block indexing).
- **Отложено:** arithmetic coding (SOF9-11 + DAC), lossless JPEG (SOF3), hierarchical (SOF5-7), 12-bit precision, CMYK (4-компонентные), ICC color profiles из APP2 (Phase 0 пропускает все APPn), WebP / AVIF (Phase 2+).
- 130 unit-тестов (PNG: 90; JPEG: 31 — base 24 + 6 progressive marker/parse + 2 progressive helpers; sniff-dispatch: 9) + 57 интеграционных (PNG: 33; JPEG: 18 — solid 4:4:4 red / 4:2:0 blue / 4:2:2 green / gray dark+light / 17×9 non-MCU-aligned / gradient_v 32×32 grayscale / gradient_rb 32×16 RGB / restart_interval 24×24 / **progressive solid 4:4:4 red / 4:2:0 blue / grayscale** / **progressive gradient v + rb** / **progressive DC refinement через jpegtran** / rejected SOF3 lossless / garbage / truncated; sniff-dispatch: 6).

### `lumen-storage` ✅ (in-memory + SQLite + Cookies + History + Bookmarks + HttpCache + Profiles + SearchProviders + Downloads + Permissions + Workspaces + TabSessions + Plugins + SearchHistory + SiteEngagement + DnsCache + ReferrerPolicies + Autofill + Notifications + SafeBrowsing + PslProvider)

- **Готово (InMemoryStorage):** `HashMap<PartitionedKey, Vec<u8>>` с полным origin-партиционированием: каждый вызов принимает `origin: Option<&str>` и `top_level_site: Option<&str>`. `None` и `""` — один namespace (глобальный профиль). Реализует `lumen_core::ext::StorageBackend` (get/put/delete/list_keys). Snapshot-формат `LUMEN_KV_V1` — текстовый, hex-encoded composite key + hex-encoded value, без внешних зависимостей. `serialize()` / `deserialize()` для in-memory round-trip; `save(path)` / `load(path)` для диска. Подходит для тестов и ephemeral session-state.
- **Готово (SqliteStorage):** persistent KV поверх **SQLite** (exception #5 в §5). Та же сигнатура `StorageBackend`, та же семантика origin-партиционирования (`None` и `Some("")` — один namespace). Одна таблица `kv (origin TEXT, top_level_site TEXT, key TEXT, value BLOB, PRIMARY KEY (origin, top_level_site, key)) WITHOUT ROWID`; `INSERT ... ON CONFLICT DO UPDATE` для put (upsert). WAL + `synchronous = NORMAL` по умолчанию — стандартный single-writer durability compromise: после crash-а commit-ы сохраняются, но без `fsync` на каждый. `Connection` обёрнут в `Mutex` (rusqlite::Connection не-Sync), что даёт `Send + Sync` на `SqliteStorage` для соблюдения trait bound. `prepare_cached` для get/list_keys — экономит парсинг SQL на hot-path. Конструкторы: `open(path)` для файла, `open_in_memory()` для ephemeral БД (тесты). Через `rusqlite` с feature `bundled` — SQLite компилируется в бинарь, нулевая runtime-зависимость от системного libsqlite3. `Error::Storage(String)` добавлен в `lumen-core::Error` для детальных сообщений SQL-движка.
- **Готово (parse_set_cookie + parse_set_cookie_with_psl):** парсер HTTP-заголовка `Set-Cookie` в `Cookie`-структуру (RFC 6265 §5.2 + RFC 6265bis §5.5). Сигнатуры `parse_set_cookie(header, default_domain, default_path, now_unix) -> Option<Cookie>` (без PSL — fail-open) и `parse_set_cookie_with_psl(header, default_domain, default_path, now_unix, psl: Option<&dyn PublicSuffixList>) -> Option<Cookie>` (с public-suffix защитой). Атрибуты (case-insensitive имена): `Expires` (RFC 1123 формат «Wed, 21 Oct 2015 07:28:00 GMT»; опционально без day-of-week-prefix), `Max-Age` (приоритетнее Expires; expires_at = now+N; отрицательное → expired immediately), `Domain` (leading-dot strip + lowercase + domain-match check с default_domain — RFC 6265 §5.3 step 4 — независимо от PSL; пустой после strip-а игнорируется), `Path` (должен начинаться с `/`, иначе fallback на default_path), `Secure` / `HttpOnly` (без значения), `SameSite=Strict|Lax|None` (default Lax). **PSL-проверка** (RFC 6265bis §5.5 step 5): если Domain= совпадает с known public suffix и не равен default_domain → cookie отвергается (`None`) — блокирует super-cookie attacks вида `Set-Cookie: tracker=1; Domain=co.uk`; если совпадает и равен default_domain (прямое посещение `co.uk`) → cookie host-only. Без PSL шаг public-suffix пропускается, шаги domain-match + invalid name остаются. Невалидные cookies (без `=` или с пустым name, или domain-mismatch) → `None`. Конверсия RFC 1123 даты в Unix timestamp через алгоритм Howard Hinnant `civil_to_unix(y, m, d, hh, mm, ss)` (учитывает leap years через формулу `(153*m_adj + 2)/5 + d - 1`). 19 unit-тестов (+8): basic name=value, full attribute parsing, Max-Age overrides Expires, RFC1123 parse + без day-of-week, Max-Age negative → expired, SameSite variants (Strict/Lax/None/garbage/missing), invalid returns None, empty value OK, Path must start with `/`, case-insensitive attribute names, civil_to_unix (epoch/Y2K/leap year), cross-origin Domain rejected, PSL super-cookie reject, PSL bare TLD reject, self-as-public-suffix host-only, legitimate registrable domain, PSL=None fail-open, domain-match independent of PSL, IDN Punycode super-cookie reject.
- **Готово (CookieJar):** RFC 6265 + RFC 6265bis (SameSite) поверх SQLite. Структура `Cookie { domain, path, name, value, expires_at: Option<i64>, secure, http_only, same_site: SameSite }`; `SameSite { Strict, Lax, None }` с default = Lax. Таблица `cookies(top_level_site, domain, path, name, value, expires_at, secure, http_only, same_site)` с composite PK + индекс `(domain, top_level_site)` для быстрого host lookup. API: `set` (upsert), `delete`, `clear_expired(now_unix) -> count`, `clear_session() -> count`, `get_for_request(host, path, is_secure, now_unix, top_level_site)`. Domain-match по RFC 6265 §5.1.3 (exact или suffix с leading-dot между host и cookie_domain); path-match по §5.1.4 (equality или prefix с разделителем `/`). Domain хранится lowercase (RFC §5.3.1). Secure cookies отдаются только при `is_secure=true`. Expired cookies (`expires_at < now`) фильтруются на чтении; session cookies (`expires_at = NULL`) хранятся пока `clear_session` не позовут. **Total cookie protection** (§9.2): партиционирование по `top_level_site` — третьесторонний cookie для одного домена-источника невидим из другого top-level контекста. Конструкторы `open(path)` / `open_in_memory()`. Только storage layer — `Set-Cookie` HTTP-парсер и интеграция с `lumen-network` — отдельные задачи.
- **Готово (Autofill):** saved form values per-(origin, field_name, value) с frequency-counter для autocomplete UI. `AutofillEntry { origin, field_name, value, frequency, last_used }`. Composite PK по (origin, field_name, value) — повторное submit-нутое значение инкрементит frequency, не дублирует. Index по (origin, field_name) для быстрого suggestions-lookup. API: `record(origin, field_name, value, now)` (empty value silently skipped); `suggestions(origin, field_name, limit)` ORDER BY frequency DESC + last_used DESC; `best_for(origin, field_name)` — top-1; `delete(origin, field_name, value)`; `clear_origin(origin)` для clear-site-data; `clear()`; `count()`. 10 unit-тестов.
- **Готово (Notifications):** Web Notifications API persistence. `Notification { id, origin, title, body, icon_url, tag, click_url, shown_at, dismissed_at, clicked_at }`. Таблица с индексами `(origin, tag)` и `shown_at DESC`. API: `show(origin, title, body, icon, tag, click_url, shown_at)` — если tag непустая и для (origin, tag) есть **активная** notification (не dismissed/clicked) → replace in-place (update); иначе insert. `mark_dismissed(id, now)`, `mark_clicked(id, now)` — устанавливают timestamps. `get(id)`, `active(limit)` (фильтрует dismissed/clicked), `history(limit)` (включает закрытые), `delete(id)`, `delete_older_than(before)`, `count()`. 14 unit-тестов.
- **Готово (ReferrerPolicies):** per-origin Referrer-Policy настройки (W3C Referrer Policy spec). `ReferrerPolicy` enum — 8 значений: `NoReferrer`, `NoReferrerWhenDowngrade`, `Origin`, `OriginWhenCrossOrigin`, `SameOrigin`, `StrictOrigin`, `StrictOriginWhenCrossOrigin` (default, как в современных браузерах), `UnsafeUrl`. `parse(s)` case-insensitive; empty string → default. `as_str()` round-trip. Таблица `referrer_policies(origin PK, policy, updated_at)` WITHOUT ROWID. API: `set(origin, policy, now)`, `get(origin) -> Option`, `get_or_default(origin)` (fallback на StrictOriginWhenCrossOrigin), `delete`, `list_all()` (ORDER BY origin ASC), `count()`. Реальный matcher (`origin → policy`) + apply при HTTP request — задача lumen-network. 14 unit-тестов.
- **Готово (SearchHistory):** recent omnibox queries для autocomplete. `SearchQuery { id, query (case-preserved), normalized (lowercase), frequency, last_used, first_used }`. Таблица `search_queries(id PK, query, normalized UNIQUE, frequency, last_used, first_used)` + индексы по last_used DESC и frequency DESC. API: `record(query, now)` (upsert: trim, lowercase для dedup; на конфликте — frequency++ и last_used MAX), `recent(limit)` (по last_used DESC), `popular(limit)` (по frequency DESC, tie-break last_used DESC), `prefix_match(prefix, limit)` (case-insensitive с escape_sql_like для `%`/`_`/`\`), `delete_query(normalized)`, `delete_older_than(before)`, `clear()`, `count()`. Empty queries после trim — silently skipped. 15 unit-тестов.
- **Готово (SiteEngagement):** per-origin engagement metrics для omnibox ranking. `SiteEngagement { origin, visit_count, total_time_seconds, last_visit, first_visit }`. Таблица `site_engagement` с PRIMARY KEY (origin) WITHOUT ROWID + индекс по last_visit DESC. API: `record_visit(origin, now)` (upsert: insert или visit_count++/MAX(last_visit)), `add_time(origin, seconds)` (clamp в 0..). `score(now, half_life_days) -> f64` — engagement score с exponential decay: `(visit_count + total_time_seconds/300) * 0.5^(age_days/half_life)` (5 минут на сайте ≈ 1 визиту). `top_by_score(now, half_life, limit)` — Vec<(SiteEngagement, score)> отсортированный DESC by score (читает всё в Rust и сортирует; для <10к origin-ов мгновенно). `delete(origin)`, `count()`. 12 unit-тестов.
- **Готово (DnsCache):** hostname → IP-адреса с TTL. `DnsEntry { hostname, addresses: Vec<String> (IPv4/IPv6 string-form), cached_at, expires_at }`. `is_fresh(now)` — true если `now < expires_at`. Таблица `dns_cache(hostname PK)` WITHOUT ROWID, addresses хранятся как comma-separated string (для multiple-A round-robin DNS). API: `put(hostname, addresses, cached_at, ttl_seconds)`, `get(hostname, now)` — фильтрует expired автоматически, возвращает None если истёк. `delete(hostname)`, `clear_expired(now)`, `clear()`, `count()`. Empty addresses (`&[]`) сохраняются — NXDOMAIN negative cache. Поддерживает Punycode-домены (`xn--…`). 11 unit-тестов.
- **Готово (CachedDnsResolver):** реализация `lumen_core::ext::DnsResolver` поверх `DnsCache`. Лежит в lumen-storage (а не lumen-network) — потому что зависит от SQLite-кеша; обратный путь «lumen-network → lumen-storage» потянул бы SQLite в любой бинарь с одной только сетью. Конструируется `CachedDnsResolver::new(inner: Arc<dyn DnsResolver>, cache: Arc<DnsCache>, default_ttl_seconds)` и подключается shell-ом в `HttpClient::with_dns_resolver(...)`. На каждый `resolve(host, port)`: (1) `cache.get(host, now)` → если fresh AND есть хоть один валидный IP-string → возврат `Vec<SocketAddr>` с port-подстановкой (порт НЕ кэшируется, чтобы один host-entry переиспользовался для resolve(host, 80) и resolve(host, 443)); невалидные IP-строки (corrupt-cache) тихо отбрасываются — если итоговый список пустой, fallback на inner. (2) miss/expired/corrupt → `inner.resolve()` → `cache.put(host, addr_strings, now, default_ttl_seconds)`. Пустой `Err` от inner в кэш не пишется (negative-caching отложен — слишком легко закэшировать эфемерный network blip). `Clock` trait (+ `SystemClock` default, `MockClock` в тестах) подменяет источник времени — иначе expiration не протестировать детерминированно. 11 unit-тестов: miss→hit, expiration, multi-IP, IPv6, port substitution, inner Err propagation, corrupt-cache fallback, object-safety, panic-on-second-call (доказательство что cache hit полностью пропускает inner), independent hostnames, monotonic SystemClock.

Таблица `plugins` + индекс по `enabled`. Capabilities хранятся как сырая JSON-строка (`["network", "clipboard", "storage:knowledge"]` etc.) — типизированный CapabilityToken enum в `lumen-core::ext` пока future. Runtime-проверка capability при вызовах плагина — задача `lumen-shell` + wasmtime sandbox (§11.2). 14 unit-тестов.

### `lumen-network` ✅ (HTTP/1.1 + HTTPS)

- **Готово:** `HttpClient` реализует `NetworkTransport` из `lumen-core::ext`. Поддержка HTTP и HTTPS (rustls + webpki-roots, exception #3). Redirect-следование до 5 хопов (абсолютные + относительные `Location`). `chunked` Transfer-Encoding decoder. URL-парсинг (scheme/host/port/path), case-insensitive заголовки. Box-обёртка вокруг TLS stream (clippy large-enum-variant). **IDN-домены** конвертятся в Punycode на этапе `parse_url` через `lumen_core::idn::domain_to_ascii` — DNS lookup, TLS SNI (`ServerName::try_from`) и `Host:` header (RFC 7230 §5.4) всегда получают ASCII-форму.
- **Готово (EventSink — принцип №4):** `HttpClient::with_sink(Arc<dyn EventSink>)` + `with_tab(TabId)` — fluent builder. `fetch_with_redirect` эмит `Event::RequestStarted { tab_id, url }` **после** успешного `parse_url` (bad scheme → ни Started, ни Completed) и перед TCP-сокетом; `Event::RequestCompleted { tab_id, url, status }` — после чтения статус-строки, **до** анализа кода ответа (2xx / 3xx / 4xx — все эмитятся как completed). Каждый редирект-hop генерит свою пару (Started/Completed), не одну на цепочку. Инвариант «Started без Completed = network failure» (DNS / refused / TLS handshake error прерывают между ними); явный `RequestFailed` добавим, когда наблюдателям станет мало. `sink: Option<Arc<dyn EventSink>>` — по умолчанию None, sink не дёргается совсем (не нужен hot-path NoopEventSink для типичного случая «события никому не нужны»).
- **Готово (RequestFilter — место для фильтров):** `HttpClient::with_filter(Arc<dyn RequestFilter>)` — fluent builder, по умолчанию `Option::None` (фильтра нет, fetch всегда уходит в сеть). При подключённом фильтре `fetch_with_redirect` зовёт `f.should_block(&url)` **после** `parse_url` и **до** `RequestStarted`/TCP: блокированный запрос НЕ эмитит Started/Completed, эмитит `Event::RequestBlocked { tab_id, url, reason }` (если sink подключён) и возвращает `Error::Network("blocked: {reason}")`. Каждый redirect-hop проверяется независимо — фильтр блокирует переход с нейтрального адреса на трекер, а сам исходный URL пропускается. На bad scheme (`ftp://...`) фильтр не зовётся — невалидный URL отбрасывается до фильтр-чека (тест-сторож `PanicOnCheck`).
- **Готово (HTTP/1.1 keep-alive + ConnectionPool):** persistent `Connection` хранит `BufReader<RawStream>` (не пересоздаётся между запросами — иначе входной буфер с остатком предыдущего ответа уходил бы в drop) плюс флаг `closed`; `read_response(&mut Connection) -> Response` не consume-ит соединение. Request теперь шлёт `Connection: keep-alive` (явно, хотя HTTP/1.1 default). `read_chunked` **дочитывает trailer-секцию** до финального `CRLF` — без этого второй запрос на pooled-сокете читал бы хвост от предыдущего ответа как новый status-line. `ConnectionPool` (`pool.rs`) — `Mutex<HashMap<(host, port, is_tls), Vec<Entry { conn, returned_at }>>>`, LIFO, **`IDLE_TIMEOUT = 30 с`** (короче серверных дефолтов 60–75 с с запасом), **`MAX_IDLE_PER_HOST = 6`** (соответствует браузерному дефолту), stale на acquire — drop без use. `Connection: close` от сервера / EOF / ошибка чтения — `closed = true`, в пул не возвращается. `HttpClient::with_pool(Arc<ConnectionPool>)` для shared-пула между клиентами; по умолчанию у каждого свой собственный. **Retry-on-stale:** `is_stale_error` распознаёт `BrokenPipe` / `ConnectionReset` / `ConnectionAborted` / `UnexpectedEof` в `Debug`-формате `io::Error` плюс наши «EOF before status line» / «EOF in headers». При попадании на pooled connection, который сервер успел закрыть, fetch автоматически делает один retry на свежем connect-е.
- **Готово (DnsResolver hook):** `HttpClient::with_dns_resolver(Arc<dyn DnsResolver>)` подменяет default `SystemDnsResolver` (поверх `(host, port).to_socket_addrs()` из std). `connect(host, port, is_tls, resolver)` теперь двухэтапный: (1) `resolver.resolve` даёт `Vec<SocketAddr>`; (2) пробуем `TcpStream::connect(&addr)` по каждому до первого успешного — корректно для DNS round-robin и IPv4+IPv6 dual-stack. Resolver зовётся per redirect-hop, симметрично с sink/filter (один URL → одно резолв-обращение). `SystemDnsResolver` — единственный default в lumen-network; `CachedDnsResolver` (lumen-storage, см. ниже) и `DohResolver` (см. ниже) — подключаются shell-уровнем через тот же builder.
- **Готово (DohResolver — RFC 8484):** `lumen_network::DohResolver` реализует `DnsResolver` поверх произвольного `NetworkTransport` (типично — собственный `HttpClient`). На каждый `resolve(host, port)` шлёт два GET-запроса к DoH endpoint-у — AAAA (QTYPE=28) и A (QTYPE=1) — и объединяет результаты (IPv6 первым, дальше IPv4 — RFC 6724 §6 dual-stack preference). Wire format запроса/ответа — собственный encoder/decoder для DNS message (RFC 1035 §4): header c флагами QR/Opcode/AA/TC/RD/RA/Z/RCODE + question section (length-prefixed QNAME labels, QTYPE, QCLASS=IN) + skip question + parse answer records с поддержкой compression pointers (§4.1.4, top-2-bits `11`). CNAME-цепочки пропускаются, финальные A/AAAA RDATA конвертятся в `Ipv4Addr` / `Ipv6Addr`. RCODE≠0 (NXDOMAIN/SERVFAIL/…) и TC=1 (truncated) → Err. **DoH-обёртка по RFC 8484 §4.1:** GET с `?dns=<base64url(query)>` — собственная реализация base64url-кодировки без padding (RFC 4648 §5, алфавит `+→-`, `/→_`). POST намеренно не используется — текущий HttpClient только GET, и base64url-encoded DNS query (~50–100 байт wire → ~70–140 base64url-символов) укладывается в любой URL-лимит. **Bootstrap-цепочка** документирована в doc-комментарии: внутренний HttpClient у DohResolver должен резолвить host endpoint-а через НЕ-DohResolver (`SystemDnsResolver` или IP-литерал в endpoint URL — `https://1.1.1.1/dns-query`), иначе бесконечная рекурсия. Литералы IPv4 (`8.8.8.8`), IPv6 (`::1`) и bracketed IPv6 (`[::1]`) обрабатываются локально без обращения к DoH. TTL из DNS-ответа игнорируется — кеш сверху (`CachedDnsResolver`) ставит свой TTL.
- **Готово (DotResolver — RFC 7858):** `lumen_network::DotResolver` реализует `DnsResolver` поверх собственного TCP+TLS-сокета (rustls, exception #3), без HTTP-обёртки. Конструируется `DotResolver::new(server_name, server_addr)` — `server_name` для TLS SNI и cert-валидации, `server_addr` — pre-resolved IP+порт DoT-сервера; фабрики `cloudflare()` / `google()` / `quad9()` зашивают hardcoded IP-литералы `1.1.1.1:853` / `8.8.8.8:853` / `9.9.9.9:853` для bootstrap без system DNS. Дефолтный порт 853 (RFC 7858 §3.1) экспортирован как `DOT_DEFAULT_PORT`. **Bootstrap-разделение** server_name/server_addr решает циклическую проблему «чтобы зарезолвить DoT-сервер по hostname, нужен резолвер»: caller передаёт IP уже извне (system resolver на старте, IP-литерал, или кэш). На каждый `resolve(host, port)` шлём ДВА запроса — AAAA + A последовательно, объединяем (IPv6 первым — RFC 6724 §6); каждый query = свежий TLS handshake (Phase 0 simplification, RFC 7858 §3.4 разрешает persistent — отложено отдельной задачей). Wire-format переиспользует `doh::encode_query` / `doh::decode_answer_ips` (переведены `pub(crate)` → `pub` под reuse); сверху — собственный TCP framing (RFC 1035 §4.2.2: `[u16 BE length][message]`) через `frame_query` / `read_framed_message`. `query_over_stream<S: Read+Write>` — низкоуровневая функция exchange-а, написана generic над stream-ом, чтобы тестироваться через `Cursor<Vec<u8>>` mock без поднятия TLS (TLS-handshake протестировать в unit-test нельзя без реального сервера). `rustls::ClientConfig` с webpki-roots кэшируется в `OnceLock` — построение config-а не бесплатно, а DoT вызывается часто. Литералы IPv4 / IPv6 / bracketed `[::1]` обрабатываются локально, без обращения к серверу (как в DoH). RCODE≠0 / TC=1 → Err. TTL игнорируется (кеш сверху).
- **Готово (HSTS enforcement — RFC 6797 end-to-end):** `HttpClient::with_hsts(Arc<dyn HstsEnforcement>)` подключает HSTS-store. Trait-граница `lumen-core::ext::HstsEnforcement` (рядом с DnsResolver / EncodingDetector / EventSink) позволяет lumen-network не зависеть от lumen-storage напрямую; реализация — `lumen-storage::hsts::HstsStore`. Без `Result`, fail-open (ошибка storage логируется eprintln, трактуется как «нет HSTS») — HSTS soft policy, не должна ронять fetch. Модуль `lumen-network::hsts` содержит pure-функции `maybe_upgrade_url_to_https(&dyn HstsEnforcement, url, now_unix) -> Result<Option<Url>>` и `process_sts_response(&dyn HstsEnforcement, scheme, host, headers, now_unix)`. **Pre-request** (`fetch_with_redirect` после hops_left-check, ДО `require_http_scheme`, фильтра и Started): если `url.scheme == "http"` и `store.is_https_only(host_ascii, now)` — переписываем на `https://`. RFC 6797 §8.3: явный `:80` убирается (станет default 443), custom-port (`:8080`) сохраняется, path/query/fragment как были. Filter и observer-ы видят уже-upgraded URL — блок-листы могут не пропустить https-вариант, network log показывает реальный трафик. **Post-response** (после `fetch_single`, до анализа status): парсится `Strict-Transport-Security` из `resp.headers`. RFC 6797 §8.1 — STS на HTTP-ответе игнорируется (active attacker мог бы её подделать); HTTPS-ответ с валидным `max-age=N` (включая `0` — «снять HSTS») сохраняется через `record_sts`. Делается на КАЖДОМ hop, не только финальном: 3xx-ответ тоже может нести STS-policy. Каждый redirect-hop проверяется независимо — симметрично с filter/sink/resolver. Phase 0 ограничения: `parse_sts_header` дублируется (lumen-storage::hsts + lumen-network::hsts) — 30 LOC, чистая функция; общая реализация в lumen-core поднимала бы все консьюмеры в core, что хуже изоляции. Header-lookup case-insensitive (RFC 7230 §3.2).
- **Готово (HTTP Range requests — RFC 7233 single-range + suffix + If-Range):** `HttpClient::fetch_range(url, RangeSpec, Option<RangeValidator>) -> Result<RangeResponse>`. Модуль `crates/network/src/range.rs` экспортирует: `RangeSpec` enum c вариантами `Closed { start, end }` / `OpenEnded { start }` / `Suffix { length }` (RFC §2.1 byte-range-spec + suffix-byte-range-spec); конструкторы `closed(s, e)`, `from(s)`, `suffix(n)`; **`RangeValidator`** enum `ETag(String) | LastModified(String)` (RFC §3.2) — header копируется в `If-Range` дословно без трансформации (spec не требует, и weak-ETag сохраняется как W/"..."); `ContentRange { start, end, total: Option<u64> }` (inclusive end, `total=None` для `*`); `RangeResponse { status, body, content_range: Option<ContentRange> }`; `parse_content_range(value) -> Option<ContentRange>` (строгий — лишние пробелы между числами отвергаются). `write_request` принимает `Option<&RangeSpec>` + `Option<&RangeValidator>` и вставляет `Range: bytes=START-END` / `bytes=START-` / `bytes=-N` + (только при наличии валидного Range) `If-Range: <validator>`. Невалидный spec (`end < start` в Closed, `length=0` в Suffix) тихо опускает Range, и If-Range тогда тоже опускается (RFC §3.2 — header без Range не имеет смысла). **Response семантика**: 206 Partial Content → `content_range` распарсен; 200 OK (либо §3.1 ignore Range, либо §3.2 If-Range mismatch — ресурс изменился, full new body) → full body, `content_range=None`; 416 / прочие 4xx/5xx → `Err`. **Redirect (3xx)**: Range + If-Range пробрасываются на target. **Phase 0 не поддерживает**: multi-range `bytes=0-99,200-299` (multipart/byteranges — отдельный пайплайн боундари; нужно для `<video>` seek). Применение в lumen-shell (resume Downloads, `<video>` seek) — задача интеграции.
- **Готово (HTTP auth — RFC 7235 + 7617 Basic + 7616 Digest):** `HttpClient::with_credentials(Arc<dyn HttpCredentialProvider>)` подключает credential-провайдер. Trait-точка `lumen-core::ext::HttpCredentialProvider` принимает `HttpAuthChallenge { origin, realm, scheme: HttpAuthScheme::{Basic|Digest} }` и возвращает `Option<HttpCredentials { username, password }>`. Реализация `StaticCredentialProvider` (in-memory таблица с fallback-chain `(origin,realm) → (origin,"") → ("",realm) → ("","")`) для тестов / curl-style конфигурации; UI-popup + keyring — следующие задачи на P4 / платформенном слое. Модуль `crates/network/src/auth.rs` изолированно содержит парсер `WWW-Authenticate` (RFC 7235 §2.1: token / quoted-string с `\` escape, корректное различение «,» как separator challenges vs auth-param), `select_best_challenge` (Digest+SHA-256 > Digest+MD5 > Basic; чужие схемы Bearer/Negotiate/NTLM возвращают None), builder-ы `build_basic_authorization` (RFC 7617 §2: `Basic base64(user:pass)` с RFC 4648 §4 стандартным алфавитом — отдельным от base64url в doh.rs) и `build_digest_authorization` (RFC 7616 §3.4: HA1 / HA2 / response по MD5 / MD5-sess / SHA-256 / SHA-256-sess; qop=auth + RFC 2069 legacy без qop; per-request `nc` через `AtomicU32` + псевдослучайный `cnonce` из timestamp+counter — `cnonce` anti-chosen-plaintext, не secret), и собственный **MD5** (RFC 1321) локально + **SHA-256** через общий `lumen_core::hash::sha256_hex` (один источник истины с Safe Browsing и SRI) — не security-критично (Digest = challenge-response, не KDF/хранение паролей), rustls/ring оставлены для TLS, где crypto реально нужен. **401 retry loop в `fetch_with_redirect`**: основной цикл переписан с `let mut authorization: Option<String> = None`; первый запрос hop-а без `Authorization`; при 401 + provider + `WWW-Authenticate` → парсим, выбираем strongest, спрашиваем provider, собираем header → `continue`. Один retry на hop: повторный 401 = `Err`. Любая неудача в цепочке (нет header-а, неподдерживаемая scheme, provider None, Digest builder вернул None из-за битого challenge) → 401 пробрасывается без retry. **Authorization намеренно НЕ переносится на 3xx-redirect**: рекурсивный `fetch_with_redirect` для redirect-target начинается с `None` и опросит provider под новый origin (RFC 7235 §3.1 — «implementations SHOULD NOT use credentials with arbitrary URIs»). **Не делается preemptive Basic auth** — `Authorization` уходит только после `WWW-Authenticate` challenge (RFC 7235 §2.1: server controls credential negotiation). Phase 0 ограничения: `qop=auth-int` (требует hash от body, у GET нечего хэшить), `SHA-512-256`, `Authentication-Info` / nextnonce, proxy-auth (`Proxy-Authenticate`/`Proxy-Authorization` на 407 — нет concept-а прокси), client certificates (mTLS).
- **Готово (Content-Encoding pipeline + Brotli):** `HttpClient::with_content_decoder(Arc<dyn ContentDecoder>)` регистрирует декодер из `lumen-core::ext::ContentDecoder` для одного encoding-имени. Можно вызывать многократно — порядок регистрации = порядок в `Accept-Encoding`. Внутренне держится `decoders: Vec<Arc<dyn ContentDecoder>>` + private `accept_encoding_header() -> Option<String>` (`Some("br, gzip, ...")` или None если декодеров нет). `Connection::write_request` принимает `Option<&str>` для `Accept-Encoding`: пустое значение или None — header вообще не пишется (server трактует как «identity only»). После `read_response` функция `apply_content_encoding(body, headers, decoders)` парсит header-значение `Content-Encoding` (case-insensitive, comma-separated), пропускает `identity` и пустые токены, для каждого encoding ищет декодер в Vec и применяет в **обратном** порядке к header-у (RFC 7231 §3.1.2.2: «content codings are listed in the order in which they were applied» — последнее объявленное снимать первым). Encoding, для которого нет зарегистрированного декодера → `Err(Error::Network("unsupported Content-Encoding 'X' (no decoder registered)"))` — это лучше, чем возвращать пользователю мусор (server нарушил Accept-Encoding contract). Decoding применяется только к финальному (не редирект) 2xx-ответу: тела редиректа выбрасываются, для 4xx/5xx caller получает `Err` по статусу. **`BrotliContentDecoder`** (`crates/network/src/brotli.rs`) — реализация `ContentDecoder` (`encoding = "br"`) поверх **provisional `brotli-decompressor` = "5"** (RFC 7932, §5 «Provisional accelerators»). Stateless (`Send + Sync`), `decode()` = `Decompressor::new(input, 4096).read_to_end(...)`. Без brotli 60–70% реального веба отдают `Content-Encoding: br` и страница декодироваться не может — это блокер для роадмап-пункта «реальный сайт показывает читаемый HTML». Graduation criterion: edge-case в RFC 7932 (формат стабилен с 2016, реалистично — никогда). Phase 0 ограничения: full-body decode (без streaming), gzip / deflate / zstd — не зарегистрированы (декодеры реально пишутся следующими подзадачами; serve и так через identity сейчас).

### `lumen-shell` 🟡 (окно + рендер + сеть)

- **Готово:** winit 0.30 с `ApplicationHandler` API. Три режима окна: `lumen` (пустое окно 1024×720), `lumen <path.html>` (файл → кодировка → HTML → layout → paint), `lumen <http(s)://...>` (сеть через `HttpClient` → те же этапы). Внешний CSS: `<link rel="stylesheet" href="...">` загружается с диска (относительно HTML-файла) или по сети (относительно базового URL). `ResourceBase` enum изолирует логику разрешения относительных URL. Inter-Regular.ttf bundled через `include_bytes!`. Обработчики Resized + RedrawRequested.
- **Готово (fetch+decode `<img src>`):** между HTML parse и layout shell обходит DOM, для каждого `<img>` с непустым `src` скачивает байты (file:// или http через `HttpClient`), декодирует через `lumen_image::decode` (PNG/JPEG sniff-dispatch). `fetch_and_decode_images(doc, base, sink) -> Vec<(String, lumen_image::Image)>`: первый pass — `collect_img_entries` собирает `(NodeId, src, has_width_attr, has_height_attr)` без mutate; второй pass — fetch (через `ResourceBase::resolve` + `std::fs::read` или `HttpClient::fetch`) + decode + intrinsic dimensions. **Intrinsic dimensions (HTML5 §10 «mapped attributes»):** если у `<img>` не задан НИ `width` НИ `height` attribute, оба проставляются из декодированного изображения как presentational hints через `apply_intrinsic_size`; author CSS перекроет (mapped attributes — самый низкий приоритет в каскаде). Если хоть один атрибут задан author-ом, не лезем (он знает, что делает с aspect-ratio). Ключ в результирующем Vec — **raw href** из `src` attribute, не resolved URL, потому что layout сохраняет тот же raw href в `BoxKind::Image { src, alt }` → `DisplayCommand::DrawImage.src`, и render-side `Renderer::register_image(src, &image)` находит свою загрузку прямым HashMap lookup-ом без отдельной нормализации. Errors fetch/decode silently пропускаются с `eprintln!` диагностикой — битая картинка не валит рендер всей страницы, paint рисует серый placeholder fill-quad через fallback в renderer. `LoadedPage`/`ParsedPage` теперь несут `images: Vec<(String, lumen_image::Image)>`; `Lumen.pending_images` — staging area до создания Renderer-а в `resumed` (заливается через `register_image` и `drain`-ится); reload — `Renderer::clear_images()` + повторная регистрация (content под тем же src мог измениться).
- **Готово (network log):** `StdoutEventSink` — простейший наблюдатель сетевых событий, печатает в **stderr** (а не stdout): `→ GET <url>`, `← <status> <url>`, `✗ <url> (<reason>)`. Подключается к `HttpClient` в shell, чтобы каждый исходящий байт был виден пользователю — это и есть Phase 0 версия network log из принципа №4. Stderr выбран, чтобы dump-режимы не пачкали свой stdout сетевыми событиями (в оконном режиме оба потока идут в терминал — видимо неотличимо). Позже заменится на структурированный UI-логгер (отдельная панель в окне).
- **Готово (window title):** `extract_title(&Document)` находит первый `<title>` в дереве, склеивает текстовые дети и сжимает whitespace через `split_whitespace().join(" ")` (отрабатывает `\n\t`, длинные пробелы). Энтити уже декодированы tokenizer-ом (RCDATA). `LoadedPage { display_list, title }` возвращается из `PageSource::load` / `render_bytes` — единая точка для будущих расширений (favicon, current URL, scroll state). `window_title(Option<&str>)` форматирует заголовок: с title — `"<title> — Lumen"`, без — fallback на `Lumen <version>`. `Lumen::reload` обновляет title окна (`window.set_title(...)`) при reload.
- **Готово (keybindings):** `PageSource` enum (`Empty` / `File(PathBuf)` / `Url(String)`) хранится в `Lumen` вместе с `Arc<dyn EventSink>` — `PageSource::load(sink)` детерминированно перезапускает fetch/parse/layout/paint. `KeyCommand` enum (`Reload` / `Exit`) и `keybinding_for(KeyCode, ModifiersState) -> Option<KeyCommand>` — изолированный от winit маппер shortcuts (F5 / Ctrl+R → Reload, Esc / Ctrl+W → Exit), тестируется юнитами. В `window_event` обрабатывается `WindowEvent::ModifiersChanged` (обновляет `Lumen.modifiers`) и `WindowEvent::KeyboardInput` (по physical_key + state на pressed, без `repeat`-эхо, маппинг через `keybinding_for`). Force-reload (`Ctrl+Shift+R`) намеренно не привязан — нет cache, чтобы было что обходить; зарезервировано на будущее. `Lumen::reload` на `PageSource::Empty` — no-op, на ошибке оставляет предыдущий display_list и логирует stderr.
- **Готово (runtime — integration в winit-loop):** `Lumen` хранит `runtime: runtime::EventLoop` и `epoch: Instant` для DOMHighResTimeStamp. Три точки диспатча: `about_to_wait` → `step()` в цикле `while StepResult::Ran` до `Idle` (cap 256 task-ов за один tick — защита от runaway, остаток идёт в следующий about_to_wait); `WindowEvent::Resized` → `deliver_observer_records(ObserverKind::Resize)`; `WindowEvent::RedrawRequested` → `run_rendering_step(epoch.elapsed_ms())` перед `Renderer::render()`. В Phase 0 никто не зарегистрирован (JS engine отсутствует), но точки диспатча уже стоят — когда подключим QuickJS, JS-callback-и (ResizeObserver, rAF, postTask) сработают автоматически без правок shell-а. **Reload-через-task** (`KeyCommand::Reload → queue_task(UserInteraction, …)`) отложен: closure task-а требует `Rc<RefCell<Lumen>>` или `Rc<RefCell<Vec<KeyCommand>>>`-канал, потому что `step()` дёргается с `&mut self`, а closure не может re-borrow его. Архитектурный рефакторинг под JS engine принесёт такую же потребность для DOM bindings — делаем тогда обе задачи разом.
- **Готово (runtime — HTML event loop framework):** Модуль `lumen-shell::runtime` (Phase 0 framework, без integration в winit-loop). Реализует HTML Living Standard §8.1.4 «Event loops» + §8.1.5.1 «Update the rendering» в минимальном виде. **`TaskQueue`** — **per-source** FIFO-очереди `Box<dyn FnOnce>` (массив `[VecDeque<Task>; 7]`, индекс через `TaskSource::as_index`). `queue` кладёт в очередь источника, `pop` обходит источники в `TaskSource::PRIORITY_ORDER` (`UserInteraction > DomManipulation > HistoryTraversal > Networking > Timer > Rendering > IdleTask` — соответствует подходу Chromium scheduler; HTML §8.1.4.2 оставляет порядок на усмотрение UA), возвращает первую найденную task. Внутри одного источника — strict FIFO. `total` кэширует суммарную длину, чтобы `len`/`is_empty` оставались O(1); `len_of(source)` — длина конкретной очереди для метрик / отладки. Это даёт UI input приоритет над фоновой загрузкой: `UserInteraction` task, поставленная позже `Networking` task, всё равно выполнится первой. **`MicrotaskQueue`** — drain-all семантика: новый microtask, поставленный из исполняющегося microtask-а, выполняется в этом же checkpoint, а не следующем (§8.1.4.4). **`EventLoop::step()`** — одна task за step + microtask checkpoint после неё; `StepResult::Idle` если task-ов нет, но checkpoint всё равно дренится (чтобы caller мог пушнуть microtask напрямую и сразу её выполнить). **`request_animation_frame(handle: u32) → AnimationFrameHandle` / `cancel_animation_frame` / `run_rendering_step(timestamp_ms: f64)`** — реализуют §8.1.5.1: snapshot текущего rAF-списка через `mem::take` перед итерацией, поэтому recursive rAF из callback-а попадает в **следующий** frame; cancelled handles проверяются перед каждым вызовом и удаляются из `cancelled_raf` set-а; после всех callback-ов — microtask checkpoint. **`request_idle_callback(closure, timeout_ms, now_ms) → IdleCallbackHandle` / `cancel_idle_callback` / `run_idle_callbacks(remaining_ms, now_ms)`** — W3C `requestIdleCallback` §3 в том же snapshot-pattern: `IdleDeadline { time_remaining(), did_timeout() }` передаётся в callback. `timeout_ms` опционально — конвертируется в абсолютный `timeout_at_ms = now_ms + timeout_ms`; при `run_idle_callbacks(_, now)` если `timeout_at_ms <= now` — callback вызывается с `did_timeout=true` и `remaining_ms=0.0` (timeout overrides caller-budget); иначе с `did_timeout=false` и переданным `remaining_ms`. Отрицательный `remaining_ms` клампится до `0.0`. Recursive `request_idle_callback` из тела попадает в следующий `run_idle_callbacks`. После batch — microtask checkpoint (по аналогии с rAF). Phase 0 не реализует «реальный» idle-detection — caller (например, Lumen в `about_to_wait`) сам решает, какой бюджет передать и когда вызывать; чтобы timeout-callback-и срабатывали при перегрузке, caller обязан периодически дёргать `run_idle_callbacks` даже с нулевым бюджетом. **Observer registries** трёх типов (`ObserverKind::Resize / Intersection / Mutation`) — `register_observer(kind, Fn) → ObserverHandle`, `disconnect_observer`, `deliver_observer_records(kind)`. Callback — `Rc<dyn Fn()>` (не `FnOnce`, потому что observer срабатывает многократно); перед delivery — snapshot активного списка через `Rc::clone`, что позволяет callback-у disconnect-ить себя или соседнего observer-а без поломки текущей итерации (изменения видны на следующем delivery). **Reentrancy:** State под `Rc<RefCell<…>>` + тонкий `EventLoopHandle::clone()` (один `Rc`); closure task-а / microtask-а / rAF / observer через handle планирует новые регистрации в ту же очередь без `borrow_mut` conflict-а — pop происходит в коротком borrow, run closure — уже без него. **Threading:** single-threaded (`Rc`/`RefCell`), как и сам HTML event loop; cross-thread обмен (фоновые fetch-и, winit user-events) — через channels на shell-уровне, не внутри runtime. JS engine не требуется: callback — Rust closure; когда подключим QuickJS, JS-функция оборачивается в closure и кидается в ту же очередь. Module помечен `#![allow(dead_code)]` — публичный API сейчас используется только из тестов, allow снимется при integration. 19 unit-тестов: FIFO Task/Microtask, source сохраняется, one task per step, microtask checkpoint после task, nested microtask в том же checkpoint (drain-all), task планирует следующую task (выполняется в next step), idle step всё равно дренит microtasks, len/is_empty для MicrotaskQueue, pending counters; rAF registration order + timestamp, cancel ДО run пропускает, cancel неизвестного handle — no-op, recursive rAF в next frame, microtask из rAF в этом же rendering step, empty rendering step — no-op; observers register/disconnect, delivery только нужного kind, многократный вызов одного observer-а, disconnect-внутри-delivery работает корректно (snapshot защищает текущую итерацию).
- **Готово (headless dump-режимы):** три CLI-флага гоняют pipeline без winit и wgpu, печатают результат фазы в stdout, диагностика в stderr. `--dump-source <path-or-url>` → `lumen_encoding::decode` после `detect`. `--dump-layout <path-or-url>` → `lumen_layout::serialize_layout_tree`. `--dump-display-list <path-or-url>` → `lumen_paint::serialize_display_list` (тот же формат, что у paint snapshot-ов). ### `lumen-bench` ✅ (baseline pipeline measurements)

- **Готово:** отдельный bin-крейт без сторонних зависимостей. Прогоняет `decode → parse_html → parse_css → layout → paint::build_display_list` на `samples/page.html` + `samples/page.css` + bundled Inter; печатает min / median / mean / p95 / max на фазу и TOTAL. `LUMEN_BENCH_ITERS` env var переопределяет число измерений (по умолчанию 100). 10 warm-up итераций перед измерениями. `std::hint::black_box` оборачивает результаты pipeline-а — защита от dead-code elimination в LTO-release-сборке. Запуск: `cargo run -p lumen-bench --release`.
- **Зачем:** до этой задачи цели плана (cold start <300 мс, RAM <100 МБ на пустую вкладку) были лозунгами без точки отсчёта. Теперь регрессии при росте функциональности отслеживаются — каждая фаза должна оставаться в своём бюджете.
- **Baseline (dev profile, samples/page.html 667 B HTML + 300 B CSS, 49 DOM-узлов, 7 CSS-правил, 18 paint-команд, на x86_64 CachyOS):** decode ~2 μs, parse_html ~19 μs, parse_css ~13 μs, layout ~48 μs (доминирует), paint ~1 μs, TOTAL ~85 μs. Release-сборка должна показать примерно те же цифры или быстрее (dev уже на opt-level=3 для deps).
- **Отложено:** более крупные strona-test-cases (реальные статьи в десятки KB), per-phase profile breakdown (cascade vs measure внутри layout), measurement of font parse cost (сейчас амортизированный — один раз перед циклом), CI-trend tracking, тесты как proper #[bench] (требует nightly или criterion как exception #5).
